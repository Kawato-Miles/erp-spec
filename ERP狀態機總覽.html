工單完成條件由 QC 紀錄驅動
需求單 業務 / 諮詢業務 建立；印務主管 評估報價
需求確認中 → 待評估報價 → 已評估報價 → 已提供報價 → 議價中
↳ 成交 → 成交 → 建立訂單
↳ 流失 → 流失 關單，不建立訂單，保留紀錄
↳ 重新提供方案 → 待評估報價 退回，與印務主管重新討論
訂單 業務 / 諮詢業務 建立；會計 確認款項 ｜ 線下以「回簽」驅動生產，線上以「付款」驅動
線下訂單（以回簽驅動）
報價待回簽 → 已回簽 ↓ 進入共用段
付款狀態（平行追蹤，不阻塞主流程）
等待付款 → 已付款
退款狀態 ⏳ 待設計
線上訂單（以付款驅動）
等待付款 → 已付款 ↓ 進入共用段
共用段（已回簽 / 已付款 後）
已回簽 / 已付款 → 稿件未上傳 → 等待審稿 → 製作等待中 → 工單已交付
→ 製作中 → 製作完成 → 出貨未處理 → 打包中 → 物流未完成 → 訂單完成
補收款（平行，不阻塞主流程）
補收款未付款 ↔ 補收款已付款 急件費用 / 重打樣補收等
工單 type 欄位區分：打樣工單 / 大貨工單 ｜ 印務 填寫；印務主管 審核
草稿 → 製程確認中 ↔ 重新確認製程 → 製程審核完成 → 工單已交付
→ 製作中 → 已完成 QC 入庫累計 ≥ 工單目標生產數量
↔ 異動 任務需求異動；生管確認後恢復製作中
工單終態由 QC 紀錄驅動，不設置 QC 狀態節點。打樣工單無「打樣不通過」狀態，打樣 OK/NG 以印件的「打樣結果」欄位管理。
印件 兩個獨立維度：審稿狀態 ＋ 印製狀態 ｜ type 欄位：打樣印件 / 大貨印件
維度一：審稿狀態（兩種印件共用）
稿件未上傳 → 等待審稿 上傳稿件後
↳ 不合格 → 已補件 → 等待審稿 loop，直到合格
↳ 合格 → 合格 → 自動建工單草稿 若有製程設定
稿件管理：印件維護多檔案池，不以版次號管理。工單建立時鎖定特定合格稿件，生產期間不受後續上傳異動。大貨印件打樣全數 OK 後才上傳稿件進入此流程。
審稿合格後：生產任務帶入規則
印件類型 生產任務帶入方式
EC 訂單印件 ✅ 自動帶入（非客製化商品，製程預先定義於 BOM）
線下 — 半客製品 ✅ 參照對應 EC 商品帶入，印務可修改內容
線下 — 參考其他訂單 ✅ 參照指定訂單的印件帶入，印務可修改內容
線下 — 全客製品 ❌ 工單草稿建立，生產任務由印務手動逐一建立
維度二：印製狀態（type 控制初始狀態、欄位顯示與可用操作）
大貨印件 — 初始狀態
等待中 （審稿合格 → 工單草稿建立後進入）
打樣控制為流程層：大貨印件建立時審稿狀態停在「稿件未上傳」，直到打樣 OK 後業務才開放稿件上傳 → 進入審稿 → 建立工單草稿。無「等待打樣確認」狀態。
打樣印件 — 初始狀態 ＋ 打樣結果
等待中 （工單草稿建立後進入）
↳ 打樣結果欄位 待確認 / OK / NG-製程 / NG-稿件
↳ NG-稿件問題 → 製作完成後 → 已棄用 手動觸發
共用段（等待中以後，兩種印件相同）
等待中 → 工單已交付 → 製作中 含：部分已出貨（出貨數 > 0） → 製作完成 所有工單已完成
→ 打包中 → 待出貨 → 出貨中 → 已出貨 → 運送中 → 已送達
打樣印件出貨 = 樣品交客戶確認；大貨印件出貨 = 正式交貨。兩種印件共用出貨流程。
任務（紙本工單） 印務 交付；生管 分派；依工廠類型走不同終態路徑
待交付 → 已作廢 尚未交付前取消
待交付 → 已交付 → 製作中 ↔ 異動 印務發起 → 已確認異動內容 生管確認 → 製作中
製作中後（依工廠類型）
自有工廠 / 加工廠
製作中 → 已完成
外包廠
製作中 → 運送中 → 已抵達 → 已完成
中國廠商
製作中 → 已送集運商 → 運送中 → 已抵達 → 已完成
生產任務 最小追蹤單位；師傅 / 工廠 報工；QC 紀錄掛在此層
待處理 → 已作廢 尚未進入生產即取消
待處理 → 製作中
製作中後（依工廠類型）
自有工廠 / 加工廠
製作中 → 已完成 報工完成數 = 任務數量
外包廠
製作中 → 運送中 → 已完成
中國廠商
製作中 → 已送集運商 → 運送中 → 已完成
報廢（任何工廠）
製作中 → 報廢 已進入生產後取消；費用仍計算
QC 單 ＋ QC 紀錄 印務 建立；QC 人員 執行；紀錄掛在任務下
待執行 印務建立，指定任務 + 目標數量 + 指派QC人員 → 執行中 → 已完成 QC 紀錄產生
↳ 缺口 = 0 → 工單達成「已完成」條件
↳ 缺口 > 0 → 工單維持「製作中」，系統顯示缺口數量，印務手動建新任務補足
QC 紀錄結構：per 影響成品的生產任務，含 生產數量 / QC通過數量 / QC不通過數量。 不影響成品的生產任務（如採購、開版）不建 QC 紀錄，不計入工單完成判斷。
QC 完成判斷（層級傳遞，由下往上）
生產任務 QC 通過數 ≥ 生產任務目標 → 該生產任務 QC 達標
任務 所有「影響成品」生產任務 QC 達標 → 該任務 QC 達標
工單 累計所有任務 QC 入庫數量 ≥ 工單目標生產數量 → 工單 → 已完成
跨實體觸發關係 狀態如何向上 / 向下傳遞
觸發來源 觸發條件 影響 → 狀態變化
需求單 議價中 → 成交 訂單 建立（進入報價待回簽 / 等待付款）
訂單 線下 → 已回簽 ／ 線上 → 已付款 訂單 進入共用段（稿件未上傳）；可開始稿件上傳 → 審稿流程
印件（審稿） → 合格 工單草稿 自動建立；印件（印製） → 等待中啟動
工單 → 工單已交付 印件（印製） → 工單已交付；訂單 → 工單已交付
任務 任一任務 → 製作中 工單 → 製作中；印件（印製） → 製作中；訂單 → 製作中
QC 單 → 已完成（缺口 = 0） 工單 → 已完成（QC 入庫 ≥ 工單目標）
工單（全部） 所有工單 → 已完成 印件（印製） → 製作完成
印件（全部） 所有印件 → 製作完成 訂單 → 製作完成
業務（手動） 打樣結果 = OK（打樣印件欄位） 大貨印件：業務通知客戶上傳稿件 → 審稿狀態從「稿件未上傳」開始流程（流程層控制，非狀態轉換）
業務 / 印務（手動） 打樣結果 = NG-稿件問題 舊打樣印件 → 已棄用；新打樣印件 建立（從等待中開始）
業務 / 印務（手動） 打樣結果 = NG-製程問題 同一打樣印件 下建立新打樣工單（舊工單保留）
⚠️ 待設計：訂單取消時各層單據（工單 / 任務 / 生產任務 / 印件）的作廢連鎖流程（open-questions #24）
