## 1. Problem Statement

審稿通過後工單建立，但系統無法自動化追蹤製程進度、廠商分派、QC 入庫與完成度計算。紙本工單缺乏廠商篩選機制（易遺漏重複分派），異動流程無紀錄（難以追責），導致印務、印務主管、生管的作業效率低落，尤其在多工廠、複雜製程場景下（如一印件多工單、多生產任務）。

---

## 2. Goals

| 目標 | 定義 | 成功指標 |
|------|------|---------|
| **G1**: 工單狀態實時自動化 | 工單狀態自動流轉（審核通過 → 製作中 → 完成），無手動介入 | 狀態轉換 ≤ 1 分鐘自動觸發；98% 準確率 |
| **G2**: 廠商自動分類與紙本工單快速生成 | 按廠商類型（自有/加工/外包/中國）自動聚合，支援一鍵按廠商篩選生成紙本工單 | 生成耗時 < 30 秒；廠商分類零錯誤 |
| **G3**: 精確的工單完成度計算 | 根據 QC 入庫自動計算完成度（採木桶原則），支援複雜場景（多工單、多生產任務各自倍數） | 計算準確度 100%；支援 scenarios.md 情境 2、5 |
| **G4**: 異動流程透明化與可追溯 | 記錄製程變更（新增生產任務、修改倍數、修改 affects_product），自動重算完成度 | 異動紀錄 100% 完整；支援完整審計軌跡 |

---

## 3. Non-Goals

| 項目 | 原因 |
|------|------|
| 與訂單 / QC 單 / 出貨單的集成邏輯 | 後續規劃；v1.0 假設訂單、印件已定義完整 |
| 打樣工單特殊流程 | 邏輯與大貨相同，暫不特殊處理 |
| 工單成本估算與利潤分析 | 屬 Dashboard / BI 範疇 |
| 多語言支持 | v2.0 規劃 |

## 4. User Stories

### 4.1 印務主管

**US-A1：審核工單製程**

業務場景：審稿通過後，印務主管需逐一檢視工單草稿的製程內容（廠商分類、生產任務、倍數等），判斷是否合理、是否缺漏，然後通過審核或退回要求調整。

```
As an 印務主管
I want to 檢視工單草稿的製程詳情（所有任務、生產任務、倍數、affects_product 標記），
       以及對比該印件的預期生產流程
So that I can 快速判斷製程邏輯是否正確，並及時通過或退回

AC：
- 工單詳情頁顯示所有任務與生產任務（含廠商、倍數、設備、影響成品標記）
- 支援批量操作（同時通過/退回多個工單）
- 退回時強制附帶備註（說明退回原因）
- 系統提示「缺少任何 affects_product=TRUE 的生產任務」時無法通過
```

---

### 4.2 印務

**US-B1：建立工單與生產任務**

業務場景：接收分派的工單草稿後，印務根據印件規格與實際製程，新增或編輯每個生產任務（廠商、倍數、設備、影響成品標記），然後提交給印務主管審核。

```
As an 印務
I want to 在工單中新增/編輯/刪除生產任務，並為每個生產任務設定廠商、倍數、設備、
       affects_product 標記
So that I can 準確描述製程需求，確保生管與工廠理解正確的執行步驟

AC：
- 工單草稿支持新增多個生產任務（按廠商自動分類為任務層）
- 每個生產任務可填：名稱、廠商、倍數（quantity_per_work_order）、
  設備（單選，可選）、affects_product 複選框
- 驗證：至少有 1 個生產任務 affects_product=TRUE（否則無法提交審核）
- 新增時系統建議倍數值（供參考，可手動覆寫）
```

**US-B2：交付生產任務給生管**

業務場景：工單通過審核後，印務需按廠商生成紙本工單（自有廠/外包廠/中國廠商各份），發給對應的生管進行派工。

```
As an 印務
I want to 按廠商篩選工單內的生產任務，自動生成 PDF 紙本工單，支持列印
So that I can 快速交付給對應的生管/工廠，減少誤傳或重複分派

AC：
- UI 提供「按廠商篩選生成紙本工單」功能
- 生成的紙本工單僅包含該廠商的生產任務（隱藏其他廠商任務）
- 支援 PDF 下載、瀏覽器列印
- 系統自動記錄交付時間、交付人員
- 工單狀態自動轉為「工單已交付」
```

**US-B3：監控工單完成度**

業務場景：工單進入製作中後，印務需即時掌握各生產任務的進度、缺口，判斷是否需要補生產或調整排程。

```
As an 印務
I want to 在首頁或工單詳情頁實時查看完成度進度（已完成 500 / 1000，缺口 500），
       並支援按廠商/客戶/狀態篩選
So that I can 掌握最新進度，及時判斷是否需要補生產、調整排程

AC：
- 工單詳情頁顯示「已完成 X / 目標 Y（缺口 Z）」
- 實時更新（每次 QC 紀錄提交後重算）
- 首頁儀表板顯示按狀態分組的工單列表（含完成度進度條）
- 支援按廠商/客戶/狀態篩選、導出 CSV
```

**US-B4：發起工單異動**

業務場景：在 QC 不通過或客戶需求變更時，印務需發起異動（新增補生產任務、修改倍數等），系統自動記錄異動內容，由生管確認後推進補生產。

```
As an 印務
I want to 在工單進入「已完成」前發起異動（新增生產任務、修改倍數、修改 affects_product），
       附帶異動原因備註
So that I can 因應 QC 不通過或客戶需求變更，系統自動記錄完整異動軌跡，
       生管可據此快速推進補生產

AC：
- 工單製作中時，點擊「發起異動」按鈕
- 異動類型選項：新增生產任務、修改倍數、修改 affects_product
- 新增生產任務時系統建議倍數（可覆寫）
- 提交異動時強制填寫備註（說明異動原因）
- 異動提交後任務狀態自動變為「異動」
- 系統自動計算修改前後的完成度差異並顯示
```

---

### 4.3 生管

**US-C1：領取並排序生產任務**

業務場景：印務交付紙本工單後，生管需查看分配給自己的生產任務列表（含廠商、交期、優先級），安排生產排程。

```
As a 生管
I want to 在系統中查看分配給我的生產任務列表（含廠商、工序、交期、優先級），
       並支援按狀態篩選、導出 CSV
So that I can 掌握當前待派工的任務，並依優先度安排生產排程

AC：
- 生管首頁顯示「待派工」、「進行中」、「已完成」三個分類的任務列表
- 每筆任務顯示：廠商、工序、目標數量、交期、優先級
- 支援按廠商/狀態/交期篩選
- 支援導出 CSV
- 交期倒數提醒（< 3 日用紅色標記）
```

**US-C2：確認異動並決定補生產時機**

業務場景：印務發起異動後，生管需確認異動內容（新增的生產任務、修改的倍數等），並決定何時返回「製作中」開始補生產（見 scenarios.md 情境 5）。

```
As a 生管
I want to 檢視印務發起的異動內容（新增生產任務、修改倍數等），確認邏輯正確，
       然後點擊「返回製作中」決定何時啟動補生產
So that I can 確保異動內容符合業務邏輯，及時推進補生產流程，
       並在系統中留下完整的異動確認紀錄

AC：
- 異動通知進入生管的「待確認異動」清單
- 異動詳情頁顯示：原任務內容、新增的生產任務、修改的倍數/affects_product
- 生管確認後，任務狀態變為「已確認異動內容」
- 點擊「返回製作中」後，新生產任務自動交付給工廠（自有廠無需紙本；
  外包廠/中國廠須由印務重新印製紙本）
- 系統自動重算工單完成度
```

## 5. Requirements

### P0（Must-Have - 核心業務流程）

| ID | 需求 | 詳細說明 | AC |
|----|------|---------|-----|
| **WO-001** | 工單狀態機 | 完整狀態流轉：草稿 → 製程確認中 ↔ 重新確認 → 製程審核完成 → 工單已交付 → 製作中 ↔ 異動 → 已完成 | 状態自動推進；每個狀態轉換記錄時間戳 |
| **WO-002** | 廠商分類與任務聚合 | 根據廠商類型（自有/加工/外包/中國）自動分類同廠商為一個任務；任務內可包含多個生產任務 | 廠商分類邏輯對應 OQ-002 答案；驗證廠商分類零錯誤 |
| **WO-003** | 生產任務欄位 | 支援：名稱、廠商、倍數（quantity_per_work_order）、設備（單選，可選）、affects_product（布林） | 每個欄位必填驗證；設備為可選欄位 |
| **WO-004** | 工單完成度計算 | 四層統計邏輯（生產任務 → 任務 → 工單 → 印件），採木桶原則，支援 affects_product 篩選 | 對應 quantity-calculation-rules.md 完整邏輯；驗證準確度 100% |
| **WO-005** | QC 入庫觸發完成判斷 | 每次 QC 紀錄提交時自動重算工單完成度；累計 ≥ 目標時自動轉「已完成」 | 系統日誌記錄每次重算時間、結果 |
| **WO-006** | 異動流程（印務發起 + 生管確認） | 印務發起異動（新增生產任務、修改倍數、修改 affects_product）→ 任務狀態變「異動」 → 生管確認「已確認異動內容」 → 生管點擊「返回製作中」啟動補生產 | 完整異動軌跡記錄；自動重算完成度 |
| **WO-007** | 紙本工單按廠商篩選生成 | 選擇廠商後一鍵生成 PDF，僅含該廠商的生產任務，支援列印 | 生成耗時 < 30 秒；自動記錄交付時間 |
| **WO-008** | 工單交付狀態推進 | 印務點擊「交付」→ 工單狀態自動變為「工單已交付」；系統記錄交付時間戳 | 状態自動推進 ≤ 1 分鐘 |

## 8. 資料模型

### WorkOrder（工單）

| 欄位名稱 | 英文鍵名 | 必填 | 型別 | 說明 |
|---------|---------|:--:|------|------|
| 系統 ID | `id` | ✓ | UUID | PK；系統自動生成 |
| 工單編號 | `wo_no` | ✓ | varchar(20) | 格式待確認；系統自動產生 |
| 所屬印件 | `print_item_id` | ✓ | FK | → PrintItem |
| 工單類型 | `type` | ✓ | enum | 打樣 / 大貨 |
| 狀態 | `status` | ✓ | enum | 見 state-machines.md |
| 每份印件生產數量 | `quantity_per_print_item` | ✓ | decimal(12,2) | 例：1000、0.5 |
| 預計生產數量 | `target_quantity` | ✓ | int | 印務自行決定 buffer |
| 負責印務 | `assigned_to` | ✓ | FK | → User（角色=印務） |
| 建立時間 | `created_at` | ✓ | datetime | 系統自動 |
| 更新時間 | `updated_at` | ✓ | datetime | 系統自動 |

|---------|---------|:--:|------|------|
| 系統 ID | `id` | ✓ | UUID | PK；系統自動生成 |
| 所屬工單 | `work_order_id` | ✓ | FK | → WorkOrder |
| 工廠 ID | `factory_id` | ✓ | varchar(100) | 廠商識別碼 |
| 工廠類型 | `factory_type` | ✓ | enum | 自有工廠 / 加工廠 / 外包廠 / 中國廠商 |
| 狀態 | `status` | ✓ | enum | 見 state-machines-ops.md；包含「異動」狀態 |
| 目標數量 | `target_quantity` | ✓ | int | |
| 建立時間 | `created_at` | ✓ | datetime | 系統自動 |
| 更新時間 | `updated_at` | ✓ | datetime | 系統自動 |
