# 審稿流程擴充：合格後返回重審 + 已確認可製作

**日期**：2026-06-09
**狀態**：設計完成，待進入 OpenSpec change 工作流

---

## 問題描述

線下單（B2B）場景中，客戶在印件通過審稿合格後仍可能要求改稿。現行設計「合格為終態」，改稿只能走「棄用原印件 + clone 新印件」路徑，存在兩個痛點：

1. **操作繁瑣**：業務需手動建新印件、重新上傳稿件、重新走一輪審稿
2. **語意不對**：同一印件改稿不應產生多筆紀錄，clone 造成訂單管理混亂

---

## 設計決策摘要

| 項目 | 決策 |
|------|------|
| 新狀態 | 審稿維度新增「已確認可製作」（新終態）與「待改稿」（新中間態） |
| 退回重審 | 「合格」狀態的印件可退回「待改稿」；「已確認可製作」後不可退回 |
| B2C 分流 | B2C 合格後系統自動跳到「已確認可製作」；B2B 線下單業務手動確認 |
| 操作粒度 | 逐件操作，不影響同訂單其他印件 |
| 重審分派 | 預設原審稿人員，有問題走既有審稿主管覆寫機制 |
| 訂單狀態 | 維持在製作段不退回，搭配警示標記提示「有印件待改稿」 |
| 與 clone 的關係 | 並存。退回重審用於「同一印件改稿」；棄用 + clone 用於「印件規格全換」 |
| 退回原因 | 必填，供審稿人員參考 |
| 設計方案 | 方案 A：審稿維度內擴充（語意清晰、改動集中、與現有架構一致） |

## 一、印件審稿維度狀態機變更

### 變更後（8 狀態，已確認可製作為新終態）

```
稿件未上傳 --> 等待審稿 --> 合格 -----> 已確認可製作（新終態）
                |             |
                |             +-> 待改稿（新狀態）--> 等待審稿（重審）
                |
                +-> 不合格 <--> 已補件

免審快速路徑：稿件未上傳 --> 合格（不變）
```

### 新增轉換規則

| 轉換 | 觸發者 | 條件 | 動作 |
|------|--------|------|------|
| 合格 --> 已確認可製作 | 業務 / 諮詢（B2B 手動）；系統（B2C 自動） | B2B：業務手動操作；B2C：合格後系統自動 | 觸發工單建立流程（B2C 自動建 + 帶生產任務；B2B 建空工單草稿） |
| 合格 --> 待改稿 | 業務 / 諮詢 | 必填退回原因 | 記錄退回原因；通知客戶需重新上傳稿件；審稿指派保留原審稿人員 |
| 待改稿 --> 等待審稿 | 系統（客戶上傳新稿後自動） | 客戶已上傳新稿件 | 新增一輪 ReviewRound；原審稿人員收到重審通知 |

### 不變的部分

- 不合格與已補件的既有雙向轉換不變
- 免審快速路徑不變（免審後直接到合格，B2C 自動跳到已確認可製作，B2B 業務手動確認）
- 棄用 + clone 路徑保留（用於印件規格全換的場景）

## 二、訂單狀態 Bubble-up 派生規則變更

### 變更後派生規則

| 條件 | 訂單狀態 |
|------|---------|
| 任一印件處於等待審稿 / 不合格 / 已補件 / 待改稿 | 審稿中 |
| 所有印件至少合格，但有印件尚未確認可製作 | 審稿中 |
| 所有印件已確認可製作 | 製作等待中（觸發工單建立） |
| 部分印件已確認可製作（已建工單），部分印件被退回待改稿 | 製作中（不退回審稿段，搭配警示標記） |

### 警示標記機制

當訂單處於製作段，但有印件狀態為「合格」或「待改稿」時：
- 訂單列表顯示警示標記（例如「1 件待改稿」）
- 業務的待辦清單中列出該印件
- 不影響已建工單的生產流程

### 段落不可逆規則

- 訂單一旦進入製作段，不因單一印件退回而退回審稿段
- 只有「所有印件都尚未確認可製作」的情境下，訂單才停留在審稿段

## 三、角色權限

| 操作 | 業務 / 諮詢 | 審稿主管 | 審稿人員 | 印務主管 | 系統 |
|------|------------|---------|---------|---------|------|
| 確認可製作（B2B） | 可 | 不可 | 不可 | 不可 | -- |
| 確認可製作（B2C） | -- | -- | -- | -- | 自動 |
| 退回重審（合格 --> 待改稿） | 可 | 不可 | 不可 | 不可 | -- |
| 填寫退回原因 | 必填 | -- | -- | -- | -- |
| 覆寫重審指派人員 | 不可 | 可 | 不可 | 不可 | -- |
| 執行重審 | 不可 | 不可 | 可 | 不可 | -- |

### 操作位置

- **確認可製作**：訂單詳情頁，印件卡片上的操作按鈕（僅「合格」狀態的印件顯示）
- **退回重審**：同上（僅「合格」狀態的印件顯示），點擊後彈出退回原因填寫框

### 審稿人員端

- 收到重審印件時，待審清單標示「重審」（與一般新件區分）
- 可查看業務填寫的退回原因
- 審稿流程與一般審稿相同（合格 / 不合格判定）

## 四、工單建立時機與邊界處理

### 工單建立時機變更

| | 現況 | 變更後 |
|--|------|--------|
| 觸發點 | 印件合格 | 印件「已確認可製作」 |
| B2C | 合格後自動建工單 + 帶生產任務 | 合格後自動跳到已確認可製作，再自動建工單（行為不變） |
| B2B | 合格後建空工單草稿 | 業務確認可製作後，建空工單草稿 |

### 邊界情境

| 情境 | 處理方式 |
|------|---------|
| 印件合格後業務遲遲不確認可製作 | 無自動催促，靠訂單警示標記提醒。不設超時自動確認 |
| 免審印件 | 免審直接到合格，B2C 自動確認 / B2B 手動確認。合格狀態下同樣可被退回重審（退回後走正常審稿流程，不再免審） |
| 待改稿印件客戶遲遲不補稿 | 與現行「不合格後客戶不補件」的處理方式一致，不另設新規則 |
| 檔案鎖定時機 | 不變。工單建立時鎖定加工檔與縮圖，時機跟著後移到「已確認可製作」後 |

## 五、連帶影響範圍

| 實體 | 影響程度 | 說明 |
|------|---------|------|
| 印件 | 高 | 審稿維度狀態機核心改動 |
| 訂單 | 高 | Bubble-up 派生規則修改 + 警示標記 |
| 工單 | 高 | 建立時機從「合格」後移到「已確認可製作」 |
| 任務 / 生產任務 | 中 | 連帶工單後移 |
| 需求單 | 低 | 不受影響 |
| 出貨單 | 低 | 不直接影響 |

## 六、與現有設計的相容性

| 現有設計 | 是否相容 | 說明 |
|---------|---------|------|
| 棄用 + clone 路徑 | 相容（並存） | 退回重審用於同一印件改稿；clone 用於印件規格全換 |
| B2C 自動建工單 | 相容 | 合格後自動跳到已確認可製作，行為不變 |
| 免審快速路徑 | 相容 | 免審直接到合格，後續同一般流程 |
| 段落不可逆規則 | 相容 | 訂單不因單一印件退回而退回審稿段 |
| 檔案鎖定機制 | 相容 | 鎖定時機跟工單建立，邏輯不變 |
| 審稿分派規則 | 相容 | 重審預設原審稿人員，覆寫走既有機制 |
