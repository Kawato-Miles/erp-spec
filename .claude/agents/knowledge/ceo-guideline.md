## 一、印刷業核心業務邏輯

印刷業是典型的 **Job-shop 生產模式**：每張訂單規格不同（尺寸、材質、色數、加工方式），無法標準化批次生產。這決定了 ERP 的核心挑戰——在高度客製化的環境中維持生產效率與報價精確性。

**業務流程的真實順序：**
客戶詢價 → 業務報價（依材料成本、機台工時、加工費估算）→ 客戶確認 → 收稿件 → 審稿（尺寸、出血、解析度、色彩模式）→ 拼版 → 製版 → 上機印刷 → 後加工（裁切、折疊、騎馬釘、膠裝等）→ QC → 出貨

每個環節都可能因客戶修改稿件而重來，這是印刷業最常見的延誤原因。版本管理與稿件狀態是生產能否準時完成的關鍵。

**關鍵成本結構：**
- 材料成本（紙張占大宗，約 40–60%）
- 製版費（每次修改都需重新製版，是隱性的單次成本）
- 機台折舊與人工（固定成本高，閒置損失大）
- 外包後加工費（燙金、軋型、特殊表面處理多外包）

報價精確性直接影響毛利率。業務報價錯誤是中小型印刷廠最常見的虧損來源。

## 二、ERP 導入失敗的真實原因

印刷業 ERP 失敗通常不是技術問題：

**1. 系統設計假設了「理想流程」**
設計者認為業務會先報價、確認後才接單。現實是客戶口頭確認就開始備料，補單是常態。系統若要求完整資料才能建單，業務會直接跳過系統。

**2. 報價與生產之間的資料斷層**
報價時的用料預估與實際生產用料常有差距。ERP 若不能快速同步這個差距，業務與生管的資料永遠不一致，大家回頭用 Excel。

**3. 生產現場的使用阻力**
印刷操作員不習慣電腦報工，老師傅尤其抗拒。若系統要求每個步驟都輸入資料，他們寧願事後補輸或根本不輸，導致即時進度數據失真。

**4. 急單與插單沒有設計進去**
急單是印刷業的常態，排程必須支援插單重排。若系統排程功能太剛性，生管寧願繼續用白板或 LINE 群組。

**5. 版本與稿件管理獨立於 ERP 之外**
稿件版本（v1、v2、final、final_final）若沒有整合進工單流程，審稿確認的節點就無法成為生產的觸發條件，品質問題容易在交接中遺漏。

## 三、各角色的真實行為模式

了解真實行為才能設計出被實際使用的系統：

**業務的行為模式：**
- 習慣用口頭或通訊軟體接單，正式建單通常事後補
- 同一客戶的重複訂單（如每月印製月曆），期望能快速複製上次訂單參數
- 急件報價憑經驗估算，不會精確計算成本
- 客戶改稿是家常便飯，希望系統能快速記錄版本差異與追加費用

**印務的行為模式：**
- 排程以機台為中心，習慣用「機台使用率」思考，不習慣以「訂單交期」為優先
- 急單插入後，不喜歡重新印出整份排程，只想看影響到的部分
- 廢張比例心裡有數，但不習慣輸入系統（認為這是揭露自己的失誤）

**出貨的行為模式：**
- 部分出貨是常態（客戶要求分批、或部分規格未完成）
- 出貨前 QC 確認常被時間壓力壓縮（趕工時直接出）
- 出貨單與發票的核對是最常出錯的環節

## 四、對 CEO 真正有價值的業務指標

ERP 對老闆有沒有價值，最終看這幾個數字：

| 指標 | 說明 | 為什麼重要 |
|------|------|----------|
| 訂單準交率 | 準時出貨的訂單比例 | 客戶黏著度的直接指標 |
| 報價轉單率 | 報價成交比例 | 業務效率與報價競爭力 |
| 廢張率 | 廢張佔總用紙比例 | 材料成本控制 |
| 機台稼動率 | 實際生產時間 / 可用時間 | 固定成本攤提效率 |
| 平均訂單毛利率（依客戶 / 品項）| 識別虧損訂單與客戶 | 聚焦對的業務 |
| 改稿次數與費用追回率 | 客戶改稿產生的追加費用是否收到 | 直接影響毛利 |

**評估一個 ERP 功能有無價值，CEO 會問的問題：**
- 這個功能能幫我少掉幾個電話或 LINE 訊息？
- 這個功能讓我的人少做了哪一件重複的事？
- 這個功能讓我第一次看到原本看不到的資訊是什麼？

## 五、當前趨勢（由 knowledge-updater 定期更新）

> 以下內容每次更新時，knowledge-updater 會綜合最新搜尋結果重寫此段。穩定的 Domain Know（§ 一至四）不在更新範圍。

**印刷業 MIS / ERP 市場動態：**
- 印刷業 ERP/MIS 向雲端化與 SaaS 訂閱模式轉移，中小型廠商逐漸接受不買斷授權
- 客戶自助下單平台（Online Ordering / Web-to-Print）成為競爭差異化重點，ERP 需要提供 API 整合能力
- 數位印刷與傳統印刷混合訂單增加，ERP 需同時支援兩種生產邏輯

**B2B SaaS 產品設計趨勢：**
- 產品設計從「功能完整」轉向「核心流程的極致簡化」，降低使用阻力是留存關鍵
- 移動端操作（生產現場報工）需求增加，但印刷業仍以電腦版為主
