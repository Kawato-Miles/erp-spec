# 三視角多輪討論協議

> 適用情境：所有 senior-pm + ceo-reviewer + erp-consultant 三方共同審查的場合，
> 包含 erp-spec Step 4.5、CLAUDE.md 路由觸發的商業目標討論、主動收尾的設計審查。

---

## 核心原則

三個 agent 不是「各說各話後由 Claude 彙整」，而是**自主討論至收斂，再把結論交給 Miles**。
Claude 的角色是輪次協調者，不是彙整者。

---

## 執行流程

### Round 1（平行啟動）

三個 agent 各自完成背景載入後，輸出初始立場。

每個 agent 的 Round 1 輸出必須包含：
1. **核心立場**：針對本次議題，從自身視角提出最重要的 2-3 個觀察或問題
2. **預期分歧點**：哪些地方你預期其他 agent 可能與你看法不同？為什麼？
3. **前提假設**：你的立場依賴哪些假設？若假設不成立，你的結論會如何改變？

---

### Round 2（平行，帶入 Round 1 全文）

每個 agent 收到另外兩個 agent 的 Round 1 完整輸出後回應。

每個 agent 的 Round 2 輸出必須包含：

**對每個 agent 的 Round 1 立場逐點回應：**
- 同意：說明為什麼同意，有無補充觀點
- 部分同意：說明哪部分同意、哪部分有保留，保留的理由是什麼
- 不同意：說明不同意的具體理由，並提出你認為更合理的方向

**新增觀察（若有）：**
看到其他 agent 的立場後，是否發現自己 Round 1 遺漏的問題？

**本輪立場摘要：**
經過交叉討論後，你目前的核心立場是什麼？是否有任何調整？

---

### 收斂判斷（Claude 執行，不告知 Miles）

Claude 在 Round 2 完成後判斷：

| 情況 | 處置 |
|------|------|
| 所有核心議題都達到「同意」或「部分同意（差異可接受）」 | 進入最終彙整 |
| 仍有明確「不同意」且影響結論 | 執行 Round 3 |
| 討論陷入循環（雙方各說各話但無新論點）| 強制進入彙整，標記為「未解爭議」 |

---

### Round 3（視需要，最多執行一次）

只針對 Round 2 中仍有「不同意」的議題：

每個 agent 的 Round 3 輸出：
1. **最終立場**：在這個爭議點上，你的最終判斷是什麼？
2. **讓步條件**：在什麼條件下你願意接受對方觀點？
3. **若無共識**：這個分歧對 Miles 的決策有什麼影響？Miles 需要做什麼選擇？

---

### 最終彙整（Claude → Miles）

所有輪次完成後，Claude 輸出給 Miles 的格式：

```
[三視角審查結論]

討論進行：X 輪（Round 1 → Round N）

共識結論（三方同意）：
1. [結論] — 依據：[三個 agent 共同認可的理由]
2. ...

修正立場（初始有分歧，討論後收斂）：
- [議題]：原本 [A 的立場] vs [B 的立場] → 收斂為 [最終共識]，收斂理由：[...]

必須處理的行動項目：
- [行動] — 優先級：[P0/P1/P2] — 負責角色：[BRD 修改 / 轉為 OQ / 記錄風險]

未解爭議（交 Miles 決定）：
- [議題]：senior-pm 認為 [...]；ceo-reviewer 認為 [...]；erp-consultant 認為 [...]
  → Miles 需要決定：[A 方向 vs B 方向，兩者的影響分別是...]
```

---

## 行為規範

- **不允許** agent 在 Round 2 對所有觀點都說「同意」——必須提出至少一個真實的質疑或補充
- **不允許** 空洞回應如「這個觀點很有道理」，必須說明具體的同意或不同意理由
- **不允許** Claude 在討論未收斂前就彙整給 Miles
- Round 3 之後無論是否收斂，都必須輸出給 Miles——剩餘分歧透明列出，不靜默跳過
- 整個討論過程（Round 1-3）Miles 不可見；只有最終彙整才呈現給 Miles
