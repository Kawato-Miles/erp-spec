# 多 Agent 輪次討論協議

> 適用情境：凡兩個以上 agent 共同審查或討論同一議題時，均依此協議執行。
> 不限定參與 agent 數量或組合，由呼叫方在啟動時指定本次參與的 agent 清單。

---

## 核心原則

多個 agent 不是「各說各話後由 Claude 彙整」，而是**自主討論至收斂，再把結論交給 Miles**。
Claude 的角色是輪次協調者，負責帶入上一輪結果並判斷是否收斂，不提前彙整。

---

## 執行流程

### Round 1（平行啟動）

所有參與 agent 各自完成背景載入後，獨立輸出初始立場。

每個 agent 的 Round 1 輸出必須包含：
1. **核心立場**：針對本次議題，從自身視角提出最重要的 2-3 個觀察或問題
2. **預期分歧點**：哪些地方你預期其他 agent 可能與你看法不同？為什麼？
3. **前提假設**：你的立場依賴哪些假設？若假設不成立，你的結論會如何改變？

---

### Round 2+（平行，帶入前一輪全文）

每個 agent 收到**所有其他參與 agent** 的前一輪完整輸出後回應。

每個 agent 的輸出必須包含：

**對每位其他參與 agent 的前一輪立場逐點回應：**
- 同意：說明為什麼同意，有無補充觀點
- 部分同意：說明哪部分同意、哪部分有保留，保留的理由是什麼
- 不同意：說明不同意的具體理由，並提出你認為更合理的方向

**新增觀察（若有）：**
看到其他 agent 的立場後，是否發現自己前一輪遺漏的問題？

**本輪立場摘要：**
經過交叉討論後，你目前的核心立場是什麼？是否有任何調整？

---

### 收斂判斷（Claude 執行，不告知 Miles）

每輪結束後，Claude 判斷：

| 情況 | 處置 |
|------|------|
| 所有核心議題都達到「同意」或「部分同意（差異可接受）」 | 進入最終彙整 |
| 仍有明確「不同意」且影響結論 | 執行下一輪 |
| 討論陷入循環（各方無新論點，立場未移動）| 強制進入彙整，標記為「未解爭議」 |

**最少執行兩輪**（Round 1 + Round 2）。第三輪起視收斂情況決定是否繼續，最多執行至第三輪後強制彙整。

---

### 最終彙整（Claude → Miles）

所有輪次完成後，Claude 輸出給 Miles 的格式：

```
[多 Agent 討論結論]

參與 agent：[列出本次參與的 agent 名稱]
討論進行：X 輪

共識結論（所有 agent 同意）：
1. [結論] — 依據：[各 agent 共同認可的理由]
2. ...

修正立場（初始有分歧，討論後收斂）：
- [議題]：原本 [A 的立場] vs [B 的立場] → 收斂為 [最終共識]，收斂理由：[...]

必須處理的行動項目：
- [行動] — 優先級：[P0/P1/P2] — 處置方式：[BRD 修改 / 轉為 OQ / 記錄風險]

未解爭議（交 Miles 決定）：
- [議題]：[Agent A] 認為 [...]；[Agent B] 認為 [...]
  → Miles 需要在 [A 方向] vs [B 方向] 之間決定；A 的影響是 [...]；B 的影響是 [...]
```

---

## 行為規範

- **不允許** 任何 agent 在 Round 2 以後對所有其他 agent 的觀點都說「同意」——必須提出至少一個真實的質疑或補充
- **不允許** 空洞回應如「這個觀點很有道理」，必須說明具體的同意或不同意理由
- **不允許** Claude 在討論未達最少兩輪前就彙整給 Miles
- Round 3 之後無論是否收斂，都必須輸出給 Miles——剩餘分歧透明列出，不靜默跳過
- 整個討論過程 Miles 不可見；只有最終彙整才呈現給 Miles
