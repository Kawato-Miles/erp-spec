# PRD Sub-issue 模板 — 印刷 ERP 系統（GitHub Issue，Task 層）

> **顆粒度定位**：角色實作層。描述「怎麼做」（How）——只寫該角色負責的實作細節。
> 使用情境、功能邏輯、資料欄位已在 Parent Issue 定義，此處只補充角色需要的實作補充說明，不重複。

---

**Issue Title 格式**：`[ Task ] - {角色} - {模組名稱} - Phase {N}`
**IssueType**：Task
**Label**：依角色（Backend / Frontend / Design）
**Parent**：#{parent_issue_number}

---

## 概述

範圍與背景請參考 Parent Issue：#{parent_issue_number}

---

## Task

> 以下依角色選擇對應區塊，其餘刪除。

---

### Design

- [ ] [設計工作項目，例如：畫面設計、流程圖、元件規格]
- [ ] [Figma 產出項目]

---

### Frontend

- [ ] [前端工作項目，例如：元件行為、互動邏輯、狀態管理]

> 若有 Parent Issue 未涵蓋的前端特定邏輯，補充於此。

---

### Backend

- [ ] [後端工作項目，例如：API 開立、商業邏輯處理、DB Migration]

> 若有 Parent Issue 未涵蓋的後端特定邏輯，補充於此。
