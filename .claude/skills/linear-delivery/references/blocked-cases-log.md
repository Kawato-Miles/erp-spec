## 提煉觀察（待累積後處理，尚未動 rubric）

- **D4 捏造共同 pattern**：實作者見上游（design.md / spec）用「保留 / 沿用 N 個 X」這類**模糊計數措辭**時，傾向自編具體清單（KPI 卡名 / 過濾條件）當定案填入交付。
  - 候選新禁令（待再 ≥ 1 案佐證後提煉進 rubric D4）：「上游只說『保留 / 沿用 N 個 X』而未列具體項時，MUST 標『沿用現況、不在此重新定義』，不得自編具體清單當定案」。
- **D1 另案標示的呈現**：缺口顯性化（標另案）本身對（D4 正例），但呈現時易把內部 OQ 編號帶出（違反 D1）。兩次都出現「標另案 = 對、但帶 OQ 編號 = 錯」。可考慮在 rubric D1 禁令補一句「標『另案』時用業務語言，不帶 OQ 編號」（待累積佐證）。
