# PRD Sub-issue 模板 — 印刷 ERP 系統（GitHub Issue，子功能層）

> **使用說明**：每個子功能建立一個 Sub-issue，掛在對應的 Parent Issue 下。
> 包含具體的功能需求與 UX 流程，供開發者直接執行。

---

**Issue Title 格式**：`[PRD] [模組名稱] - [子功能名稱]`
**Labels**：`prd`, `[模組名稱]`
**Parent Issue**：#[Parent Issue 號]

## 功能需求（Functional Requirements）

> 格式：「系統應…（System shall）」，每條需求含可測試驗收條件。
> 優先級：P0（必做）/ P1（重要）/ P2（Nice-to-have）

| ID | 優先級 | 需求描述 | 驗收條件 |
|----|--------|----------|----------|
| FR-[XXX] | P0 | 系統應 [動作描述] | 當 [條件] 時，系統 [預期行為] |
| FR-[XXX] | P1 | 系統應 [動作描述] | [具體可驗證的結果] |

---

## UX / 操作流程

### 主線流程

```
[角色]：[動作 1]
  → [狀態 / 結果]
  → [動作 2]
    → [成功路徑]
    → [失敗 / 例外路徑]
```

### 錯誤處理

| 情境 | 系統行為 | 顯示訊息 |
|------|----------|----------|
| [使用者輸入無效資料] | 阻擋送出 | 「[欄位名稱] 為必填，請填寫後再繼續。」 |
| [狀態前置條件不符] | 阻擋操作 | 「[說明原因，引導下一步]。」 |
