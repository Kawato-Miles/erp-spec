## Why

業務主管目前無法更改諮詢單 / 需求單 / 訂單三種前段業務單據的負責業務。實務上兩個情境因此無解：(1) 業務離職 / 長假時，名下未結單據無人接手，回到「漏單沒處理」痛點（對應 [[AFT-1-業務離職轉派]] 前段三單據部分）；(2) 日常工作負荷不均，主管無法重新分配。本 change 新增「業務主管改派負責業務」能力，補上此管理缺口。

商業背景與既有規則對齊：
- 角色權責：[業務](../../../memory/erp/ERP_Vault/03-roles/業務.md)、[業務主管](../../../memory/erp/ERP_Vault/03-roles/業務主管.md)
- 三單據實體：[訂單](../../../memory/erp/ERP_Vault/05-entities/訂單.md)、[需求單](../../../memory/erp/ERP_Vault/05-entities/需求單.md)、[諮詢單](../../../memory/erp/ERP_Vault/05-entities/諮詢單.md)
- 相關 OQ（本 change 拍板關閉）：[XM-008](../../../memory/erp/ERP_Vault/08-open-questions/XM-008-業務主管改派負責業務的權限邊界與機制.md)、[CR-4](../../../memory/erp/ERP_Vault/08-open-questions/CR-4-諮詢單認領後改派狀態限制.md)；部分涵蓋 [AFT-1](../../../memory/erp/ERP_Vault/08-open-questions/AFT-1-業務離職轉派.md)

## What Changes

- **新增**：業務主管可於三單據詳情頁逐張改派負責業務（諮詢單改負責諮詢人員、需求單與訂單改負責業務），單張操作、即時生效。
- **明確區分「改派」與「分享」兩種機制**：分享（職務代理 / 檢視，不改負責人，業務本人可做，沿用既有權限管理）；改派（改負責人，限業務主管）。
- **BREAKING（修正既有矛盾）**：收斂「業務角色可自行轉單」為「業務僅可分享、不可改派負責人」；改派負責人收歸業務主管。
- **改派理由結構化**：必選理由分類（離職交接 / 長假代理 / 工作負荷平衡 / 客戶要求換窗口 / 其他）+ 選填補述；此分類為「通知對象 / 分享成員處置 / 稽核留痕」的共同樞紐。
- **稽核留痕**：每次改派寫入活動紀錄，含原負責人 / 新負責人 / 改派時間 / 理由 / 操作主管五要素，供績效歸屬時間切分。
- **完成訂單可改派 + 售後負責人衍生**：訂單未取消即可改派（含已完成），因售後服務的負責人衍生自訂單負責人，改派訂單連帶轉移售後歸屬；已取消訂單禁改派。需求單 / 諮詢單維持終態禁改派（無售後繼承問題）。
- **改派候選人篩選**：候選新負責人 = 具對應模組權限（Role）的使用者（訂單 → 具訂單管理權限、需求單 → 具需求單權限、諮詢單 → 具諮詢權限）；ERP 無部門概念，改派範圍為全公司 + 全程留痕。
- **電商訂單防呆**：電商來源訂單若無負責業務，改派入口 disabled（電商訂單流程後階段才開發）。
- **改派不改狀態**：改派只換負責人，MUST NOT 觸發或改變任何單據狀態（不回退、不推進）。

不涵蓋（範疇邊界）：批次改派、自動改派規則、改審核業務主管 / 改印務主管（不同語意的轉單）、分享機制本身、名下單據停滯主動偵測（後驗 epic）。

## Capabilities

### New Capabilities

（無新 capability；本 change 全為既有 capability 的需求變更）

### Modified Capabilities

- `user-roles`: 收斂「業務角色可轉單」為「業務僅分享」；新增業務主管「跨三單據改派負責業務」職責（全公司）+ 標註「改派 = 管理動作（全公司）≠ 核可 = 部門內」職責邊界；改派候選人以 Role 模組權限篩選。
- `order-management`: 新增「訂單負責業務改派」（未取消即可、含完成）；釐清分享 ≠ 改派；售後服務負責人衍生自訂單負責人、改派連帶轉移。
- `quote-request`: 新增「需求單負責業務改派」；釐清分享 ≠ 改派；成交 / 流失終態禁改派。
- `consultation-request`: 新增「諮詢單負責人改派」（重新指定認領人）；釐清「業務主管代為認領（首次指定）」與「改派（覆寫已有負責人）」為不同途徑。
- `state-machines`: 補「改派 MUST NOT 觸發狀態轉移、MUST NOT 改變單據狀態」+ 三單據終態禁改派約束（訂單已取消、需求單成交 / 流失、諮詢單完成 / 取消）。

## Impact

- **Spec（delta）**：user-roles / order-management / quote-request / consultation-request / state-machines 五份 delta spec。
- **Prototype（`sens-erp-prototype/src/`）**：三單據詳情頁新增「改派負責人」入口（與分享入口分開）+ 共用改派 Dialog（選新負責人 + 必選理由分類 + 選填補述）+ store 改派 action（更新 owner + 寫 ActivityLog + 依理由處置分享成員 + 觸發通知）+ 改派理由 LOV + 候選人以 Role 模組權限篩選。
- **既有可複用**：`supervisorReassignManager` reassign 骨架（架構參考）、`PermissionManagement`（分享機制對比，不改）、ActivityLog 機制、`currentUser.role` 角色守衛 + RoleGuard。
- **角色權責**：業務主管取得首個「跨人改負責業務」寫入權；業務 / 諮詢的「轉單」能力收斂為「分享」。
