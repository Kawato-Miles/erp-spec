## MODIFIED Requirements

### Requirement: 批次派工操作（建立工作包）

原有的「批次派工」操作由「建立工作包」取代。系統 SHALL 在日程面板提供「建立工作包」按鈕（取代原「分派」按鈕）。操作流程：勾選待處理任務 → 點擊「建立工作包」 → 建立工作包對話框 → 確認。

日程面板僅負責建立工作包，不負責工作包的查看與管理（查看在生產任務列表）。

**依賴邊就緒過濾**：
- 日程面板 SHALL 顯示所有「待處理」任務（含依賴邊佇列為零者），以便生管掌握全局
- 任務列下方 MUST 以三欄迷你表格顯示各依賴邊：上游 PT 名 / 目前佇列量 / 消耗比例
- 任一依賴邊佇列 = 0 任務 MUST 以**禁用樣式**（灰階 + 「依賴未滿足」Badge）顯示
- 該任務的**勾選框 MUST 禁用**（無法被勾選入建立工作包）
- 若生管懸停於被禁用的勾選框，Tooltip MUST 顯示阻擋原因（例：「依賴邊 P1-印刷 目前佇列為 0」）

詳細規格見 work-package spec「建立工作包」Requirement。

#### Scenario: 建立工作包取代分派

- **WHEN** 生管在日程面板勾選 2 筆「所有依賴邊佇列 ≥ 1」任務，點擊「建立工作包」
- **THEN** 系統彈出建立工作包對話框（含師傅選擇、備註、確樣需求、任務清單）
- **WHEN** 生管確認建立
- **THEN** 工作包建立完成，任務從日程面板移除（已派工）

#### Scenario: 未選取任務時按鈕不可用

- **WHEN** 生管未勾選任何任務
- **THEN** 「建立工作包」按鈕不顯示

#### Scenario: 依賴邊佇列為零顯示禁用樣式

- **WHEN** 日程面板渲染「依賴邊佇列為 0」的任務
- **THEN** 該任務列 MUST 以灰階樣式顯示
- **AND** 列上 MUST 顯示「依賴未滿足」Badge
- **AND** 該任務的勾選框 MUST 禁用（disabled）

#### Scenario: 任務列下方顯示佇列量表格

- **WHEN** 日程面板渲染有依賴邊的任務
- **THEN** 任務列下方 MUST 以三欄迷你表格顯示：上游 PT 名稱 / 目前佇列量 / 消耗比例
- **AND** 佇列量 = 0 的列 MUST 以紅色強調
- **AND** 佇列量為負值的列 MUST 以橙色強調並顯示「補料不足」標籤

#### Scenario: 懸停顯示阻擋原因

- **WHEN** 生管滑鼠懸停於被禁用任務的勾選框
- **THEN** Tooltip MUST 顯示具體哪些依賴邊佇列為 0、上游任務名稱

#### Scenario: 生管無法將佇列為零任務納入工作包

- **WHEN** 生管嘗試透過 API / 批次操作將「依賴邊佇列 = 0」任務加入工作包
- **THEN** 系統 MUST 拒絕，回報「任務 [編號] 依賴邊佇列為 0，無法派工」

## ADDED Requirements

### Requirement: 派工板依賴邊佇列視覺化

日程面板 SHALL 清楚區分三種任務狀態，協助生管快速判斷可派工範圍：

**視覺狀態分類**：
- **可派工**：`status=待處理 AND 所有依賴邊佇列 ≥ 1` → 正常色樣、勾選框啟用、可批次操作
- **依賴未滿足**：`status=待處理 AND 任一依賴邊佇列 = 0` → 灰階、「依賴未滿足」Badge、勾選框禁用
- **已派工後處理中**：`status=製作中` 或後續 → 不顯示於日程面板（已歸入生產任務列表）

**佇列量顯示位置**：
- 任務列：每個任務列正下方顯示三欄迷你表格（上游 PT / 佇列量 / 消耗比例），每條依賴邊一列
- 派工板標頭：顯示「可派工 N / 依賴未滿足 M」計數摘要
- 「僅顯示可派工」切換：快速過濾掉依賴未滿足任務

**即時更新**：
- 上游 TransferTicket Header 確認送達 → 下游任務的依賴邊佇列即時增加，UI 即時刷新
- 上游 Header 作廢 → 佇列即時減少，UI 即時刷新（包含負值警示）

#### Scenario: 三類任務並陳顯示

- **WHEN** 日程面板有 10 筆待處理任務，其中 3 筆任一依賴邊佇列為 0
- **THEN** 面板 SHALL 顯示全部 10 筆
- **AND** 7 筆為可派工正常色樣
- **AND** 3 筆為依賴未滿足灰階樣式
- **AND** 標頭顯示「可派工 7 / 依賴未滿足 3」

#### Scenario: 僅顯示可派工切換

- **WHEN** 生管點擊「僅顯示可派工」切換
- **THEN** 面板 SHALL 過濾掉所有「依賴未滿足」任務
- **AND** 計數標頭 SHALL 顯示「可派工 7 / 依賴未滿足 3（隱藏）」

#### Scenario: 依賴邊解鎖即時反應

- **WHEN** 日程面板開啟中，某上游 TransferTicket Header 確認送達
- **THEN** 該下游任務的依賴邊佇列 SHALL 即時刷新
- **AND** 若所有依賴邊佇列 ≥ 1，任務樣式 SHALL 從「依賴未滿足」變為「可派工」
- **AND** 計數標頭 SHALL 即時更新

#### Scenario: 任務列下方表格內容

- **WHEN** 任務 BOOK-裝訂 有兩條依賴邊（P1-裁切 比例 1、P2-折頁 比例 5）、目前佇列分別為 70、100
- **THEN** 任務列下方 SHALL 顯示三欄表格兩列：
  - 第一列：P1-裁切 / 70 / 1
  - 第二列：P2-折頁 / 100 / 5
- **AND** 兩列佇列量皆 > 0 → 此任務狀態為可派工

#### Scenario: 佇列負值警示樣式

- **WHEN** 某依賴邊佇列為負（例：-20，表示已生產量超過已送達量）
- **THEN** 任務列下方該邊列 MUST 以橙色強調
- **AND** MUST 顯示「補料不足」標籤
- **AND** 整個任務列維持「製作中」狀態（不阻擋）但提示需補送料
