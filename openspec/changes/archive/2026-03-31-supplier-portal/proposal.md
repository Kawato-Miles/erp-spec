# 工廠平台（Factory Portal）

## Problem

1. **外包廠溝通散落**：外包廠與中國廠商透過 Slack / LINE / 電話接收生產任務、回報進度與報價，生管需手動彙整，報價比對困難，進度追蹤仰賴人工催問
2. **生管入口不明確**：生管的日程面板、分派、報工等功能目前散落在 ERP 各模組中，缺乏統一的工廠平台入口

## Approach

建立工廠平台，統一生管、外包廠商、中國廠商的操作入口。生管在工廠平台內完成日程管理、分派、報工、報價審核等日常作業；外包廠商與中國廠商查看被分派的任務、報價、報工。

工廠平台涵蓋的角色（依 user-roles spec 的平台歸屬）：
- 生管：日程面板、分派、報工、報價審核
- 師傅：師傅任務平台（已在 operator-task-portal change 實作）
- 外包廠商：查看任務、報價、報工
- QC：（Phase 2）
- 出貨：（Phase 2）

中國廠商使用獨立的「中國供應商平台」，功能內容與工廠平台的外包廠商部分相同。

Phase 1 新增功能範圍（本 change）：
- 外包廠商任務清單：查看被分派的生產任務（含工序、數量、交期要求）
- 外包廠商報工：回報完成數量與預計完工日期
- 外包廠商報價：針對分派的生產任務提交單價報價
- 生管報價審核：在日程面板審核供應商報價
- Prototype User 切換：生管角色切換後進入工廠平台（非 ERP 中台）

已完成功能（由其他 change 提供，納入工廠平台範疇）：
- 生管日程面板（wo-dispatch-ux）
- 師傅任務平台（operator-task-portal）

## Capabilities

- factory-portal: 工廠平台 — 統一生管、外包廠商、師傅等工廠角色的操作入口

## Affected Specs

- production-task: 新增供應商報價欄位（unit_price_quoted / quoted_at），新增供應商自助報工 Requirement
- user-roles: 更新外包廠商與中國廠商角色權責（新增「查看分派任務」「報價」「報工」權限）；生管平台歸屬從中台調整至工廠平台
- state-machines: 無新狀態，供應商觸發的狀態轉換沿用既有路徑
- business-processes: 新增報價審核流程（供應商報價 → 生管確認 → 核准 / 退回重報）

## Design Constraints

- 平台限制：僅支援桌機瀏覽器（與 ERP 系統統一）
- 認證：使用 ERP 統一帳號，由 Supervisor/生管 建立並設定平台權限
- 平台分離：外包廠商在工廠平台內操作，中國廠商使用獨立的「中國供應商平台」
- 資料範圍：供應商僅能看到分派給該廠商的生產任務
- 報價隔離：不同供應商的報價互不可見（價格敏感資訊）
- Prototype User 切換：角色切換至生管/外包廠商/師傅時，自動進入工廠平台

## Dependencies

- wo-dispatch-ux change 的生產任務分派機制與生管日程面板
- operator-task-portal change 的師傅任務平台
- 生產任務 spec 的外包廠路徑狀態機

## Open Questions

- 報價流程：是先分派再報價，還是先詢價再分派？
- 報價審核：由生管單獨確認，還是需要印務主管核准？
- 是否需要比價功能（同一任務詢問多家廠商報價後選擇）？
- 供應商帳號的權限粒度：一個廠商帳號看所有任務，還是可分操作員各看各的？
