## Context

外包廠商與中國廠商透過 Slack / LINE / 電話接收任務、回報進度與報價。生管的日程面板等功能散落在 ERP 各模組中，缺乏統一入口。

本 change 將「工廠平台」定義為統一入口，涵蓋生管、外包廠商、師傅等工廠角色的操作。新增外包廠商的任務查看、報價、報工功能，並將生管既有功能（日程面板）納入工廠平台範疇。

前提：
- wo-dispatch-ux change 已定義生管日程面板、外包廠生產任務的分派流程
- operator-task-portal change 定義了師傅任務平台
- 外包廠與中國廠商的狀態機路徑已在 state-machines spec 定義

## Goals / Non-Goals

**Goals:**
- 定義工廠平台為統一入口，涵蓋生管、外包廠商、師傅等角色
- 供應商可查看被分派的生產任務（含工序、數量、交期）
- 供應商可針對任務報價
- 供應商可回報生產進度與標記完成
- 生管可在工廠平台的日程面板審核供應商報價
- Prototype User 切換配合調整：生管進入工廠平台

**Non-Goals:**
- 不設計比價功能（Phase 2）
- 不設計對帳與結算功能（Phase 2）
- 不設計物流追蹤整合（Phase 2）
- 不設計報價模板 / 常用工序快速報價（Phase 2）
- 不設計離線模式
- 不設計簡體中文介面（Phase 2 中國廠商需求）

## Decisions

### Decision 1：工廠平台為統一入口

工廠平台涵蓋以下角色與功能：

| 角色 | 功能 | 來源 |
|------|------|------|
| 生管 | 日程面板（分派、報工、設備覆蓋、異動確認）+ 報價審核 | wo-dispatch-ux + 本 change |
| 師傅 | 師傅任務平台（今日任務、報工） | operator-task-portal |
| 外包廠商 | 任務清單、報價、報工 | 本 change |
| QC | （Phase 2） | — |
| 出貨 | （Phase 2） | — |

Prototype 的 User 切換（RoleSwitcher）調整：
- 生管角色切換後 → 進入工廠平台（側選單顯示日程面板等功能）
- 外包廠商角色切換後 → 進入工廠平台的供應商視圖
- 師傅角色切換後 → 進入工廠平台的師傅任務平台

**理由**：user-roles spec 已將生管、師傅、外包廠商歸屬於工廠平台。統一入口讓同一平台的角色切換更直覺，也反映實際部署結構。

### Decision 2：外包廠商與中國廠商為兩個獨立平台

外包廠商在工廠平台內操作，中國廠商使用獨立的「中國供應商平台」。功能內容相同，狀態路徑有差異：
- 外包廠：製作中 → 運送中 → 已完成
- 中國廠商：製作中 → 已送集運商 → 運送中 → 已完成

**理由**：user-roles spec 將兩者分屬不同平台，獨立部署確保資料隔離，且未來中國廠商平台可獨立擴展。

### Decision 3：報價與生產獨立流程

報價流程（quote_status）與生產狀態機（status）完全獨立：
- 供應商可以先報工再報價，或先報價再報工
- 報價未確認不影響生產進度
- 生產完成不影響報價審核

```
報價流程：待報價 → 已報價 → 已確認
                    ↘ 已退回 → 已報價（重新報價）

生產流程：待處理 → 製作中 → 運送中 → 已完成

兩者平行，互不阻擋。
```

**理由**：實務上外包廠經常先開工再議價。若報價未確認就不能開工，會嚴重影響生產時程。

### Decision 4：桌機瀏覽器設計

工廠平台與 ERP 系統相同，僅支援桌機瀏覽器。所有 ERP 相關平台統一桌機限定。

### Decision 5：ERP 統一帳號登入

所有角色使用 ERP 帳號登入，透過角色切換進入對應功能：
- 生管 → 工廠平台（日程面板 + 報價審核）
- 外包廠商 → 工廠平台（供應商視圖）
- 中國廠商 → 中國供應商平台
- 一個廠商可有多個操作員帳號
- 所有操作員帳號看到同一組任務（依 assigned_factory 篩選）

### Decision 6：報價表單設計

供應商報價只填：
- **單價**（必填）：每單位的價格
- **備註**（選填）：報價說明

不含：稅額計算、運費、MOQ 等複雜定價欄位。Phase 1 簡化為純單價。

**理由**：目前外包廠報價都是口頭報單價，Phase 1 先數位化這個流程。

### Decision 7：供應商視圖三頁式結構

1. **任務清單**（預設頁）：所有被分派的生產任務，依狀態分組
2. **任務詳情**：點擊進入單筆任務，查看工序資訊、報價、報工
3. **報價記錄**：歷史報價清單（含狀態：已報價 / 已確認 / 已退回）

### Decision 8：生管報價審核整合至日程面板

生管審核報價的操作在日程面板內完成，不另建獨立介面：
- 外包廠生產任務卡片上顯示報價狀態 badge
- 點擊可展開報價詳情（單價、報價時間、供應商名稱）
- 提供「確認」「退回」按鈕

**理由**：生管已在日程面板管理外包廠任務，報價審核是同一工作流的延伸。

## 成功指標

| 指標 | 說明 | 目標 |
|------|------|------|
| 供應商平台使用率 | 透過平台報工的外包任務比例 | 上線 2 個月後 >= 40% |
| 報價數位化率 | 透過平台報價的外包任務比例 | 上線 2 個月後 >= 50% |
| 報價審核時間 | 供應商報價到生管確認的平均時間 | < 24 小時 |

## Risks / Trade-offs

**[報價與生產獨立的審計風險]** → 生產完成但報價未確認可能導致結算糾紛。Phase 2 可設定提醒機制。

**[多操作員的權責問題]** → 同一廠商多人操作時，報工可追溯到個人，但報價由誰提交可能造成內部責任不清。Phase 2 可考慮報價需主管確認。
