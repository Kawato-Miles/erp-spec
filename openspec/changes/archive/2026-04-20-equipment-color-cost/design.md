## Context

目前工單的生產任務沒有顏色相關欄位，印務在規劃階段無法預估設備成本。印刷業的設備成本高度依賴印刷色數（單黑 vs CMYK vs 特殊色），不同顏色組合會倍數放大設備使用費。此外，ProductionTaskWorkRecord 的 A/B/C 群組（紙材/印刷/加工）是早期設計殘留，與現行 Process.category（材料/工序/裝訂）BOM 體系不一致，需同步清除。

設備主檔已有獨立模組，包含單黑/CMYK 單價及特殊色倍率預設值，本次不需修改設備主檔。

## Goals / Non-Goals

**Goals:**
- 讓印務在規劃生產任務時，能選擇顏色選項並即時看到設備預計成本
- 讓印務在工單層級看到所有生產任務的設備成本彙總
- 清除不一致的 A/B/C 群組欄位，避免後續設計混亂

**Non-Goals:**
- 不修改設備主檔（已有獨立模組）
- 不設計報工層的新細節欄位（留待後續設計報工功能）
- 不做工序 category 防呆（所有工序 category 皆可選顏色）
- 不在工單層儲存成本彙總欄位（純 UI 計算）

## Decisions

### 1. 顏色選項放在 ProductionTask 規劃層，非報工層

**選擇**：新增欄位至 ProductionTask Data Model

**理由**：顏色選項影響的是「事前成本預估」，與報工層的「事後成本記錄」是不同階段。放在規劃層讓印務排程時就能評估成本。

### 2. 倍率類選項用 nullable 小數，非布林 + 獨立欄位

**選擇**：color_pantone_multiplier / color_metallic_multiplier / color_metallic_min_multiplier 為 `小數 | null`

**理由**：null 代表未選、有值代表已選且含倍率，一個欄位同時表達「是否啟用」和「倍率值」，減少欄位數量。

**替代方案**：布林 + 獨立倍率欄位（如 has_pantone + pantone_multiplier），欄位數翻倍但語意更明確。考量到目前只有 3 個倍率類選項，nullable 小數的簡潔性優先。

### 3. 基礎價雙選時取加總

**選擇**：單黑 + CMYK 同時選時，base_price_per_ream = 單黑單價 + CMYK 單價

**理由**：單黑和 CMYK 本身語意為「疊加」（兩者都印），倍率類選項的額外成本基於所有已選基礎色的加總放大。若取較高者或讓使用者指定，會增加操作複雜度。

### 4. 工單層成本彙總為 UI 計算，不儲存欄位

**選擇**：工單詳情頁的成本彙總 = SUM(生產任務.estimated_equipment_cost)，不在 WorkOrder 新增欄位

**理由**：彙總是即時查詢可得的結果。若在 WorkOrder 存合計，每次任一生產任務的成本變動都需更新工單，增加一致性風險。

### 5. 數量換算固定 1 令 = 500 張

**選擇**：令數 = pt_target_qty / 500

**理由**：設備單價以「令」計價，pt_target_qty 以「張」記錄。1 令 = 500 張為印刷業標準換算。

## Risks / Trade-offs

| 風險 | 緩解 |
|------|------|
| 設備無單價資料時成本無法計算 | UI 顯示「待排程」，不計入合計 |
| 材料/裝訂工序選顏色無實際意義 | 先不做防呆（MVP 決策），後續可加工序 category 限制 |
| 令數非整數時的成本精度 | 使用小數計算，不四捨五入令數 |
| A/B/C 群組移除為 BREAKING | 目前無實作依賴這些欄位（Prototype 未使用），風險低 |
