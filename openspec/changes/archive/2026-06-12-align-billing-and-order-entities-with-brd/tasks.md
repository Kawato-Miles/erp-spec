## 1. 型別宣告與 Data Model 變更

- [x] 1.1 修改 `src/types/order.ts`：更新 `Order` 介面，新增 `複製來源`、`主訂單`、`是否為補收款`、`可獲得紅利點數` 欄位；修訂訂單層級的 `付款狀態` 型別，新增 `'已退款'` 值，並移除 `@deprecated` 記。
- [x] 1.2 修改 `src/types/payment.ts`：
    *   新增 `款項類型` 欄位型別（`'收款' | '退款'`）。
    *   更名 `paymentMethod`（收款方式）為 `paymentMethod`（款項方式），修改列舉範圍為 `現金 / 信用卡 / 銀行轉帳 / 支票 / 其他`（刪除舊 `'退款'`）。
    *   更名 `paymentStatus`（收款狀態）為 `paymentStatus`（款項狀態），修改列舉範圍為 `處理中 / 已完成 / 已取消`，並刪除 `是否已取消`、`取消時間`、`取消原因` 等舊有技術與邏輯刪除欄位。
- [x] 1.3 修改 `src/types/billingInstallment.ts`：新增 `預計金額（未稅）`、`已收金額` 與 `已退金額` 三個彙整欄位。將開票狀態與收款狀態之底層型別或推導參數修正。
- [x] 1.4 修改 `src/types/paymentAllocation.ts`：刪除 `是否自動分配` 與 `是否手動覆寫` 等技術欄位。
- [x] 1.5 修改 `src/types/salesAllowance.ts` 與 `src/types/invoice.ts`：清理英文 API 欄位對照，確保折讓單保留 `折讓單號`（`allowanceNumber`）且折讓金額以正值表達。

## 2. 業務邏輯與狀態推導重構

- [x] 2.1 修改 `src/types/order.ts` 內付款狀態推導邏輯：實作「請款期次齊套推導」算法，遍歷未取消期次之收款狀態，依據齊套條件（取最落後者）決定訂單付款狀態（未付款 / 部分付款 / 已付款 / 已退款）。
- [x] 2.2 修改 `src/types/billingInstallment.ts` 內期次收款狀態推導邏輯：以 `已收金額 - 已退金額` 作為實收淨額，與期次的 `預計金額` 比較，推導出 `'未收' | '部分收款' | '已收訖'`。
- [x] 2.3 修改 `src/types/paymentAllocation.ts` 內手動核銷分配邏輯：
    *   移除自動依序填滿之分配計算。
    *   在新增/更新款項分配時，系統自動累加並更新請款期次的 `已收金額` 與 `已退金額`。
- [x] 2.4 修改 `src/utils/reconciliationCsv.ts`：修正 CSV 導出邏輯中，將 Payment 金額正值化之對帳金額累算（依據款項類型為收款時累加、退款時累減），並修訂欄位名稱為款項方式、款項狀態。
