# 供應商平台（Supplier Portal）

## Problem

外包廠與中國廠商目前透過 Slack / LINE / 電話接收生產任務、回報進度與報價，資訊散落在多個管道。生管需要手動彙整供應商回覆，報價比對困難，生產進度追蹤仰賴人工催問。外包廠無法自行查看被分派的任務全貌，也無法在統一平台上完成報價與報工。

## Approach

建立供應商平台，讓外包廠商與中國廠商以獨立帳號登入，查看被分派的生產任務，並在平台上完成報價與報工。

Phase 1 功能範圍：
- 任務清單：查看被分派的生產任務（含工序、數量、交期要求）
- 報工：回報完成數量與預計完工日期
- 報價：針對分派的生產任務提交單價報價

Phase 2（後續）：
- 歷史訂單查詢與對帳
- 報價模板（常用工序的快速報價）
- 出貨物流追蹤整合

## Capabilities

- supplier-portal: 供應商平台 — 獨立入口，外包廠商與中國廠商查看任務、報價、報工

## Affected Specs

- production-task: 新增供應商報價欄位（unit_price_quoted / quoted_at），新增供應商自助報工 Requirement
- user-roles: 更新外包廠商與中國廠商角色權責（新增「查看分派任務」「報價」「報工」權限）
- state-machines: 可能需要在外包廠路徑新增「已報價」或「待確認報價」中間態（待設計決策）
- business-processes: 新增報價審核流程（供應商報價 → 生管確認 → 核准 / 退回重報）

## Design Constraints

- 平台限制：僅支援桌機瀏覽器（與 ERP 系統統一）
- 認證：使用 ERP 統一帳號，由 Supervisor/生管 建立並設定平台權限
- 平台分離：外包廠商使用「供應商平台」，中國廠商使用「中國供應商平台」，兩個獨立入口，功能內容相同
- 資料範圍：供應商僅能看到分派給該廠商的生產任務
- 報價隔離：不同供應商的報價互不可見（價格敏感資訊）
- 多語言：中國廠商可能需要簡體中文介面（Phase 2 考慮）

## Dependencies

- wo-dispatch-ux change 的生產任務分派機制為前提
- 生產任務 spec 的外包廠路徑狀態機

## Open Questions

- 報價流程：是先分派再報價，還是先詢價再分派？
- 報價審核：由生管單獨確認，還是需要印務主管核准？
- 是否需要比價功能（同一任務詢問多家廠商報價後選擇）？
- 供應商帳號的權限粒度：一個廠商帳號看所有任務，還是可分操作員各看各的？
