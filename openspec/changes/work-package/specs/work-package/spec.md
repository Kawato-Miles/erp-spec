## Purpose

工作包（Work Package）-- 生產任務的派工分組實體。一個工作包包含多筆同工序的生產任務，指派給一位師傅負責。工作包無狀態，僅作為分組容器，提供可搜尋的 Key 與可列印的工作單。

**問題**：
- 批次派工後各生產任務散開追蹤，無法回溯「同批派出的任務」
- 缺乏可列印的師傅工作清單，現場仍靠口頭或紙條

**目標**：
- 建立持久化的派工分組，以工作包為單位管理、搜尋、列印
- 取代現有批次派工，成為生管派工的標準操作

---

## Requirements

### Requirement: 工作包 Data Model

系統 SHALL 維護 WorkPackage 資料表，欄位定義如下：

| 欄位 | 英文名稱 | 型別 | 必填 | 說明 |
|------|---------|------|------|------|
| 工作包 ID | id | UUID | Y | PK |
| 工作包編號 | key | 字串 | Y | 唯一，系統自動產生，格式：WP-{工序}-{MMDD}-{流水號三碼} |
| 指派師傅 | operator_id | FK | Y | FK -> 使用者（師傅） |
| 備註 | notes | 文字 | | 生管填寫的工作包備註 |
| 確樣需求 | sample_notes | 文字 | | 生管填寫，說明此工作包需特別注意的確樣事項 |
| 建立時間 | created_at | datetime | Y | 系統自動填入 |
| 建立者 | created_by | FK | Y | FK -> 使用者（生管） |

WorkPackage MUST NOT 包含狀態欄位。工作包的進度由包內生產任務狀態動態推導。

#### Scenario: 工作包 Key 自動產生
- **WHEN** 生管建立一個工序為「印刷」的工作包，日期為 2026-04-02，當日該工序已有 2 個工作包
- **THEN** 系統產生 Key = `WP-印刷-0402-003`

#### Scenario: 工作包 Key 唯一性
- **WHEN** 系統產生工作包 Key
- **THEN** Key MUST 在全域唯一，不得與已存在的工作包 Key 重複

### Requirement: 建立工作包

生管在任務分派介面勾選同工序的多筆「待處理」生產任務後，系統 SHALL 提供「建立工作包」操作，取代原有的「派工」操作。

建立工作包對話框 MUST 包含：
- 師傅選擇（必填）：下拉選單
- 備註（選填）：文字輸入
- 確樣需求（選填）：文字輸入（工作包層級，非逐筆任務）
- 任務清單：顯示已勾選的生產任務明細（工單號、印件名稱、目標數量）

確認建立後，系統 SHALL 一次完成以下動作：
1. 建立 WorkPackage 記錄（Key 自動產生，含 notes、sample_notes）
2. 各 ProductionTask 設定 work_package_id = 新工作包 ID
3. 各 ProductionTask 設定 assigned_operator = 選定師傅
4. 各 ProductionTask 設定 actual_start_date = 當天日期

生產任務狀態 MUST NOT 因建立工作包而變更（維持「待處理」，待師傅首次報工時才進入「製作中」）。

#### Scenario: 建立工作包完整流程
- **WHEN** 生管在「印刷」工序卡片勾選 PT-001、PT-002、PT-005，點擊「建立工作包」
- **THEN** 系統彈出建立工作包對話框，顯示 3 筆任務明細
- **WHEN** 生管選擇師傅「王大明」，填寫備註「急件優先」，填寫確樣需求「需對色」，確認建立
- **THEN** 系統建立工作包 WP-印刷-0402-001，3 筆任務歸入該工作包，assigned_operator 設為王大明
- **THEN** 3 筆任務狀態維持「待處理」不變

#### Scenario: 未選擇師傅時無法建立
- **WHEN** 生管未選擇師傅即點擊確認
- **THEN** 系統顯示驗證提示「請選擇師傅」

#### Scenario: 僅待處理且未歸屬工作包的任務可勾選
- **WHEN** 生產任務狀態非「待處理」，或已屬於某工作包
- **THEN** 該任務的勾選框 MUST 為禁用狀態

### Requirement: 工作包搜尋

系統 SHALL 在進行中區與生產任務列表提供工作包搜尋欄位。搜尋 SHALL 支援以下條件：
- 工作包 Key（模糊比對）
- 師傅名稱（模糊比對）

#### Scenario: 以 Key 搜尋工作包
- **WHEN** 生管在搜尋欄輸入「0402」
- **THEN** 系統篩選所有 Key 包含「0402」的工作包

#### Scenario: 以師傅名稱搜尋
- **WHEN** 生管在搜尋欄輸入「王大明」
- **THEN** 系統篩選所有 operator 為「王大明」的工作包

### Requirement: 移出任務

生管 SHALL 可將單筆生產任務從工作包中移出。移出後：
- ProductionTask.work_package_id 設為 null
- ProductionTask.assigned_operator 設為 null
- ProductionTask.actual_start_date 設為 null
- 任務退回待分派區

已進入「製作中」狀態（已有報工紀錄）的生產任務 MUST NOT 允許移出。

#### Scenario: 移出待處理任務
- **WHEN** 生管在工作包 WP-印刷-0402-001 中對「待處理」的 PT-002 執行移出
- **THEN** PT-002 退回待分派區，工作包內剩餘 2 筆任務

#### Scenario: 無法移出製作中任務
- **WHEN** 生管嘗試移出「製作中」的 PT-001
- **THEN** 系統顯示提示「製作中的任務無法移出，請先完成或作廢」

#### Scenario: 移出最後一筆任務
- **WHEN** 工作包僅剩 1 筆任務，生管執行移出
- **THEN** 任務移出後，工作包成為空包，系統提示生管是否刪除空工作包

### Requirement: 整包移轉師傅

生管 SHALL 可將工作包整包移轉給另一位師傅。移轉後：
- WorkPackage.operator_id 更新為新師傅
- 包內所有 ProductionTask.assigned_operator 更新為新師傅

#### Scenario: 移轉工作包
- **WHEN** 生管將 WP-印刷-0402-001（原師傅王大明）移轉給李師傅
- **THEN** 工作包與包內所有任務的 assigned_operator 更新為李師傅

### Requirement: 刪除工作包

生管 SHALL 可刪除工作包。刪除前系統 MUST 檢查包內是否有「製作中」的生產任務：
- 若有「製作中」任務：阻擋刪除，顯示提示說明哪些任務阻擋刪除
- 若無「製作中」任務：刪除 WorkPackage 記錄，包內所有任務按移出邏輯處理（退回待分派）

#### Scenario: 刪除無製作中任務的工作包
- **WHEN** 生管刪除 WP-印刷-0402-002，包內 2 筆任務均為「待處理」
- **THEN** 系統刪除工作包，2 筆任務退回待分派區

#### Scenario: 無法刪除含製作中任務的工作包
- **WHEN** 生管嘗試刪除包含 1 筆「製作中」任務的工作包
- **THEN** 系統阻擋刪除，顯示「工作包內有 1 筆製作中的任務，請先完成或作廢後再刪除」

### Requirement: 工作單列印

系統 SHALL 提供以工作包為單位的工作單列印功能。入口位置為工作包卡片上的「列印工作單」操作按鈕。

工作單列印的詳細欄位與版面設計不在本次 change 範圍，後續獨立 change 定義。

#### Scenario: 列印入口
- **WHEN** 生管在某工作包卡片點擊「列印工作單」
- **THEN** 系統開啟列印預覽（或觸發瀏覽器列印），以該工作包為單位輸出內容
