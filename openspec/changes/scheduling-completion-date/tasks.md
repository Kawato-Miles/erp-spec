## 1. 生產任務 Data Model 與排程欄位

- [ ] 1.1 ProductionTask Data Model 新增 estimated_duration_days（整數，自有工廠必填）與 planned_end_date（日期，自有工廠唯讀/外包廠可編輯）欄位，更新 main spec
- [ ] 1.2 驗證欄位定義與現有 scheduled_date、planned_equipment 等排程欄位的一致性

## 1.5 工單詳情頁 — 新增生產任務抽屜 UI

- [ ] 1.5.1 「新增生產任務」按鈕開啟全屏右抽屜（Drawer），抽屜寬度 90vw、高度 100%，支援橫向滾動
- [ ] 1.5.2 抽屜頂部顯示工單上下文資訊（工單號、客戶名稱、交貨日期、工單數量），幫助印務邊填邊參考
- [ ] 1.5.3 抽屜內為生產任務表格，14 欄：任務名稱、製程、數量、單位、尺寸、工廠、設備、工期(天)、完工日、影響成品、備註、操作（刪除）
- [ ] 1.5.4 「新增列」按鈕可單列新增，「批量快速新增」可預設常見組合（如「印刷 → 裁切 → 折疊 → 裁邊」），一鍵展開多列
- [ ] 1.5.5 表格行內編輯，用戶編輯後無需額外確認；工廠欄位應預填（從工單設定或前一列帶入），印務可按需修改
- [ ] 1.5.6 抽屜底部「儲存」和「取消」按鈕，儲存後返回工單詳情頁，新任務出現在生產任務列表中
- [ ] 1.5.7 任務創建時 estimated_duration_days 和 planned_end_date 為空（暫不自動計算），待派工排程階段填入
- [ ] 1.5.8 取消前若有未保存的更改，提示確認

## 1.6 新增生產任務抽屜 — 右側試算面板

- [ ] 1.6.1 抽屜右側（約 40% 寬）可收合試算面板，含「拼版試算」與「工單成本試算」兩區塊，預設收合
- [ ] 1.6.2 拼版試算：母版尺寸下拉（菊全/菊對/菊四/四六全/四六對等固定清單，選後自動帶入 W×H mm）
- [ ] 1.6.3 拼版試算：成品寬度（mm）、成品高度（mm）、出血（mm）、咬口（mm）輸入欄位
- [ ] 1.6.4 拼版試算：即時計算結果 — 拼法（N 模，列×行）、需要印張數、紙張利用率（%），橫拼 vs 直拼取最大模數
- [ ] 1.6.5 工單成本試算：即時從左側表格計算各列 `數量 × 單價`，按製程分類彙總，顯示合計金額
- [ ] 1.6.6 左側表格數量或內容變更時，成本試算即時更新
- [ ] 1.6.7 試算面板為獨立工具，結果不自動寫入表格欄位

## 2. 派工板排程 UI 調整

- [ ] 2.1 自有工廠排程表單新增「預估工期（天）」輸入欄位（整數），填寫後即時推算並顯示 planned_end_date（唯讀，跳過週日）
- [ ] 2.2 外包廠排程表單新增「預計完工日」日期選擇器，含基本校驗（>= 今天；超過交期 + 30 天時提示確認）

## 3. 生產任務交期預警（兩級）

- [ ] 3.1 紅色預警：planned_end_date > delivery_date 時以紅色標記並顯示「超期 N 天」
- [ ] 3.2 黃色預警：planned_end_date 在 delivery_date 前 2 天以內（含等於當天）以黃色標記並顯示「距交期 N 天」
- [ ] 3.3 排程調整後預警即時更新

## 4. 任務交付前置條件補強

- [ ] 4.1 任務交付前置條件新增檢查：所有生產任務 MUST 已設定 planned_end_date
- [ ] 4.2 自有工廠生產任務交付前置條件新增檢查：MUST 已填寫 estimated_duration_days
- [ ] 4.3 阻擋交付時提示缺少排程資訊的任務清單

## 5. 工單預估完成日與預警

- [ ] 5.1 工單詳情頁新增「預估完成日」計算欄位顯示：max(所有生產任務的 planned_end_date)
- [ ] 5.2 工單詳情頁新增「N/M 已排程」標記，顯示排程完成度
- [ ] 5.3 工單交期預警（兩級）：紅色（超期）+ 黃色（即將超期，距交期 <= 2 天）
- [ ] 5.4 驗證情境：所有任務已排程 / 部分已排程 / 全部未排程三種情境的顯示正確性

## 6. 印件預估完成日與預警

- [ ] 6.4 印件總覽新增「N/M 工單已排程」標記

## 7. 訂單印件清單擴充（業務端）

- [ ] 7.2 業務端顯示規則：所有大貨工單通過製程審核且所有生產任務都已設定 planned_end_date → 顯示日期；否則 → 顯示「排程中」
- [ ] 7.3 預估完成日超過交貨日期時以紅色標示，即將超期時以黃色標示

## 8. 端到端驗證

- [ ] 8.1 驗證情境：自有工廠排程 → 工期填寫 → 完工日推算（含跳過週日）→ 工單彙總 → 印件彙總 → 交期預警完整鏈路
- [ ] 8.2 驗證情境：外包廠排程 → 完工日手動填寫（含校驗）→ 工單彙總 → 印件彙總 → 交期預警完整鏈路
- [ ] 8.3 驗證情境：混合工廠（自有 + 外包）的工單預估完成日取 max 正確性
- [ ] 8.4 驗證情境：部分未排程時印務端「N/M 已排程」標記與業務端「排程中」的正確顯示
- [ ] 8.5 驗證情境：打樣工單不影響印件預估完成日
- [ ] 8.6 驗證情境：黃色預警（即將超期）與紅色預警（已超期）的閾值正確性
