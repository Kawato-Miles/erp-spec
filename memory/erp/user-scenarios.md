# ERP 使用者情境（User Scenarios）

> 描述各角色在真實工作情境下的操作需求。
> 所有功能設計與 spec 需能對應到至少一個使用者情境。
>
> 最後更新：2026-02-21
> 狀態：業務、審稿、印務、印務主管、生管初版完成；QC / 出貨待補

---

## 業務

### 情境 1：建立需求單（售前評估）

**角色**：業務 / 諮詢業務
**觸發**：客戶詢問報價
**流程**：
1. 登打客戶基本資料
2. 填寫購買的印件（品項、規格、數量）
3. 與**製程審核人**確認預計成本
4. 確認後可提供報價給客戶
5. 客戶確認後，將需求單**轉為訂單**

**成功條件**：需求單成功轉為訂單，訂單進入打單階段
**待確認**：需求單轉訂單時，資料是否全部帶入，還是需要業務再補填？

---

### 情境 2：訂單款項與發票管理

**角色**：業務
**觸發**：訂單成立後，依付款條件收款
**流程**：
1. 依照訂單付款條件（訂金、中期款、尾款）登記收款紀錄
2. 收款後開立發票（或由會計處理）
3. 追蹤尚未收款的款項

**成功條件**：所有款項收訖，發票開立完成
**連動角色**：會計負責金流查帳與發票開立

---

### 情境 3：確認打樣結果（打樣決策點）

**角色**：業務
**觸發**：印務完成打樣工單，打樣成品交由業務確認
**流程**：
1. 業務取得打樣成品，與客戶確認
2. **決策點 A：打樣 OK**
   - 業務通知印務可開始印製大貨
   - 進入大貨印製流程
3. **決策點 B：打樣不 OK — 需調整製程**
   - 由印務調整製程設定（不需重新審稿）
   - 重新進行打樣
4. **決策點 C：打樣不 OK — 稿件有問題**
   - 產生新稿件，重新走一次完整審稿流程
   - 審稿通過後，再進行打樣

**待確認**：
- 決策 B（調整製程）與決策 C（重新審稿）的觸發條件是否由業務判斷，或由審稿 / 印務判斷？
- 打樣不 OK 時，現有打樣工單的狀態如何更新？是關閉並新建，還是原工單重新執行？

## 審稿

### 情境 1：審核稿件

**角色**：審稿人員
**觸發**：訂單中的印件進入審稿階段（稿件上傳後）
**流程**：
1. 依照訂單中的印件，逐一審核稿件是否合規（出血、解析度、色彩模式等）
2. **稿件合格**：標記合格，系統自動進入製程審核 → 工單建立流程
3. **稿件不合格**：退稿，通知業務 / 客戶修改

**成功條件**：稿件標記合格，進入下一階段

---

### 情境 2：打樣後需重新處理稿件

**角色**：審稿人員
**觸發**：打樣結果不 OK 且問題來自稿件（非製程）
**流程**：
1. 客戶 / 業務提供修改後的新稿件
2. **新稿件視為全新稿件**，重新跑一次完整的審稿流程（非修改原稿）
3. 審稿通過後，再進行打樣

**成功條件**：新稿件審核通過，可重新安排打樣
**注意**：每次重新審稿都是新的稿件紀錄，保留歷史版本

## 印務

### 情境 1：建立工單內容

**角色**：印務
**觸發**：印務主管將工單草稿分派給印務後
**前置條件**：印務主管已建立工單草稿並完成分派
**流程**：
1. 接收分派的工單草稿
2. 填寫工單詳細內容（製程、材料、生產任務規格）
3. 設定生產任務的數量與「是否影響成品」旗標
4. 工單建立完成後，生管接手分派生產任務給工廠

**成功條件**：工單內容填寫完畢，生管可接手執行
**注意**：一個訂單可由**多位印務**分開負責；一張工單對應**一位負責印務**

---

### 情境 2：追蹤與管理生產任務

**角色**：印務
**觸發**：工單進入生產中
**流程**：
1. 追蹤各生產任務的進度（透過生管 / 師傅的報工紀錄）
2. 若有異動需求（訂單需求變更、任務變更）：由印務提出，印務主管執行異動
3. 確認 QC 結果，紀錄 QC 通過數量

**成功條件**：所有生產任務完成，QC 通過數量達到目標

---

### 情境 3：打樣工單 vs 大貨工單

**角色**：印務
**說明**：工單分為兩種類型，流程不同

| 類型 | 說明 | 後續流程 |
|------|------|----------|
| **打樣工單** | 先行生產少量樣品供業務 / 客戶確認 | 打樣成品交業務確認 → 打樣 OK 後才建立大貨工單 |
| **大貨工單** | 正式大量生產 | 直接進入生產 → QC → 出貨 |

**標準流程**：印件通常先跑打樣工單，確認 OK 後再建大貨工單
**待確認**：打樣工單與大貨工單是分別建立，還是在同一工單中用狀態區分？

## 印務主管

### 情境 1：分配工單給印務

**角色**：印務主管
**觸發**：訂單審稿通過，進入工單建立階段
**流程**：
1. 建立工單草稿（依印件規格與製程自動帶入，或手動建立）
2. 評估各印務目前負載，決定分配
3. 將工單草稿分派給負責的印務
4. 追蹤工單執行狀況，必要時進行工單異動

**成功條件**：所有工單均已分派，各印務接手填寫工單內容
**注意**：一個訂單內的不同工單可分派給不同印務

---

## 生管（生產廠長）

### 情境 1：接收任務並分派生產任務

**角色**：生管
**觸發**：印務完成工單內容填寫後
**流程**：
1. 接收印務建立完成的工單任務清單
2. 評估各工廠 / 師傅的產能與排程
3. 分派生產任務給對應的工廠或師傅（自有廠 / 外包廠）
4. 追蹤各生產任務進度（透過師傅每日報工紀錄）
5. 若有異動，回報印務處理

**成功條件**：所有生產任務分派完畢，工廠開始執行
**注意**：生管以紙本工單為主要作業依據；系統輔助追蹤
