# 打樣流程

> 核心：打樣工單與大貨工單透過 `type` 欄位區分。打樣結果決定後續分支。

## 一、打樣與大貨的關係

- 若需打樣，**打樣印件與大貨印件 SHALL 同時建立**
- 打樣 / 大貨各為獨立的印件，但屬於同一張訂單

## 二、打樣決策結果處理

| 結果 | 處理 |
|------|------|
| **OK**（通過）| 開放對應大貨印件的稿件上傳功能 |
| **NG（製程問題）** | 同印件下建立新工單重新打樣 |
| **NG（稿件問題）** | 引導建立新打樣印件 |

## 三、典型情境

### 情境 1：打樣通過後開放大貨

- 打樣結果 = OK
- 系統開放對應大貨印件的稿件上傳

### 情境 2：打樣失敗（製程問題）

- 打樣結果 = NG，原因 = 製程問題
- 系統允許在同一印件下建立新工單重新打樣（同一稿件，不同工單）

### 情境 3：打樣失敗（稿件問題）

- 打樣結果 = NG，原因 = 稿件問題
- 系統引導建立新打樣印件（重新審稿）

## 四、與付款發票的關係

打樣相關的補收 / 退款情境見 [[付款發票邏輯]]：

- **打樣酌收**：原規格送打樣，但打樣後不製作大貨 → OrderAdjustment（打樣費用補收）
- **第二次打樣是因第一次出問題**：走原工單不補收（公司認賠）
