## 一、本週學到什麼

### 1. 售後 ticket 模組陷入 reactive 補丁循環 — 需 propose 階段加 PM 推演

- 來源：[[../../12-insights/2026-05-20-售後ticket-reactive-補丁循環]] insight + 本週 archive 售後 4 change（add-after-sales-ticket / refactor-my-after-sales / refine-after-sales-refund / refine-supplementary-print-skip-review）
- 提煉：連續 5 個 change 都因「Miles 試用 → 發現新 gap → 補丁」開單，根因是 propose 階段缺端到端 user journey 整合驗證
- 應用：未來新模組 propose 階段 MUST 含 PM 推演（已寫入 [[../../11-review-knowledge/_shared/review-loading-checklist]] § 四「端到端推演準則」）

### 2. Karpathy LLM Wiki + Yu Anti-Model-Collapse 可共存 — 不必為借鑑就重組 vault

- 來源：本週 vault-ingest skill 建置（[[../../raw/README]] + [[../../00-meta/vault-charter]] 2026-05-21 升級紀錄）
- 提煉：原本可能誤判「採 Karpathy 模式 = 重組目錄為 wiki/」；實際上 Sens 既有 Zettelkasten 原子卡已繞開「page 邊界決策成本」（Yu 文章核心警告），只要把 Karpathy 三件事（ingest 矛盾偵測 / 跨卡連鎖更新 / lint）加進來，搭配 Yu 四道防線（claude-self-capture 須確認 / claude-research 須真實來源 / Mode B cards diff 須批准 / vault-insight 不讀 raw）即可
- 應用：未來「借鑑外部模式」要先判斷「形式上的相同」vs「實質的學習」，本週方向先校準（讀 Yu 文章後重定位）省了一次大規模重組

### 3. Skill 設計初版必有盲點 — 主動跑 + 收反饋 + 改規約勝過閉門設計

- 來源：本週 daily-brief / weekly-review skill 連續演化（從 5 條規則 → 10/11 條，三次 skill-revision 紀錄於 [[../../00-meta/audit-log]] 2026-05-21 條目）
- 提煉：初版 skill 寫了完整流程 + Anti-Pattern 防護，但實際跑一次後 Miles 連續給 4 次反饋（產出位置/預估時間 → 排序邏輯 → 結構 → 阻擋/被阻擋措辭），每次都修一道規約；最快收斂方式是先做 minimal version 觸發實用反饋
- 應用：後續新 skill（如未來的 monthly-review / vault-audit 維度 11/12 擴充）採「先跑 → 收反饋 → 加規約」三步迭代，不一次設計到完美

### 4. 跨層次知識的「穩定性分離」設計範式 — User Story 兩階段紀律

- 來源：本週 13-user-stories pilot（commit 1b7d568 + 7abe9a1）+ erp-user-story skill 設計
- 提煉：user story 拆「業務情境段（穩定）」+「UI 操作段（易變）」兩階段，Prototype 改版時只動 UI 段，business 段保持不變。同樣分離邏輯也用在：business-logic 卡 vs UI DESIGN.md、entity 卡 vs spec § Scenarios
- 應用：未來新建跨層次內容（如 KPI 卡 vs metric 計算公式、scenario 卡 vs 介面操作）都套此範式

### 5. PRC 用語滲透不易自察 — CLAUDE.md § 用語範例庫是必要結構

- 來源：本週「桌面走查 / 端到端走查」用語反饋（commit 531aa41）+ CLAUDE.md § 8 加 2 條變換範例
- 提煉：「走查」是 walkthrough 直譯（PRC 用語），但既有 4 個 vault 卡 + 多個 skill description 都已採用，Claude 自審時沒抓出來。原因：CLAUDE.md § 8「避免大陸技術用語」只列了 2-3 例（暴露/全綠），未涵蓋本案
- 應用：CLAUDE.md § 8 範例變換表要持續累積（已加「走查→推演/試用」+「狀態快照→狀態說明」），每次發現新案就補入
