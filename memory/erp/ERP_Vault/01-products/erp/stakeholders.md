# 利害關係人

> [!note] Phase A 暫以「角色 = 利害關係人」處理
> Notion 產品目標頁有內嵌「利害關係人 DB」（`https://www.notion.so/32e3886511fa8074bceec44ab608db20`），但與「核心角色權責表」DB 內容高度重疊。Phase A MVP 階段先以 [[03-roles/業務|核心角色權責表]] 的 16 個角色作為主要利害關係人清單。
>
> Phase B Stage 5 補完整利害關係人 DB 內容後，再決定：
> - 兩 DB 合併
> - 或分工：利害關係人 DB 強調「利害關係程度 / 預期阻力 / 關切點」（已在角色 DB 中）

## 預期阻力概要

依角色 DB 整理，主要阻力：

| 阻力來源 | 角色 | 緩解策略 |
|---------|------|----------|
| 操作習慣改變（紙本 → 系統） | [[03-roles/師傅|師傅]]、[[03-roles/生管|生管]] | 管理層政策支持、UI 簡化、漸進導入 |
| 補單與規格異動增加輸入負擔 | [[03-roles/業務|業務]]、[[03-roles/諮詢|諮詢]] | 簡化補單流程、明確「漏收追加費用」責任歸屬 |
| 系統未支援時的過渡方案 | [[03-roles/中國廠商|中國廠商]]、[[03-roles/外包廠商|外包廠商]] | 紙本傳真作為過渡 |
| 派工合批邏輯不符合實際 | [[03-roles/生管|生管]]、[[03-roles/訂單管理人|訂單管理人]] | OQ XM-003 確認後再實作 |
| 產能可見度未達預期 | [[03-roles/Supervisor|Supervisor]] | Phase 切換時間點明確 |
