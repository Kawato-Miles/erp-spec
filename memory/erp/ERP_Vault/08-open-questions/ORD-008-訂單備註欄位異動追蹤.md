# ORD-008：訂單備註欄位是否需要異動追蹤（last_updated_by / audit trail）

## 背景

新增的三個備註欄位可能會被多人多次編輯（業務 + 諮詢 + 業務主管），無記錄會難追溯。但完整 audit trail 增加實作成本。

## 問題

異動追蹤要做到哪一層？

候選做法：

1. **不做追蹤**：只存最新值，沿用 Prototype 簡化原則
2. **只記 metadata**：每個欄位加 `lastUpdatedBy` / `lastUpdatedAt`
3. **完整 audit trail**：另開 OrderNoteHistory 實體，記錄每次變更前後值與作者
4. **跟著 OrderAdjustment 走**：備註異動視為訂單異動，走既有 OA 流程

## 影響範圍

- Data Model：是否新增 metadata 欄位
- UI：是否顯示「最後修改者 / 時間」
- spec：order-management § Data Model + Audit Requirements

## 待釐清

- 印刷業實務上備註是否需要法務 / 對帳級的追蹤？
- 客訴情境中「業務最後改了什麼」會被追問嗎？
- 跟既有 staffNotes / authNotes 的追蹤規範一致？
