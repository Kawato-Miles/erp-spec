# BI-16：差額警示文案區分「待開票」與「退款待執行」兩種善後缺口語意

## 問題描述

對帳主軸改「已開立發票」後（[[BI-15-對帳主軸改已開發票後未開票應收呈現|BI-15]] 採方案 A），未開票應收與退款未執行皆靠差額警示兜底。但兩種善後缺口的語意方向相反：

- **應收 > 發票淨額**：尚有收入未開立發票 → 業務待開票
- **收款淨額 > 應收**：已收金額超過應收（如諮詢取消半額退費，退款 OA 已核可但退款 Payment 尚未切已完成）→ 退款待執行

若兩者共用同一句差額警示文案（如僅顯示「應收與發票淨額不符 N 元」），會計／業務無法從文案判斷應「去開票」還是「去執行退款」，易誤讀缺口性質、做錯善後動作。

## 涉及範圍

- 模組：order-management / billing 對帳
- 相關卡：[[../05-entities/訂單]]
- 影響範圍：對帳差額警示文案（OrderReconciliationPanel / OrderBillingInstallmentSection 警示 banner）、會計與業務善後動作正確性
- 連動：[[BI-15-對帳主軸改已開發票後未開票應收呈現|BI-15]]（對帳主軸 + 差額警示兜底範圍）、[[BI-11-三方對帳警示banner觸發條件|BI-11]]（對帳警示 banner 觸發條件與 UX 文案，補收 OA vs 期次差額面向；本 OQ 聚焦「待開票 vs 退款待執行」方向語意區分，與 BI-11 不同面向）

## 待解答

- [ ] 差額警示是否需依「應收 > 發票淨額（待開票）」vs「收款淨額 > 應收（退款待執行）」分兩種文案
- [ ] 各文案的對外措辭（會計／業務皆能正確判斷善後動作方向）
- [ ] 是否需各自的 action button 引導（待開票 → 引導開票流程；退款待執行 → 引導退款 Payment 切已完成）

## 候選方案

### prototype 階段
- 差額警示以數值方式呈現（顯示「應收 / 發票淨額 / 收款淨額」三數字 + 差額），由閱讀者自行判讀方向即可，不強制做對外文案分流。

### 上線前驗證
- 對外正式文案需驗證：依差額方向產出兩種語意明確的提示（「尚有 N 元未開立發票」vs「已收較應收多 N 元、退款待執行」），並各自引導正確善後動作。

## 說明

本 OQ 屬「上線前需驗證對外溝通方式」，非 prototype 階段強制項。prototype 階段差額數值呈現已足夠驗證對帳完整性邏輯；對外文案的語意區分待上線前與會計／業務確認措辭後落實。
