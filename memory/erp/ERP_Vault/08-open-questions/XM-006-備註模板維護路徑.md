# XM-006：備註模板維護路徑（Phase 2 議題）

## 背景

Phase 1 模板 seed data 寫死在 `src/data/orderNoteTemplates.ts`，業務無法自助新增 / 編輯 / 刪除。未來必須要有維護介面（業務反映「想加新條款」「想改文字」「想刪過時條款」）。

## 問題

模板維護介面歸屬哪個模組？由誰維護？

候選做法（歸屬模組）：

1. **「系統設定」模組**：全公司統一管理（需新增系統設定模組）
2. **「業務工具」模組**：業務主管維護（與 sales-platform 整合）
3. **「主檔管理」模組**：對齊 material-master / process-master / binding-master 模式
4. **訂單管理模組內**：在訂單詳情頁有「管理模板」入口

候選做法（維護權限）：

1. **業務主管 + Admin**
2. **全業務都能編**
3. **公司共用 + 個人模板雙層**
4. **依模組分權**：訂單備註 by 業務主管、生產備註 by 印務主管

## 影響範圍

- 系統範疇：是否要新建模組
- 資料模型：模板存 DB 還是 localStorage
- 角色權限：誰能改全公司共用模板
- 與既有 material-master / process-master 的對齊（主檔管理模式）

## 待釐清

- 公司模板使用穩定度（多久改一次？）
- 個人模板是否真的需要（業務有自己的口頭禪）
- 與其他可能模板化的內容（如發票備註、信件回覆）的整合
- 模板版本管理（過時模板要不要保留歷史）
