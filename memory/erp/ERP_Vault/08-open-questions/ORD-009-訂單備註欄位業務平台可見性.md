# ORD-009：新增的 3 個備註欄位是否顯示於業務平台訂單列表 / 我的訂單頁

## 背景

業務平台（sales-platform）有「我的訂單」「印件總覽」等視圖。新增的 orderNote / deliveryNote / paymentNote 是否要在列表 / 卡片視圖出現？

## 問題

列表 / 卡片視圖的可見性策略是什麼？

候選做法：

1. **不顯示**：只在訂單詳情頁出現，列表不擴張資訊量
2. **列表 hover tooltip**：滑鼠移到訂單列顯示前 60 字預覽
3. **列表加欄位**：表格直接加三欄（資訊密度大幅增加）
4. **重點訂單卡片摘要**：只在「我的訂單」卡片視圖摘要顯示
5. **依角色決定**：業務看「訂單備註」、出貨看「交貨備註」、會計看「付款備註」

## 影響範圍

- sales-platform spec
- 業務認知負擔（列表資訊密度）
- 跨角色資訊流（出貨 / 會計也需要看到嗎？）

## 待釐清

- 業務在列表階段最需要看的備註類別是哪個？
- 列表加欄位 vs 詳情頁查看的取捨（一次掃 10 筆 vs 點進去看）
- 與既有「我的訂單」既定設計的對齊
