# AR-11 補件停滯處理機制與輪次上限

## 問題

US-AR-009 / 010 補件流程目前無「輪次上限」「停滯處理」「客戶不回應」機制：

1. **補件次數上限**：客戶 / 業務反覆補件 ≥ 3 輪仍不合格時是否觸發業務主管警示？是否強制升級處理？
2. **客戶不回應 / 補件延誤**：退件後 N 天客戶仍未補件，系統如何提醒 / 升級？
3. **棄單處理**（B2C）：B2C 會員 N 天不上傳，是否自動取消訂單？或轉業務處理？

senior-pm 視角審查指出多輪補件循環在 B2B / B2C 場景下對應的業務 SOP 完全缺失，可能造成印件無限期卡在「不合格」狀態。

## 影響

- 影響 US-AR-009 / 010 acceptance criteria（是否補「輪次上限警示」「停滯 SLA」條件）
- 影響業務主管工作量（無自動警示則需主管定期人工檢查）
- 影響客戶體驗（B2C 多輪失敗無引導 / 棄單無提醒）
- 影響 KPI（補件成功率 / 平均輪次 / 停滯時間）

## 選項

| 議題 | 選項 | 規則 |
|------|------|------|
| **輪次上限** | A | 無上限 |
| | B | ≥ 3 輪後系統警示業務主管 |
| | C | ≥ 5 輪後強制業務主管核可才能繼續 |
| | D | 客戶承擔（B2B）/ 系統建議放棄重提稿（B2C）|
| **停滯 SLA** | A | 無 SLA |
| | B | 退件後 N 天系統提醒對應補件方（業務 / 會員）|
| | C | 退件後 N 天升級到業務主管 |
| **棄單處理（B2C）** | A | 不自動處理（無限期等待）|
| | B | N 天後系統提醒會員 |
| | C | N 天後自動取消訂單（依退款政策）|
| | D | N 天後轉業務代操作（連動 US-AR-009）|
