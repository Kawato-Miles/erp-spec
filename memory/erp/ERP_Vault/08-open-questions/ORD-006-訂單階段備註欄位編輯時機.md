# ORD-006：訂單階段新增 3 個備註欄位的編輯時機（field lock 規則）

## 背景

本 change 新增的三個欄位（orderNote / deliveryNote / paymentNote）需要決定編輯時機。

[訂單管理 spec](../../../openspec/specs/order-management/spec.md) 對既有欄位的鎖定規則：
- 「製作等待中（含）」階段內可自由編輯
- 「製作中」之後欄位多數鎖定
- paymentTermsNote 從 Quote 帶入後即鎖定

## 問題

這三個新欄位何時可編輯、何時鎖定？

候選做法：

1. **整個生命週期都可編輯**：訂單建立到完成 / 已關閉皆可改
2. **訂單成立後到完成前**：訂單完成（completed_at 有值）即鎖定
3. **訂單成立後到出貨前**：第一筆出貨完成即鎖定（用於對齊出貨溝通時點）
4. **跟既有欄位閘門一致**：用 spec 既有「製作等待中（含）」閘門，製作中 / 完成後鎖定

## 影響範圍

- spec § Field Lock Rules
- UI：textarea disabled 條件 + 對應「插入常用備註」按鈕 disabled 條件
- 售後流程：訂單完成後的售後 ticket 是否能補備註（傾向另開售後 ticket 內欄位處理）

## 待釐清

- 業務在「訂單已完成」階段是否還需要改備註（例：補記客戶口頭說明的事項）？
- 與既有「製作等待中閘門」一致還是放寬？
- 出貨後備註是否會誤導出貨人員 / 客服？

## 來源

- senior-pm 前期介入
- change `add-order-note-section-with-template-tool` proposal.md

## 解答（2026-05-28，relax-order-detail-edit-conditions change）

採做法 **「訂單未取消即可編輯」粗粒度規則**（非 1-4 原本候選清單，比候選 1「整個生命週期」更精確）：

- 訂單成立後（status >= 訂單建立）且訂單未取消（`order.status !== '已取消'`）：三個欄位皆可編輯
- 訂單已取消（`order.status === '已取消'`）：三個欄位 SHALL 鎖定為唯讀

**翻轉的設計**：v1.7 既有「訂單完成（`completed_at IS NOT NULL`）即鎖定」設計被本次廢止；訂單完成後仍允許業務 / 諮詢 / 訂單管理人補上客戶口頭說明等資訊（業務反映此需求在實務常見）。

業務 / 會計對帳的歷史一致性透過 ActivityLog 變更紀錄保留（不依賴鎖定機制）。
