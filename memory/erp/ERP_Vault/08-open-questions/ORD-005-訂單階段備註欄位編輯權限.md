# ORD-005：訂單階段新增 3 個備註欄位（orderNote / deliveryNote / paymentNote）的編輯權限

## 背景

本 change 在 Order 實體新增三個 free-text 欄位（訂單備註 / 交貨備註 / 付款備註），由業務 / 諮詢在訂單階段填寫。需確認編輯權限。

ERP 既有 5 個備註欄位的編輯規則不一致：
- `paymentTermsNote`：從 Quote 帶入並鎖定
- `staffNotes`：業務 owner 可編輯
- `productionNote`：線下單業務填、線上單 EC 帶入
- `authNotes`：業務 owner 可編輯

## 問題

新增的三個欄位由誰可以編輯？

候選做法：

1. **只有業務 owner**：訂單 sales_id 對應的業務可編
2. **業務 owner + 諮詢**：諮詢角色（共用業務權限機制）可編
3. **業務 owner + 諮詢 + 業務主管**：業務主管可代編
4. **業務 owner + 諮詢 + 業務主管 + 訂單管理人**：訂單管理人也可編

## 影響範圍

- spec § Roles & Permissions：order-management spec 需明確列出編輯角色
- UI：OrderInfoEditDialog 三個欄位的 disabled 條件
- 業務責任邊界：誰寫的備註誰負責

## 待釐清

- 諮詢角色在訂單階段是否能改業務寫的備註？
- 業務主管「代編」場景多不多（常見業務請假 / 離職時補備註）？
- 訂單管理人需不需要編輯權限（vs 只看權限）？
- 與 [[XM-003-訂單管理人 vs 業務權責邊界]] 的整合方式
