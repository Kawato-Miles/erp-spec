# ORD-010：備註模板插入工具的重複插入防呆機制（Phase 2 議題）

## 背景

NoteTemplatePopover 第一版（Phase 1）採「不偵測 / 不防重」最簡策略：使用者每次開 popover、checkbox 狀態歸零、勾選後 append、可能造成 textarea 出現同條模板兩次。

## 問題

未來是否需要做防重？做法為何？

候選做法：

1. **不做（維持 Phase 1）**：相信業務自己控管
2. **文字比對偵測**：開 popover 時掃 textarea 已含的模板 text，已含的 checkbox 預設打勾並 disabled / 灰
3. **插入記錄追蹤**：另存「該欄位已插入過的 templateId array」，開 popover 時依此 lock
4. **插入前提示**：點插入時若偵測到重複，跳 confirm「偵測到 N 條已存在，仍要插入嗎？」

## 影響範圍

- NoteTemplatePopover 元件 API（需要 prop 接受「已插入記錄」？）
- store：是否要在 Order 加 `noteTemplateInsertions: { fieldKey: templateId[] }`
- 業務修改文字後的偵測準確度（手改過的字無法 match）

## 待釐清

- 業務 UAT 後實際出現多少重複插入情況
- 文字比對 vs ID 追蹤的 trade-off
- 是否搭配「插入後不可再插同條」的硬限制
