# 訂單（Order）

> ERP 核心實體。**三種訂單類型**：一般訂單（需求單成交）/ 諮詢訂單（諮詢結尾）/ EC 線上訂單。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `order_no` | 系統 ID / 業務看見的編號 |
| `status` | 狀態（見 [[訂單狀態]]，線下 / 線上 / 諮詢路徑不同）|
| `order_type` | `一般` / `諮詢` / `EC` |
| `customer_id` | 客戶 |
| `sales_id` | 負責業務 |
| `billing_company_id` | 帳務公司 |
| `payment_terms_note` | 收款條件備註（從需求單帶入並鎖定；UI label 加註「（來自需求單）」）|
| `case_name` | 案件名稱 |
| `expected_delivery_date` | 預定交期 |
| `signed_at` | 報價回簽時間 |
| `linked_quote_request_id` | 來源需求單 → [[需求單]] |
| `linked_consultation_request_id` | 來源諮詢單 → [[諮詢單]] |
| `completed_at` | 訂單完成時間 |
| `order_note` | 訂單條件備註（add-order-note-section-with-template-tool 2026-05-20，UI label「訂單須知」；印刷須知 / 不打樣聲明 / 風險告知；訂單階段業務 / 諮詢補充，500 字）|
| `delivery_note` | 交貨條件備註（同上 change；UI label「交貨備註」；工作天 / 急件 / 外島運費；500 字）|
| `payment_note` | 付款條件備註（同上 change；UI label「付款備註」；全額 / 頭尾款 / 指定日期付款補充；與 `payment_terms_note` 並存區分為「報價約定 vs 訂單階段補充」；500 字）|

## 關鍵關聯

- 包含 `OrderPrintItem[]`：訂單下的印件 → [[印件]]
- 每筆 OrderPrintItem 包含 `WorkOrder[]` → [[工單]]
- 包含 `OrderAdjustment[]`：訂單異動單（金額異動載具，獨立狀態機；refine-after-sales-refund-and-add-supplementary-print 2026-05-20 後「已執行」綁退款 Payment 建立事件，不再走獨立執行按鈕）
- 包含 `Invoice[]` + `SalesAllowance[]`：發票 + 折讓單
- 包含 `Payment[]` + `PaymentPlan[]`：付款紀錄 + 分期付款計畫
- 包含 `ShippingRecord[]` → [[出貨單]]
- 線上訂單關聯 EC payment
- 已完成後可有 `AfterSalesTicket[]` → [[售後服務]]

## OrderAdjustment（訂單異動單）行為摘要

> 詳見 [付款發票邏輯](../04-business-logic/付款發票邏輯.md) 與 OpenSpec [order-management/spec.md § Requirement: 訂單異動建立與審核](../../../../openspec/specs/order-management/spec.md)（v1.5，refine-after-sales-refund-and-add-supplementary-print 2026-05-20 歸檔）。

**狀態機**：草稿 → 待主管審核 → 已核可 / 已退回 → 已執行 / 已取消

**金額編輯閘門**（refine-after-sales-refund-and-add-supplementary-print 2026-05-20 變更）：

| status | 業務可改金額？ | 規則 |
|--------|--------------|------|
| 草稿 | 可（不限次數）| 維持「草稿」|
| 待主管審核 | 不可（送出後鎖定）| 等主管動作 |
| 已退回 | 可（修正後重送）| 重新送出 → 待主管審核 |
| 已核可（未執行）| **可（不重新送審）**| 卡片顯示「主管核可金額 vs 當前金額」對照欄位 + 半透明監督 |
| 已執行 | 不可（金錢已發生，鎖定）| — |
| 已取消 | 不可（終態）| — |

**「已執行」推進機制**（add-payment-status-and-decouple-oa-execution change 2026-05-21 變更）：

- 推進條件 MODIFY 為：對應 OA 的關聯 Payment（`linkedOrderAdjustmentId = OA.id` AND `paymentStatus = '已完成'`）累計 amount = OA.amount（**含符號比較**）
- 對稱適用退款 OA（amount < 0，配對 paymentMethod = '退款' 的負值 Payment）與補收 OA（amount > 0，配對非「退款」的正值 Payment）
- 業務於 OA 編輯介面建立關聯 Payment（預設「處理中」）+ 補齊 paidAt / attachments 後切「已完成」才推進
- 業務手動執行按鈕已移除（refine-after-sales-refund 既有設計沿用）；補收 OA「自動建補收 Payment」既有設計（spec L1719 / L1734）已**棄用**改為對稱手動建
- 回退機制（resolve [ORD-003](../08-open-questions/ORD-003-取消退款Payment是否回退OA.md)）：取消已完成 Payment 致累計不足時 OA 自動回退至「已核可」

**audit log**：每次金額異動寫入 `audit_log[]`（時間 / 操作者 / 舊金額 / 新金額 / 異動當時 status）；對應 [AFT-3 OQ](../08-open-questions/AFT-3-OA已核可改金額是否通知主管.md)（已核可改金額是否主動通知主管，留下一輪規劃）。

## 訂單備註欄位群（兩維度並存）

訂單實體含**兩組備註欄位**，分屬不同維度：

- **既有「訂單備註三類分欄」**（[order-management/spec.md § 1496-1525](../../../../openspec/specs/order-management/spec.md)）按「來源／可見性」分：
  - `customer_note`（僅線上單）— EC 前台客戶下單時填寫，唯讀
  - `internal_note`（線上 / 線下皆有）— 內部員工備註，客戶不可見
  - `production_note`（僅線下單）— 訂單製作 / 交易 / 出貨備註彙整
- **新增「訂單備註 section」**（add-order-note-section-with-template-tool 2026-05-20）按「業務主題」分：
  - `order_note` / `delivery_note` / `payment_note` — 訂單階段業務 / 諮詢補充對客戶說明的標準化條款，後續匯出至報價單／訂單確認單帶出（[ORD-012](../08-open-questions/ORD-012-訂單備註匯出至客戶文件路徑.md)）
  - UI 上 section header 含「編輯」按鈕 → 開 `OrderNotesEditDialog`（不沿用 OrderInfoEditDialog，原因見 [ORD-014](../08-open-questions/ORD-014-訂單備註與訂單資訊編輯dialog分開.md)）
  - 編輯時機：訂單完成前可編輯；訂單完成 / 取消後鎖定為唯讀
  - 編輯權限：業務 owner / 諮詢 / 業務主管 / 訂單管理人

兩維度並存的職責分工見 [ORD-013](../08-open-questions/ORD-013-新三類與既有production_note職責分工.md)。
