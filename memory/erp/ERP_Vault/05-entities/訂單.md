# 訂單（Order）

> ERP 核心實體。**三種訂單類型**：一般訂單（需求單成交）/ 諮詢訂單（諮詢結尾）/ EC 線上訂單。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `order_no` | 系統 ID / 業務看見的編號 |
| `status` | 狀態（見 [[訂單狀態]]，線下 / 線上 / 諮詢路徑不同）|
| `order_type` | `一般` / `諮詢` / `EC` |
| `customer_id` | 客戶 |
| `sales_id` | 負責業務 |
| `billing_company_id` | 帳務公司 |
| `payment_terms_note` | 收款條件備註（從需求單帶入）|
| `case_name` | 案件名稱 |
| `expected_delivery_date` | 預定交期 |
| `signed_at` | 報價回簽時間 |
| `linked_quote_request_id` | 來源需求單 → [[需求單]] |
| `linked_consultation_request_id` | 來源諮詢單 → [[諮詢單]] |
| `completed_at` | 訂單完成時間 |

## 關鍵關聯

- 包含 `OrderPrintItem[]`：訂單下的印件 → [[印件]]
- 每筆 OrderPrintItem 包含 `WorkOrder[]` → [[工單]]
- 包含 `OrderAdjustment[]`：訂單異動單（金額異動載具，獨立狀態機；refine-after-sales-refund-and-add-supplementary-print 2026-05-20 後「已執行」綁退款 Payment 建立事件，不再走獨立執行按鈕）
- 包含 `Invoice[]` + `SalesAllowance[]`：發票 + 折讓單
- 包含 `Payment[]` + `PaymentPlan[]`：付款紀錄 + 分期付款計畫
- 包含 `ShippingRecord[]` → [[出貨單]]
- 線上訂單關聯 EC payment
- 已完成後可有 `AfterSalesTicket[]` → [[售後服務]]

## OrderAdjustment（訂單異動單）行為摘要

> 詳見 [付款發票邏輯](../04-business-logic/付款發票邏輯.md) 與 OpenSpec [order-management/spec.md § Requirement: 訂單異動建立與審核](../../../../openspec/specs/order-management/spec.md)（v1.5，refine-after-sales-refund-and-add-supplementary-print 2026-05-20 歸檔）。

**狀態機**：草稿 → 待主管審核 → 已核可 / 已退回 → 已執行 / 已取消

**金額編輯閘門**（refine-after-sales-refund-and-add-supplementary-print 2026-05-20 變更）：

| status | 業務可改金額？ | 規則 |
|--------|--------------|------|
| 草稿 | 可（不限次數）| 維持「草稿」|
| 待主管審核 | 不可（送出後鎖定）| 等主管動作 |
| 已退回 | 可（修正後重送）| 重新送出 → 待主管審核 |
| 已核可（未執行）| **可（不重新送審）**| 卡片顯示「主管核可金額 vs 當前金額」對照欄位 + 半透明監督 |
| 已執行 | 不可（金錢已發生，鎖定）| — |
| 已取消 | 不可（終態）| — |

**「已執行」推進機制**：透過「建立關聯退款 Payment」事件自動推進（業務手動執行按鈕已移除）。Payment 建立需附對帳資料（退款日期 / 對帳附件 / 對帳備註），保證 OA「已執行」狀態必有對應金錢發生事實。

**audit log**：每次金額異動寫入 `audit_log[]`（時間 / 操作者 / 舊金額 / 新金額 / 異動當時 status）；對應 [AFT-3 OQ](../08-open-questions/AFT-3-OA已核可改金額是否通知主管.md)（已核可改金額是否主動通知主管，留下一輪規劃）。
