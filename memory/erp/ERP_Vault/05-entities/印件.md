# 印件（PrintItem / OrderPrintItem）

> 訂單下的「單個印刷產品項」。**雙維度狀態**：審稿維度 + 印製維度（見 [[印件狀態]]）。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `item_no` | 系統 ID / 訂單內項次 |
| `item_name` | 印件名稱（例：「2026 年度型錄」）|
| `spec_notes` | 規格說明 |
| `quantity` | 目標生產數量 |
| `difficulty_level` | 1-10 整數（[[難易度機制]]）|
| `skip_review` | 是否免審稿（[[免審決策樹]]）|
| `client_note` | 印件檔案備註（500 字，[[印件檔案備註上限]]）|
| `current_round_id` | 指向最新一輪 ReviewRound（見 [[稿件管理規則]]）|
| `審稿維度狀態` | 稿件未上傳 / 等待審稿 / 合格 / 不合格 / 已補件 / 再審 |
| `印製維度狀態` | 等待中 / 工單已交付 / 部分工單製作中 / 製作中 / 製作完成 / 出貨中 / 已送達 / 已棄用 |
| `shipping_method` / `delivery_date` | 出貨方式 / 交期 |
| `reference_file` | 參考檔 |
| `expected_production_lines` | 預計產線 |

## 關鍵關聯

- 屬於一張 `Order` → [[訂單]]
- 包含多個 `WorkOrder` → [[工單]]（齊套邏輯：一印件可有 N 個工單，例：本體 + 內頁 + 配件）
- 包含 ReviewRound 歷史（透過 `current_round_id` 指最新一輪）
- 補印場景關聯 `AfterSalesTicket` → [[售後服務]]（`related_after_sales_ticket_id`）

## 印件分類（影響生產任務帶入方式）

| 分類 | 行為 |
|------|------|
| **EC 自動產品** | 系統自動帶入生產任務 |
| **半客製產品** | 參照範本帶入，可調整 |
| **全客製產品** | 印務手動建立生產任務 |

詳見 [[印件生產流程]] § 三。

## Quote → Order 跨層欄位

| 欄位 | 行為 |
|------|------|
| `difficulty_level` | parity 同步（兩邊變動互相同步）|
| `skip_review` | parity 同步 |
| `client_note` | **脫鉤型**（成交後各自獨立編輯，不同步）|
