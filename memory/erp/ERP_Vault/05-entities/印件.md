# 印件（PrintItem / OrderPrintItem）

> 訂單下的「單個印刷產品項」。**雙維度狀態**：審稿維度 + 印製維度（見 [[印件狀態]]）。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `item_no` | 系統 ID / 訂單內項次 |
| `item_name` | 印件名稱（例：「2026 年度型錄」）|
| `type` | 印件類型（**三值**：打樣印件 / 大貨印件 / 補印印件；refine-after-sales-refund-and-add-supplementary-print 2026-05-20 擴充補印）|
| `spec_notes` | 規格說明 |
| `quantity` | 目標生產數量 |
| `difficulty_level` | 1-10 整數（[[難易度機制]]）|
| `skip_review` | 是否免審稿（[[免審決策樹]]）|
| `client_note` | 印件檔案備註（500 字，[[印件檔案備註上限]]）|
| `current_round_id` | 指向最新一輪 ReviewRound（見 [[稿件管理規則]]）|
| `審稿維度狀態` | 稿件未上傳 / 等待審稿 / 合格 / 不合格 / 已補件 / 再審 |
| `印製維度狀態` | 等待中 / 工單已交付 / 部分工單製作中 / 製作中 / 製作完成 / 出貨中 / 已送達 / **已棄用**（資料模型欄位名 `PrintItemStatus`；「已棄用」唯一觸發場景為 `sampleResult = NG-稿件問題` 自動棄用，見 [[打樣流程]]）|
| `sampleResult` | 打樣結果 enum（**四值**：`待確認` / `OK` / `NG-製程問題` / `NG-稿件問題`，預設 `待確認`；僅打樣印件使用，業務於打樣 WorkOrder 詳情頁判定）|
| `derived_from_print_item_id` | FK PrintItem nullable（**resolve-prepress-review-gaps-ar-10-ar-12 新增**）— clone 自其他印件時指向原印件，提供結構化反向查詢能力（補 `notes` 文字追溯）|
| `shipping_method` / `delivery_date` | 出貨方式 / 交期 |
| `reference_file` | 參考檔 |
| `expected_production_lines` | 預計產線 |

## 關鍵關聯

- 屬於一張 `Order` → [[訂單]]
- 包含多個 `WorkOrder` → [[工單]]（齊套邏輯：一印件可有 N 個工單，例：本體 + 內頁 + 配件）
- 包含 ReviewRound 歷史（透過 `current_round_id` 指最新一輪）
- 補印場景關聯 `AfterSalesTicket` → [[售後服務]]（`related_after_sales_ticket_id`）

## 印件分類（影響生產任務帶入方式）

| 分類 | 行為 |
|------|------|
| **EC 自動產品** | 系統自動帶入生產任務 |
| **半客製產品** | 參照範本帶入，可調整 |
| **全客製產品** | 印務手動建立生產任務 |

詳見 [[印件生產流程]] § 三。

## 印件類型（type 三值）

> refine-after-sales-refund-and-add-supplementary-print（2026-05-20）擴充補印；refine-supplementary-print-skip-review（2026-05-20）補審稿自動通過機制。

| type | 業務語意 | 建立時機 |
|------|---------|---------|
| `打樣印件` | 訂單成立後業務先打樣確認用、數量少 | 訂單階段 |
| `大貨印件` | 正式量產印件 | 訂單階段 |
| `補印印件` | 售後服務單觸發、因瑕疵 / 規格不符 / 物流問題等需重新製作 | 售後服務單階段 |

**補印印件特殊規則**：

- 必滿足 `type = '補印印件' AND related_after_sales_ticket_id IS NOT NULL`（invariant）
- **跳過審稿環節**：建立同 transaction 系統自動產生「售後補印自動通過審稿輪次」（ReviewRoundSource = '售後補印'、`sourcePrintItemId` 指向來源印件）
- **沿用來源印件最終合格稿件**：`reviewFiles` 完整拷貝來源
- 印製狀態 = 待生產（=「等待中」分類），印務主管立即可在印件總覽分配工單
- 走完整生產流程（工單 → 排程 → 生產任務 → 品檢 → 出貨），與大貨印件等效
- **補印不可同時改稿**（[AFT-6 answered](../08-open-questions/AFT-6-補印改稿情境入口設計.md)）：客戶要改稿則走新訂單流程，不在售後路徑
- **跨頁面識別**：透過共用 `PrintItemTypeLabel` 元件呈現三值（橙色「補印」標籤、hover 顯示來源 ticket、click 跳轉）

**加印（additional print）**：訂單期間客戶追加印件、不形式化為 type 列舉值（隱含設計、未來需要時擴充）

對齊業界 MES 主流（Tharstern / PrintEPS / Printavo / PrintNow 普遍預設 reorder / reprint 跳 artwork approval 直接到 production）。

## Quote → Order 跨層欄位

| 欄位 | 行為 |
|------|------|
| `difficulty_level` | parity 同步（兩邊變動互相同步）|
| `skip_review` | parity 同步 |
| `client_note` | **脫鉤型**（成交後各自獨立編輯，不同步）|
