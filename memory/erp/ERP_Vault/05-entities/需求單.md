# 需求單（QuoteRequest）

> 業務向客戶蒐集印件需求、評估成本、報價、議價的單據。**成交後生成 Order**、**流失後關閉**。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `quote_no` | 系統 ID / 業務看見的編號 |
| `title` | 案件名稱 |
| `status` | 狀態（見 [[需求單狀態]]）|
| `customer_id` | 客戶 |
| `sales_id` | 負責業務（業務主管可改派，見 XM-008 / add-sales-manager-reassign-owner change）|
| `account_company` / `billing_company_id` | 接單公司 / 帳務公司（v3.2 自動推導，唯讀） |
| `estimated_by_manager_id` | 印務主管（負責成本評估） |
| `payment_terms_note` | 收款條件備註（業務跟客戶談的紀錄） |
| `requirement_note` | 需求說明 |
| `linked_consultation_request_id` | 來源諮詢單（若從諮詢轉來）|
| `lost_reason` | 流失原因 |
| `amount_excl_tax` / `total_incl_tax` | 報價金額 |

## 關鍵關聯

- 包含 `QuoteRequestItem[]`：多筆印件項目（業務需求單階段的 `printItems`），每項對應後續[[印件]]
- 每筆印件可包含 `QuoteRequestItemAttachment[]`（參考附件）
- 每筆印件可包含 `QuoteRequestItemExpectedLine[]`（預計產線多對多）
- 來自諮詢時 `linked_consultation_request_id` 非空 → [[諮詢單]]
- 成交後 → [[訂單]]（`linked_order_id` 反向參照）
- 關聯 `QuoteRequestPriceLog`（報價紀錄快照）+ `QuoteRequestActivityLog`（操作記錄）

## 印件子實體核心欄位

| 欄位 | 說明 |
|------|------|
| `itemName` / `specNotes` | 印件名稱 / 規格 |
| `difficultyLevel` | 1-10 整數，**業務必填**（見 [[難易度機制]]）|
| `skipReview` | 是否免審稿（見 [[免審決策樹]]）|
| `clientNote` | 印件檔案備註（500 字上限，見 [[印件檔案備註上限]]）|
| `shippingMethod` / `deliveryDate` | 出貨方式 / 交期 |
| `costEstimate` / `pricePerUnit` | 印務主管評估成本 / 單價 |
