# 任務（Task / WorkGroup）

> 將工單下「**同工廠**的生產任務」歸類為同一任務。屬於工單與生產任務之間的中間層。

## 為什麼需要任務層

- 一張工單可能含多個生產任務（印刷、裁切、燙金 ...）
- 多個生產任務可能屬於不同工廠
- **同工廠的生產任務集中為一個 Task**，方便派工與管理

## 核心欄位（推測）

| 欄位 | 說明 |
|------|------|
| `id` | 系統 ID |
| `work_order_id` | 所屬工單 → [[工單]] |
| `factory_name` | 工廠名稱（分組依據）|
| `factory_type` | 工廠類型（自有 / 加工廠 / 外包 / 中國廠商）|
| `status` | 狀態（見 [[任務狀態]]）|

## 關鍵關聯

- 屬於一張 `WorkOrder` → [[工單]]
- 包含 `ProductionTask[]` → [[生產任務]]（同工廠的）

## 狀態與向上推進

```
待交付 → 已交付 → 製作中 → 已完成
```

- 工單交付後任務同步交付
- 首個生產任務進入「製作中」→ 任務自動進入「製作中」
- 所有生產任務完成 → 任務自動完成

詳見 [[任務狀態]]。

## 異動流程

- 印務發起工單異動 → 任務進入「異動」
- [[生管]]確認異動收到 → 「已確認異動內容」
- 完成異動安排 → 回到「製作中」

## 相關角色

- [[生管]]：任務分派、確認異動
- [[印務]]：建立生產任務
