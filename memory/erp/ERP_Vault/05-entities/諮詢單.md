# 諮詢單（ConsultationRequest）

> 客戶透過 surveycake webhook 自動建單，由[[諮詢]]人員自我認領處理。**雙出口**：轉需求單 / 諮詢訂單。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` | 系統 ID |
| `status` | 狀態（見 [[諮詢單狀態]]）|
| `consultation_fee` | 諮詢費（通常 1000 元）|
| `customer_type` | 客戶類型 |
| `company_name` / `contact_name` / `mobile` / `email` | 客戶聯絡資訊 |
| `consultation_topic` | 客戶 surveycake 原話（**唯讀**，對齊 ISO 9001 客戶指示可追溯不可竄改原則）|
| `consultant_note` | 諮詢人員與客戶溝通記錄（最長 2000 字，非必填，可多次編輯，每次編輯寫入 ActivityLog）|
| `estimated_quantity_band` | 預估數量區間 |
| `reserved_date` / `reserved_time` | 預約諮詢時間 |
| `consultant_id` | 諮詢人員（諮詢人員自我認領時寫入；主管可代為認領）|
| `linked_quote_request_id` | 若轉需求單則關聯 [[需求單]] |
| `linked_consultation_order_id` | 若完成諮詢或流失則關聯諮詢[[訂單]] |
| `consulted_at` | 諮詢完成時間 |

## 關鍵關聯

- 包含 `Payment[]`：諮詢費 Payment（`linked_entity_type = ConsultationRequest`）
- 轉需求單時 `linked_quote_request_id` 非空 → [[需求單]]，**Payment 轉移至主訂單**；`consultation_topic` + `consultant_note` 以雙區塊格式合併寫入需求單 `requirement_note`
- 完成不做大貨 / 取消時 `linked_consultation_order_id` 非空 → [[訂單]]（`order_type = 諮詢訂單`）

## 業務流程（雙出口）

```
surveycake webhook → 自動建單（待諮詢）
  ↓ Slack 通知諮詢人員群組廣播待認領
諮詢人員自我認領（consultant_id 寫入；主管可代為認領）
  ↓ 諮詢人員可於 consultant_note 編輯溝通記錄，consultation_topic 唯讀
諮詢人員諮詢 → 選擇結局：
  ├── 轉需求單做大貨 → 建 QuoteRequest（requirement_note 雙區塊預設值），Payment 轉移
  ├── 完成不做大貨 → 建諮詢訂單（含諮詢費）
  └── 取消（待諮詢取消）→ 退費（Payment 反向）
```
