# 出貨單狀態（ShippingStatus）

> [[出貨單]] 的狀態機。**出貨類型決定狀態路徑**（自行出貨 / 第三方物流）。

## 狀態路徑

### 1. 自行出貨路徑

```
未處理 → 打包中 → 待出貨 → 出貨中 → 已送達
```

### 2. 第三方物流路徑

```
未處理 → 已出貨 → 運送中 → 已送達
```

## 主要轉移觸發

| From → To | 觸發 | 推進方式 |
|-----------|------|---------|
| - → 未處理 | 訂單建立出貨單 | 手動 |
| 未處理 → 打包中（自行）| 倉管開始打包 | 手動 |
| 打包中 → 待出貨 | 完成打包 | 手動 |
| 待出貨 → 出貨中 | 開始出貨 | 手動 |
| 出貨中 → 已送達 | 貨物送達客戶 | 手動 |
| 未處理 → 已出貨（第三方）| 交由第三方物流 | 手動 |
| 已出貨 → 運送中 | 物流公司開始運送 | 手動 |
| 運送中 → 已送達 | 物流送達 | 手動 |

## 自動推進

**無**。全部由[[出貨]]手動推進。

## 數量防呆

```
出貨數量 ≤ QC 入庫數 - 已出貨數
```

範例：QC 入庫 500，已出貨 300 → 新出貨上限 = 200

詳見 [[印件生產流程]] § 五、入庫與出貨數量規則。

## 出貨單作廢回算（SHP-003）

```
新可出貨額度 = 入庫數量 - (已出貨數量 - 作廢數量)
```

詳見 [[出貨單]] § 出貨單作廢回算。

## 多工單齊套性（SHP-002）

當印件含多工單時，**取最小入庫數**為可出貨上限：

```
可出貨 = min(工單 A 入庫, 工單 B 入庫, ...)
```

詳見 [[齊套邏輯]] § 三、典型情境 § 情境 3。

## 分批出貨（OQ SHP-005）

> [!question] OQ
> 分批出貨觸發節點設計尚未確認。[[出貨]]角色關切此議題。

## 影響訂單狀態

```
所有印件出貨完成 → 訂單狀態：出貨中 → 訂單完成
```

詳見 [[訂單狀態]]。
