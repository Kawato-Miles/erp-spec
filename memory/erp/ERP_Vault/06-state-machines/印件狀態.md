# 印件狀態（PrintItemStatus，雙維度）

> [[印件]] 同時擁有**兩個維度**的狀態：**審稿維度** + **印製維度**。兩維度獨立推進但相互關聯。

## 一、審稿維度

```
稿件未上傳 → 等待審稿 → 合格 ↔ 不合格 ↔ 已補件 ↔ 再審
                                            ↓
                                        免審稿直通：稿件未上傳 → 合格
```

### 主要轉移觸發

| From → To | 觸發 |
|-----------|------|
| 稿件未上傳 → 等待審稿 | 稿件首次上傳 |
| 等待審稿 → 合格 | 審稿人員首審通過 |
| 等待審稿 → 不合格 | 審稿人員首審不通過 |
| 不合格 → 已補件 | 客戶 / 業務補件完成 |
| 已補件 → 再審 → 合格 / 不合格 | 審稿人員重審 |
| 稿件未上傳 → 合格 | **免審稿直通**（[[免審決策樹]]） |
| 任何狀態 → 已棄用 | 打樣 NG 判定不製作大貨時 |

## 二、印製維度

```
等待中 → 工單已交付 → 部分工單製作中 → 製作中 → 製作完成 → 出貨中 → 已送達
                                                                ↓
                                                            已棄用
```

### 主要轉移觸發

| From → To | 觸發 |
|-----------|------|
| 等待中 → 工單已交付 | 至少一張工單交付 |
| 工單已交付 → 部分工單製作中 | 部分工單進入製作中（向上傳遞）|
| 部分工單製作中 → 製作中 | 所有工單進入製作中 |
| 製作中 → 製作完成 | 齊套達標（見 [[齊套邏輯]]）|
| 製作完成 → 出貨中 | 出貨單建立 |
| 出貨中 → 已送達 | 全數送達 |

## 三、兩維度的關係

| 規則 | 說明 |
|------|------|
| 審稿維度合格後才開始推進印製維度 | 不合格 / 待補件期間印製維度停在「等待中」|
| 審稿維度推進不影響已建工單的鎖定檔案 | 見 [[稿件管理規則]] § 三 |
| 印製維度由下層[[工單狀態]] / [[生產任務狀態]]向上傳遞驅動 | Bubble-up |

## 四、自動推進邏輯

**雙維度都有自動推進**：

- 審稿維度：審稿人員結果直接驅動
- 印製維度：底層生產任務狀態 → 任務 → 工單 → 印件（多層 bubble-up）
