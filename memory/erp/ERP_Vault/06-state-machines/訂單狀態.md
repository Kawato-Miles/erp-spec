# 訂單狀態（OrderStatus）

> [[訂單]] 的狀態機。**三條路徑**（線下 / 線上 / 諮詢）+ 共用段。

## 狀態清單

### 線下路徑

```
報價待回簽 → 已回簽 → [共用段]
```

### 線上路徑

```
等待付款 → 已付款 → [共用段]
```

### 諮詢訂單路徑

依收尾情境分兩終態（2026-05-30 converge-consultation-cancel-to-order-cancel-flow 收斂）：

```
建立 →（諮詢取消）→ 已取消
建立 →（不做大貨 / 需求單流失）→ 訂單完成
```

- **諮詢取消**：終態「已取消」（沒成交的生意、語意為已取消而非完成）；建立時系統自動建退款 OA（負 1000、已核可、approved_by=system）+ 退款 Payment（負 1000、處理中），不自動開 Invoice / SalesAllowance / 待開發票；善後金流不鎖（退款 Payment 切已完成推進 OA 已執行，但不再變動訂單已取消終態）
- **不做大貨 / 需求單流失**：終態「訂單完成」；建立時系統自動建待開發票 1 筆（2000）作提醒
- 諮詢訂單不經「已開發票」中間態（refine-consultation-cancellation-and-invoice-flow 廢止）

### 共用段（線下 / 線上接續）

```
稿件未上傳 → 等待審稿 ↔ 待補件 → 製作等待中
  → 工單已交付 → 製作中 → 製作完成 → 出貨中 → 訂單完成
```

## 主要轉移觸發

| From → To | 觸發 | 推進方式 |
|-----------|------|---------|
| 報價待回簽 → 已回簽 | 線下業務上傳回簽檔案或手動確認 | 手動 |
| 等待付款 → 已付款 | 線上訂單 EC 付款完成 | **自動** |
| 已回簽 / 已付款 → 稿件未上傳 | 進入共用段 | 自動 |
| 稿件未上傳 → 等待審稿 | 客戶 / 業務上傳稿件 | 自動 |
| 等待審稿 → 待補件 | 審稿不合格 | 自動（審稿維度 bubble-up）|
| 製作等待中 → 工單已交付 | 首個工單已交付 | **自動向上傳遞** |
| 工單已交付 → 製作中 | 首個生產任務進入製作中 | **自動向上傳遞** |
| 製作中 → 製作完成 | 所有印件製作完成 | 自動（齊套達標）|
| 製作完成 → 出貨中 | 出貨單建立 | 自動 |
| 出貨中 → 訂單完成 | 所有出貨單送達 | 自動 |

## 自動推進（向上傳遞）

訂單狀態大量依靠**狀態向上傳遞**（[[glossary-erp#系統行為術語|Bubble-up]]）：

```
生產任務製作中 → 任務製作中 → 工單製作中 → 訂單製作中
```

詳見 [[印件狀態]] § 雙維度狀態。

## 訂單編輯閘門

- v1.4 change：訂單編輯閘門從「已回簽」改為「**製作等待中（含）**」
- 審稿段內可自由編輯

## 訂單階段業務主管 gate

- 業務主管審核發生在「訂單建立後 → 報價待回簽前」
- 需求單階段業務獨立完成，無 gate
