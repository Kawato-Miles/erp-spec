# 諮詢單狀態（ConsultationRequestStatus）

> [[諮詢單]] 的狀態機。surveycake webhook 自動建單，**雙出口**：轉需求單 / 諮詢訂單。

## 狀態清單

```
待諮詢 → 諮詢中 → 已完成（雙出口）
                  ↓
             待諮詢取消（退費）
```

## 狀態說明（推測，Phase B 補完整）

| 狀態 | 說明 |
|------|------|
| 待諮詢 | webhook 自動建立後，等待業務指派諮詢人員 |
| 諮詢中 | 諮詢人員執行諮詢 |
| 已完成 - 轉需求單 | 諮詢人員選擇「轉做大貨」→ 建 [[需求單]]，Payment 轉移 |
| 已完成 - 完成不做大貨 | 諮詢人員選擇「完成不做大貨」→ 建諮詢[[訂單]]（含諮詢費 1000）|
| 待諮詢取消 | 諮詢前取消 → 退費（Payment 反向）|

## 主要轉移觸發

| From → To | 觸發 |
|-----------|------|
| - → 待諮詢 | surveycake webhook 自動建單 |
| 待諮詢 → 諮詢中 | 業務指派諮詢人員 |
| 諮詢中 → 已完成（轉需求單）| 諮詢人員點「轉需求單」，建 QuoteRequest，Payment 轉移 |
| 諮詢中 → 已完成（不做大貨）| 諮詢人員點「完成不做大貨」，建諮詢訂單 |
| 待諮詢 → 待諮詢取消 | 諮詢前客戶取消 → 退費 |

## 自動推進

**無**。但有自動建單觸發（surveycake webhook → 待諮詢）。

## 關聯狀態機

- → [[需求單狀態]]（轉需求單時）
- → [[訂單狀態]]（諮詢訂單時，order_type = 諮詢）
