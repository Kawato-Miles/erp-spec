## 一、單據層級結構

四層單據依序串聯，上層建立後才能產生下層：

```
需求單
 └─ 訂單（含 N 個印件）
      └─ 工單（含 N 個任務）
           └─ 任務（紙本工單，含 N 個生產任務）
```

| 單據 | 核心內容 | 主要使用者 |
|------|----------|------------|
| 需求單 | 基本資料、印件規格、預計製程、報價紀錄、活動紀錄 | 業務 / 諮詢業務 |
| 訂單 | 基本資料、印件 × N（含稿件/預覽圖）、工單 × N、付款紀錄、出貨單、盈利資料、補收款訂單 | 業務、訂單管理 |
| 工單 | 任務 × N、生產任務（工序）× N、QC 紀錄 | 印務、印務主管 |
| 任務（紙本工單） | 印件、生產任務（工序）× N、報工紀錄 | 生產廠長、師傅、外包廠 |

**建立規則：**
- 審稿合格後，系統自動依印件中設定的製程與材料產生工單內容
- 若為全客製化商品，由印務自行建立工單
- 同廠商的工序歸類在同一任務中

## 二、業務流程五大階段

```
評估階段 → 打單階段 → 審稿階段 → 打樣/印製階段 → 出貨階段
```

| 階段 | 主要活動 | 主要角色 |
|------|----------|----------|
| 評估階段 | 客戶需求確認、成本評估、需求單建立、報價 | 業務、諮詢業務、製程審核、印務主管 |
| 打單階段 | 正式訂單建立（線上/線下）、款項確認 | 業務、諮詢業務、會計 |
| 審稿階段 | 稿件合規確認、Preflight 檢查、稿件退回/通過 | 審稿、審稿主管 |
| 打樣/印製階段 | 工單建立、生產任務分派、師傅執行、報工紀錄、QC 確認 | 印務主管、印務、生產廠長、師傅、外包廠、QC |
| 出貨階段 | 成品入庫、分批出貨、出貨單管理 | 出貨、業務 |

## 三、角色與資料權限

| 角色 | 主要職責 | 資料限制 |
|------|----------|----------|
| Supervisor | Dashboard、Forecast、KPI 追蹤 | 不直接編輯資料 |
| 業務 | Sales process、需求單、線下訂單、款項發票 | 只能看屬於自己或被授權的單；可轉單給其他業務/諮詢 |
| 諮詢業務 | 同業務，另含 EC 金額/資料異動客服 | 同業務限制 |
| 印務主管 | 製程審核、建立工單草稿、分派印務、追蹤工單 | BOM 表管理 |
| 印務 | 建立生產任務、工單追蹤與異動管理 | 僅可查閱印件規格；客戶資料變更需由業務/諮詢處理 |
| 生產廠長 | 工作任務安排、工單異動執行 | 以紙本為主；僅可編輯生產任務進度，不可改任務內容 |
| 師傅 | 執行生產任務、每日報工 | 以紙本為主；僅可回填進度 |
| 外包廠 / 自有廠 | 依任務報價、執行生產 | 僅可回填進度與是否接單報價 |
| 審稿 | 稿件合規確認、EC 初步客服 | 排除工單與營收資料 |
| 審稿主管 | 審稿案件分派 | 只能看基本資料與印件規格 |
| QC | 確認成品是否符合規格 | 僅可編輯 QC 紀錄 |
| 出貨 | 出貨單管理 | 訂單僅可編輯出貨單；工單僅可查閱狀態與 QC 紀錄 |
| 製程審核 | 售前成本評估、BOM 表管理、工單製程審核 | 主要為審核與 BOM 維護 |
| 會計 | 金流查帳、發票開立 | 只能看基本資料與付款紀錄 |
| 工程師 | BOM 表管理 | 主要業務為 BOM 維護 |
| 訂單管理人 | 追蹤與管理全公司訂單進度、工單分配給印務 | R/W（評估、打單、印製、出貨階段） |

## 四、印件數量計算規則（核心邏輯）

### 4.1 前提假設

1. **以印件為單位**統計生產數量與出貨，不以生產任務數量計
2. **生產任務數量 ≠ 印件數量**，需經換算（如：生產 2 個壓克力才完成 1 個印件）
3. 工單建立時，可能僅製作部分部件（不含完整印件），此類工單不需出貨
4. 分批出貨：在工單尚未全部完成前即可出貨，這也是需要即時計算印件完成數量的原因

### 4.2 換算層級：逐層計算（非跨層）

```
印件（業務填寫數量）
 └─ 工單（每份印件生產數量 = 工單相對印件的比例）
      └─ 生產任務（每份工單需生產數量 = 任務相對工單的比例）
```

**關鍵設計決策：生產任務的比例參照直接上層（工單），而非印件。**
這樣每層只需了解與直接上層的關係，避免跨層換算的複雜性。

### 4.3 工單完成數量計算

```
工單完成數 = floor( min over 所有「影響成品」的生產任務 of
                   ( 生產任務 QC 通過數 ÷ 每份工單需生產數量 ) )
```

- **影響成品**：需設旗標標記，由印務建立工單時設定
- **不影響成品**（如採購紙張、開版）：不計入工單完成數計算
- 取 **floor（無條件捨去）**，零頭視為「製作中」，需在 UI 顯示以避免誤讀

### 4.4 印件完成數量計算

```
印件完成數 = floor( min over 所有工單 of
                   ( 工單完成數 ÷ 每份印件生產數量 ) )
```

### 4.5 入庫與出貨

```
入庫數量 = 生產任務 QC 通過後才計入（非生產完成即入庫）
出貨數量 ≤ 入庫數量（不可超出已入庫的成品）
```

### 4.6 數量計算範例

**情境：1式 = 1000 個專輯 + 300 本歌詞本**

| 層級 | 欄位 | 值 |
|------|------|----|
| 印件 | 數量 | 1（1式） |
| 工單A（專輯） | 每份印件生產數量 | 1000 |
| 工單A 生產任務X | 每份**工單**需生產數量 | 2 |
| 工單B（歌詞本） | 每份印件生產數量 | 300 |
| 工單B 生產任務Y | 每份**工單**需生產數量 | 1 |

計算過程（假設任務X QC通過 1200，任務Y QC通過 280）：
- 工單A完成數 = floor(1200 ÷ 2) = 600
- 工單B完成數 = floor(280 ÷ 1) = 280
- 印件完成數 = floor(min(600 ÷ 1000, 280 ÷ 300)) = floor(min(0.6, 0.933)) = floor(0.6) = **0**

（工單A尚未完成1000個目標，印件完成數為0）

### 4.7 預計生產數量

- 由**印務自行決定 buffer**，規則：預計生產數量 **不可小於**印件所需數量
- 預計生產數量設定於工單層，生產任務預計數量依工單預計數量 × 比例自動計算

### 4.8 報價單印件填寫原則

| 情境 | 建議做法 |
|------|----------|
| 同批商品確定一起出貨 | 合併為一個印件 |
| 各別商品獨立出貨 | 拆開為多個印件 |
| 業務填「1式」 | 建議轉換為實際數量（如 300 個），以利生產數量計算 |
| 打樣 vs 大貨 | 建議分開建立印件 |

## 五、打樣流程

### 5.1 工單類型

| 類型 | 說明 | 建立時機 |
|------|------|----------|
| **打樣工單** | 生產少量樣品，供業務與客戶確認成品 | 印件審稿通過後 |
| **大貨工單** | 正式大量生產 | 打樣確認 OK 後 |

**以 type 欄位區分**，兩種工單分別建立。

### 5.2 印件與工單的建立時機

- 報價單建立時，**打樣印件 + 大貨印件同時建立**（方案 A）
- 大貨印件在打樣確認前處於「等待打樣確認」狀態，不進入生產流程
- 打樣 OK 後，大貨印件才啟動後續審稿 → 工單建立流程

### 5.3 打樣決策點（業務確認後）

業務與印務討論後，共同決定重新打樣的原因，並在系統中觸發對應動作：

```
打樣完成 → 業務取得成品 → 業務 + 印務確認
    ├─ OK
    │    → 業務通知印務啟動大貨印件
    │
    ├─ 不 OK（製程問題）
    │    → 在同一打樣印件下，建立新的打樣工單
    │    → 印務調整製程設定後執行新工單
    │    → 舊打樣工單標記為「打樣不通過」保留紀錄
    │
    └─ 不 OK（稿件問題）
         → 建立新的打樣印件，走完整的印件流程
         → 新打樣印件：審稿 → 製程審核 → 打樣工單建立 → 生產
         → 原打樣印件標記為「作廢」保留紀錄
```

**重新打樣的判斷責任**：印務 + 業務共同討論決定，系統需提供明確的「觸發重新打樣」動作，要求選擇原因（製程問題 / 稿件問題）。

### 5.4 稿件管理規則

- 每個印件可獨立上傳多個稿件檔案
- 打樣後若需重新處理稿件，**建立全新的打樣印件**，重新跑完整印件流程；不修改原稿件，歷史保存於原打樣印件
- **大貨印件的稿件**：不需在報價單建立時就上傳；等打樣全部確認 OK 後，再手動更新大貨印件的稿件

## 六、工單分派流程

```
審稿通過
 → 印務主管建立工單草稿
 → 訂單管理人分派工單給印務（依負載評估）
 → 印務填寫工單詳細內容（製程、材料、生產任務）
 → 生管接手，分派生產任務給工廠 / 師傅
```

**關鍵規則**：
- 一個訂單可由**多位印務**分開負責
- 一張工單對應**一位負責印務**
- 工單異動：由印務提出需求，**印務主管執行**

### 8.1 訂單狀態機

#### 線下訂單路徑
```
報價待回簽 → 已回簽 → 稿件未上傳 → 稿件已上傳 → 付款未完成 → 已付款
```

#### 線上訂單路徑
```
等待付款 → 已付款
```

#### 共通後續（已付款後）
```
已付款
 → 製作等待中（工單尚未建立）
 → 工單已交付（工單草稿分派後）
 → 製作中（任一工單開始製作）
 → 製作完成（所有印件製作完成）
 → 出貨未處理 → 打包中 → 物流未完成
 → 訂單完成
```

#### 補收款（平行，不阻塞主流程）
```
補收款未付款 ↔ 補收款已付款
```

> 補收款適用於急件費用、重新打樣補收等情境；與主訂單流程獨立追蹤，不影響生產推進。

---

### 8.2 工單狀態機

```
草稿
 → 製程確認中（印務填寫工單內容後送審）
   ↔ 重新確認製程（印務主管退回要求調整）
 → 製程審核完成（印務主管確認通過）
 → 工單已交付（正式交付生產）
 → 製作中（生產任務開始執行）
   ↔ 異動（任務需求異動，需生管確認後恢復製作中）
 → 待 QC（生產任務完成後）
 → QC 已通過 → 入庫
   QC 不通過 → 重建任務（重新執行生產任務）
```

**角色對應：**
- **印務**：填寫工單內容，送製程審核
- **印務主管**：執行製程審核（通過 / 退回）、執行工單異動
- **生管**：確認異動、分派生產任務
- **QC**：執行品質確認，決定入庫或重建任務

> 工單狀態機不含出貨追蹤；出貨狀態移至印件層管理。

---

### 8.3 印件狀態機

#### 審稿與製程階段
```
等待稿件
 → 稿件審核中（稿件上傳後）
   → 稿件不合格 → 退稿（回等待稿件）
 → 稿件合格（審稿通過）
 → 製程審核中（印務主管確認製程與工單規格）
 → 工單製作中（工單草稿建立完成）
```

#### 印製與出貨階段
```
等待中（工單尚未交付）
 → 工單已交付
 → 製作中（生產執行中）
   → 部分已出貨（條件：印件出貨數量 > 0，分批出貨中）
 → 製作完成（條件：印件中所有工單皆為「已完成」）
 → 已入庫（條件：印件中所有工單皆已入庫，QC 全數通過）
 → 打包中 → 待出貨 → 出貨中
 → 已出貨（全部出貨完成）
 → 運送中
 → 已送達
```

**分批出貨說明：**
- 同一印件可分批出貨（例：共 1000 個，先出 300、再出 700）
- 已出貨數量 > 0 時，印件進入**部分已出貨**狀態
- 全數出貨後才轉為**已出貨**

> 舊版流程圖中「已出貨 ↔ 運送中」雙向箭頭為錯誤，正確為單向：已出貨 → 運送中 → 已送達。

### 8.4 任務（紙本工單）狀態機

> 任務是派發給各工廠的執行單位，每個任務含 1～N 筆生產任務。狀態依工廠類型有不同的終態路徑。

#### 主流程

```
待交付（待印務提供工單給工廠）
 → 已作廢（取消，尚未交付前）
 → 已交付（印務已提供工單）
 → 製作中（生管確認開始，任一生產任務進入製作中）
   ↔ 異動（印務發起異動工單）
     → 已確認異動內容（生管確認後重新印製工單）
     → 回到製作中
 → [依工廠類型分流，見下]
```

#### 終態路徑（依工廠類型）

| 工廠類型 | 製作中後的狀態路徑 |
|----------|-------------------|
| 自有工廠 / 加工廠 | → 已完成 |
| 外包廠 | → 運送中 → 已抵達 → 已完成 |
| 中國廠商 | → 已送集運商 → 運送中 → 已抵達 → 已完成 |

> 「已完成」代表此廠商的工序結束，材料已回場。後續場內加工為另一筆獨立的生產任務（自有工廠執行），兩者間存在任務相依性（前置任務完成才啟動下一筆）。相依機制（自動 vs 手動觸發）待 UX / 開發階段設計。

---

### 8.5 生產任務狀態機

> 生產任務是最小追蹤單位，用於計算工單完成數量（QC 通過數）。工廠類型於建立時由 BOM 參數設定決定，決定對應狀態路徑。

```
待處理
 → 已作廢（尚未進入生產即取消）
 → 製作中（生產開始）
   → 已完成（自有工廠 / 加工廠；條件：所有報工完成數量 = 任務數量）
   → 運送中（外包廠；完成生產後交付物流）
     → 已完成
   → 已送集運商 → 運送中（中國廠商）
     → 已完成
   → 報廢（已進入生產但取消；以報工完成數量計算成本）
```

**工廠類型對應狀態路徑：**

| 工廠類型 | 路徑 |
|----------|------|
| 自有工廠 / 加工廠 | 製作中 → 已完成 |
| 外包廠 | 製作中 → 運送中 → 已完成 |
| 中國廠商 | 製作中 → 已送集運商 → 運送中 → 已完成 |

**終止狀態說明：**
- `已作廢`：尚未進入生產即取消
- `報廢`：已進入生產後取消，仍以報工完成數量計算成本
- `已完成`：所有報工完成數量 = 任務數量，且 QC 通過數記錄完成
