# ERP 狀態機（State Machines）

> 所有單據的狀態轉換規則。設計 spec 或 prototype 時，所有狀態必須與本文件對齊。
>
> 最後更新：2026-02-21
> 狀態：六層相依鏈全數確認完成

---

**層級相依概覽（由上至下）：**
```
需求單 → 訂單 → 工單 → 印件 → 任務 → 生產任務
```
上層狀態由下層聚合決定；下層需等上層建立後才能產生。

---

## 需求單狀態機

> 概念類似 Salesforce Sales Cloud 的機會管理（Opportunity）。成交後銜接訂單流程；流失則關單，不建立訂單。

```
需求確認中（收到客戶需求，釐清規格中）
 → 待評估報價（需求確認，待印務主管評估成本與售價）
 → 已評估報價（成本與售價確認，可對外報價）
 → 已提供報價（已將報價提供給客戶）
 → 議價中（待客戶回應，可能需調整方案）
   → 重新提供方案：退回「待評估報價」，與印務主管重新討論
   → 成交：需求成立 → 接訂單流程（建立正式報價單）
   → 流失：需求不成立 → 關單，不建立訂單
```

**角色對應：**
- **業務 / 諮詢業務**：建立需求單、推進報價流程、與客戶議價
- **印務主管**：評估製程成本與售價

**注意事項：**
- 各階段可設定成交機率，供 pipeline 管理與營收預估使用（功能待設計）
- 流失後需求單保留紀錄，不得刪除

## 訂單狀態機

#### 線下訂單路徑
```
報價待回簽 → 已回簽 → 稿件未上傳 → 稿件已上傳 → 付款未完成 → 已付款
```

#### 線上訂單路徑
```
等待付款 → 已付款
```

#### 共通後續（已付款後）
```
已付款
 → 製作等待中（工單尚未建立）
 → 工單已交付（工單草稿分派後）
 → 製作中（任一工單開始製作）
 → 製作完成（所有印件製作完成）
 → 出貨未處理 → 打包中 → 物流未完成
 → 訂單完成
```

#### 補收款（平行，不阻塞主流程）
```
補收款未付款 ↔ 補收款已付款
```

> 補收款適用於急件費用、重新打樣補收等情境；與主訂單流程獨立追蹤，不影響生產推進。

## 工單狀態機

```
草稿
 → 製程確認中（印務填寫工單內容後送審）
   ↔ 重新確認製程（印務主管退回要求調整）
 → 製程審核完成（印務主管確認通過）
 → 工單已交付（正式交付生產）
 → 製作中（生產任務開始執行）
   ↔ 異動（任務需求異動，需生管確認後恢復製作中）
 → 待 QC（生產任務完成後）
 → QC 已通過 → 入庫
   QC 不通過 → 重建任務（重新執行生產任務）
```

**角色對應：**
- **印務**：填寫工單內容，送製程審核
- **印務主管**：執行製程審核（通過 / 退回）、執行工單異動
- **生管**：確認異動、分派生產任務
- **QC**：執行品質確認，決定入庫或重建任務

> 工單狀態機不含出貨追蹤；出貨狀態移至印件層管理。

## 印件狀態機

#### 審稿與製程階段
```
等待稿件
 → 稿件審核中（稿件上傳後）
   → 稿件不合格 → 退稿（回等待稿件）
 → 稿件合格（審稿通過）
 → 製程審核中（印務主管確認製程與工單規格）
 → 工單製作中（工單草稿建立完成）
```

#### 印製與出貨階段
```
等待中（工單尚未交付）
 → 工單已交付
 → 製作中（生產執行中）
   → 部分已出貨（條件：印件出貨數量 > 0，分批出貨中）
 → 製作完成（條件：印件中所有工單皆為「已完成」）
 → 已入庫（條件：印件中所有工單皆已入庫，QC 全數通過）
 → 打包中 → 待出貨 → 出貨中
 → 已出貨（全部出貨完成）
 → 運送中
 → 已送達
```

**分批出貨說明：**
- 同一印件可分批出貨（例：共 1000 個，先出 300、再出 700）
- 已出貨數量 > 0 時，印件進入**部分已出貨**狀態
- 全數出貨後才轉為**已出貨**

> 舊版流程圖中「已出貨 ↔ 運送中」雙向箭頭為錯誤，正確為單向：已出貨 → 運送中 → 已送達。

## 任務（紙本工單）狀態機

> 任務是派發給各工廠的執行單位，每個任務含 1～N 筆生產任務。狀態依工廠類型有不同的終態路徑。

#### 主流程

```
待交付（待印務提供工單給工廠）
 → 已作廢（取消，尚未交付前）
 → 已交付（印務已提供工單）
 → 製作中（生管確認開始，任一生產任務進入製作中）
   ↔ 異動（印務發起異動工單）
     → 已確認異動內容（生管確認後重新印製工單）
     → 回到製作中
 → [依工廠類型分流，見下]
```

#### 終態路徑（依工廠類型）

| 工廠類型 | 製作中後的狀態路徑 |
|----------|-------------------|
| 自有工廠 / 加工廠 | → 已完成 |
| 外包廠 | → 運送中 → 已抵達 → 已完成 |
| 中國廠商 | → 已送集運商 → 運送中 → 已抵達 → 已完成 |

> 「已完成」代表此廠商的工序結束，材料已回場。後續場內加工為另一筆獨立的生產任務（自有工廠執行），兩者間存在任務相依性（前置任務完成才啟動下一筆）。相依機制（自動 vs 手動觸發）待 UX / 開發階段設計。

## 生產任務狀態機

> 生產任務是最小追蹤單位，用於計算工單完成數量（QC 通過數）。工廠類型於建立時由 BOM 參數設定決定，決定對應狀態路徑。

```
待處理
 → 已作廢（尚未進入生產即取消）
 → 製作中（生產開始）
   → 已完成（自有工廠 / 加工廠；條件：所有報工完成數量 = 任務數量）
   → 運送中（外包廠；完成生產後交付物流）→ 已完成
   → 已送集運商 → 運送中（中國廠商）→ 已完成
   → 報廢（已進入生產但取消；以報工完成數量計算成本）
```

**工廠類型對應狀態路徑：**

| 工廠類型 | 路徑 |
|----------|------|
| 自有工廠 / 加工廠 | 製作中 → 已完成 |
| 外包廠 | 製作中 → 運送中 → 已完成 |
| 中國廠商 | 製作中 → 已送集運商 → 運送中 → 已完成 |

**終止狀態說明：**
- `已作廢`：尚未進入生產即取消
- `報廢`：已進入生產後取消，仍以報工完成數量計算成本
- `已完成`：所有報工完成數量 = 任務數量，且 QC 通過數記錄完成
