## 設計決策（已確認項目）

> v2.0 檢視權限功能設計確認（已實裝 Prototype）

### QR-005：需求單檢視權限設計

**決策：** 擁有者獨占編輯，被授權人檢視專用

| 層面 | 決策 | 備註 |
|------|------|------|
| **角色定義** | 擁有者 = 建立者<br/>被授權人 = 由擁有者授予檢視權限的人 | — |
| **編輯權限** | 僅擁有者可編輯所有欄位和功能 | 基本資料、印件規格、權限管理皆限擁有者 |
| **被授權人檢視** | 只讀模式：可查閱所有資訊，無編輯功能 | 包括基本資料、印件規格、報價紀錄、活動紀錄 |
| **權限管理** | 僅擁有者可新增/移除檢視權限 | 新增時需搜尋業務/諮詢；移除時需點擊移除按鈕 |
| **UI 提示** | 被授權人進入詳情頁時顯示「你有檢視權限」警告橫幅 | 黃色背景，包含建立者名稱和「無法編輯」提示 |
| **權限記錄** | 每次授予權限記錄：授予者、授予時間 | 在權限管理 Tab 顯示；移除後權限紀錄保留於活動紀錄 |
| **列表頁指示** | 「僅我」（紫色）或「已分享 (N 人)」（綠色）| 方便業務快速判斷是否已分享 |
| **邊界情境** | — 成交/流失後是否仍可分享？ → **可分享**（成交/流失不影響權限管理）<br/>— 刪除需求單時權限如何處理？ → **不刪除**（保留權限紀錄於活動日誌，便於追溯） | 設計原則：權限為獨立維度，不與業務流程狀態綁定 |

**實裝版本：** v2.0（Prototype 已實作完整功能）
