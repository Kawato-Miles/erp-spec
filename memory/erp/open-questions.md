### QR-005：需求單檢視權限設計

**決策：** 擁有者獨占編輯，被授權人檢視專用

| 層面 | 決策 | 備註 |
|------|------|------|
| **角色定義** | 擁有者 = 建立者<br/>被授權人 = 由擁有者授予檢視權限的人 | — |
| **編輯權限** | 僅擁有者可編輯所有欄位和功能 | 基本資料、印件規格、權限管理皆限擁有者 |
| **被授權人檢視** | 只讀模式：可查閱所有資訊，無編輯功能 | 包括基本資料、印件規格、報價紀錄、活動紀錄 |
| **權限管理** | 僅擁有者可新增/移除檢視權限 | 新增時需搜尋業務/諮詢；移除時需點擊移除按鈕 |
| **UI 提示** | 被授權人進入詳情頁時顯示「你有檢視權限」警告橫幅 | 黃色背景，包含建立者名稱和「無法編輯」提示 |
| **權限記錄** | 每次授予權限記錄：授予者、授予時間 | 在權限管理 Tab 顯示；移除後權限紀錄保留於活動紀錄 |
| **列表頁指示** | 「僅我」（紫色）或「已分享 (N 人)」（綠色）| 方便業務快速判斷是否已分享 |
| **邊界情境** | — 成交/流失後是否仍可分享？ → **可分享**（成交/流失不影響權限管理）<br/>— 刪除需求單時權限如何處理？ → **不刪除**（保留權限紀錄於活動日誌，便於追溯） | 設計原則：權限為獨立維度，不與業務流程狀態綁定 |

### SHP-002 ~ SHP-004：出貨設計核心決策

**決策日期：** 2026-03-02

#### SHP-002：一印件多工單時出貨的前置條件

**問題：** 當一個印件包含多個工單（如情境 2）時，出貨防呆應如何判斷「可出貨額度」？

**決策：** 出貨永遠**以印件為單位**；防呆檢查應使用**印件層級的 QC 入庫數量**（跨工單統計）

| 層面 | 決策內容 | 說明 |
|------|--------|------|
| **出貨單位** | 印件（不分工單出貨） | 出貨單結構：出貨單 ⊃ 出貨明細（印件 + 數量） |
| **防呆判斷** | 以印件層級累計 QC 入庫為基準 | 本次出貨 ≤ 印件累計 QC 入庫 - 已出貨 |
| **入庫計算規則** | 參考 `quantity-calculation-rules.md` | 多工單時，印件完成數 = min(floor(WO-A完成 ÷ WO-A倍數), floor(WO-B完成 ÷ WO-B倍數), ...)；見場景 C |
| **應用場景** | 一印件多工單時的出貨前置 | 即便工單 A 已完成、工單 B 仍進行，出貨額度也受限於「印件層級統計後的入庫數」（木桶理論） |
| **邊界案例** | 工單 A 完成 600，工單 B 完成 1000，目標 1000 → 印件入庫 = min(600, 1000) = 600 → 可出貨額度 = 600 | 需等工單 A 再完成 400 後，印件層級入庫才達 1000 |

**實裝建議：** 出貨防呆邏輯應讀取「印件層級 QC 入庫數」（不分工單），此數值由 quantity-calculation-rules.md 的統計規則決定。

#### SHP-003：出貨單作廢時的數量回算

**問題：** 出貨單 A（500 個）已建立並交付物流，後來需要作廢（如運送異常、客戶取消）。該印件的「已出貨數量」應如何調整？

**決策：** 出貨單作廢時，該出貨單涉及的數量**加回**印件的「已出貨」額度

| 層面 | 決策內容 | 說明 |
|------|--------|------|
| **回算邏輯** | 作廢出貨單 × 涉及印件列表 → 該出貨明細的數量加回 | 解放該部分出貨額度，允許重新出貨或業務調整 |
| **可出貨額度重算** | 印件累計 QC 入庫 - (已出貨 - 作廢數量) = 新可出貨額度 | 實時更新系統額度 |
| **狀態轉換** | 出貨單 → 已作廢（新狀態） | 防止誤操作；作廢後不可恢復（改為新建出貨單） |
| **稽核記錄** | 作廢者、作廢時間、原因 | 保留於出貨單活動日誌 |

**邊界情境：**
- 若出貨單已部分送達（多個明細，某些已送達、某些仍在途），如何作廢？
  → **決策：** 若涉及已送達的明細，作廢權限限制（需主管確認）；可選擇「全數作廢」或「部分明細作廢」

#### SHP-004：跨工單合併出貨的統計條件

**問題：** 一筆出貨單可包含來自不同工單的多個印件（如印件 X 來自工單 A、印件 Y 來自工單 B）。這種「跨工單合併出貨」在邏輯上應如何統計？

**決策：** 出貨維度是**以印件層級**來獨立追蹤（不在工單層級做跨工單統計）

| 層面 | 決策內容 | 說明 |
|------|--------|------|
| **出貨單結構** | 出貨單 ⊃ 出貨明細 ⊃ {印件 + 數量} | 一筆出貨單可包含 N 個印件的明細 |
| **防呆規則** | **各印件的數量限制獨立檢查** | 印件 X：本次出貨 ≤ X 的入庫 - X 的已出貨<br/>印件 Y：本次出貨 ≤ Y 的入庫 - Y 的已出貨 |
| **累計統計** | **不在出貨單層級做跨印件累計** | 印件層級的出貨歷史獨立累計；出貨單只是物流單據 |
| **狀態轉換** | 出貨單的狀態獨立於所含印件 | 出貨單可能「已出貨」，但所含的多個印件可能狀態不同（一個出貨中、一個已送達） |
| **何時統計** | 印件狀態判定時才跨出貨單統計 | 判斷「印件已送達」時，累計該印件所有出貨單的送達數量 |

**範例：**
```
出貨單 A（2026-03-02）：
  - 印件 X（工單 A）× 500
  - 印件 Y（工單 B）× 300

出貨單 B（2026-03-05）：
  - 印件 X（工單 A）× 500
  - 印件 Z（工單 C）× 200

檢查：
  - 印件 X 本次（出貨單 A）能出 500？→ 檢查 X 的入庫 - X 的已出貨 ≥ 500
  - 印件 Y 本次能出 300？→ 檢查 Y 的入庫 - Y 的已出貨 ≥ 300
  - 印件 X 本次（出貨單 B）能再出 500？→ 檢查 X 的入庫 - (500 + 500) ≥ 500

累計判斷（是否全數送達）：
  - 印件 X 已送達？→ 累計 (出貨單 A 送達 + 出貨單 B 送達) ≥ 印件 X 目標？
```
