# 出貨部門實際職能與流程整合作業說明（工廠端現況）

## 原始素材

工廠端提供的出貨部門現場作業現況。原檔見 [[出貨部門實際職能與流程整合作業說明.docx]]。重點摘要：

**四大職能**：收貨（中國與台灣到貨件接收、到貨通知與系統更新）／打包（封箱前點收、放出貨明細與入庫確認）／出貨（系統拋單、物流單列印與實際出貨）／維護（物流狀態、現場動線與異常追蹤）。

**收貨流程**：到貨接收（集運／順豐／台灣件）→ 現場支付物流費與關稅並系統留底 → 發布商品照片與到貨資訊 → **到貨工單與貨品交付品檢部** → 出貨部門切「到貨狀態」、品檢部收到工單再切「品檢狀態」。文件自註：與 ERP 中台狀態邏輯可能衝突，需評估拆分到貨／品檢狀態。

**打包流程**：手工完成的貨物，工單先放入未封箱的箱內（大量訂單堆疊暫放不立即封箱）→ 打包人員依工單核對品項與數量 → 放入出貨明細 → 封箱 → 依品項切「已入庫」→ 訂單全數入庫後將工單交付出貨人員。

**出貨流程**：出貨人員逐筆系統拋單、列印物流單 → 通知出貨手張貼物流單 → 出貨手與打包手共同上貨 → 即時更新狀態。

**文件內疑問點**：打包手應該驗什麼——品項是否齊、出貨明細是否放入、外箱是否正確（箱規／標示／特殊需求），**不應該驗數量**。

## 第一輪初步分析

- 現場交接機制＝紙本工單跟著貨走（放箱內、交付品檢部、交付出貨人員）——轉交單要數位化的鏈路。
- 「收貨」「打包」在 ERP 現無對應概念；品檢／入庫／出貨分別對應 QC PT／入庫量／出貨單。
- 候選升級路徑：出貨單狀態卡（等候多時的訪談素材）、出貨人員角色卡充實、轉交流程設計輸入。
