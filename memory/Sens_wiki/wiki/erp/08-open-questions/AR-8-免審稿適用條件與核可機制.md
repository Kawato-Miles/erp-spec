# AR-8 免審稿適用條件與核可機制

## 問題

US-AR-002 描述業務在需求單建立印件時可勾選免審稿快速路徑，但 spec 與 user story 均未明示：

1. **適用條件**：哪些印件類型可走免審稿？是否有業務規則限制（如：特殊工藝印件 / 高難易度印件不可免審）？
2. **核可機制**：免審稿是否由業務單方決定？或須業務主管核可？高額訂單是否需上層核可？
3. **責任歸屬**：免審稿印件若後續出問題（印刷錯誤 / 客訴），責任在業務還是審稿部門？

spec § 免審決策樹.md 描述了免審稿的下游邏輯（建 source=免審稿 Round / 不進入分配），但未涵蓋「適用條件」與「核可機制」前端規則。

## 影響

- 影響 US-AR-002 acceptance criteria（是否補「免審稿適用條件 + 核可流程」條件）
- 影響業務 vs 業務主管的權限設計（spec L88-115 § 審稿主管覆寫分配是審稿主管職權，但業務面免審稿是否有對應主管覆核機制？）
- 影響 ActivityLog 設計（是否要記錄免審稿核可事件 / 責任人）

## 選項

| 選項 | 規則 | 風險 |
|------|------|------|
| A | 業務單方決定，無核可機制 | 業務濫用免審稿快速通道，品質不穩 |
| B | 業務勾選 + 業務主管核可（高於 N 元的訂單必核可）| 流程多一道，但責任明確 |
| C | 系統定規則（如：特殊工藝印件禁免審），業務勾選不違規即可 | 規則需明確列出 |
| D | 業務單方決定，但 ActivityLog 留責任稽核軌跡（事後追責不事前阻擋）| 平衡彈性與責任 |

## 暫定處理

- US-AR-002 acceptance criteria 引此 OQ wiki link，業務流程 step 2 「業務判斷是否屬於免審稿情境」加註「核可機制待 [[AR-8-免審稿適用條件與核可機制|AR-8]] 解答後補入」
- 暫不影響系統行為描述
