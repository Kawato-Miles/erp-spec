# SHP-008：揀貨裝箱時是否驗數量（現場實況待確認）

## 問題描述

揀貨階段確立後（2026-07-08 拍板：依出貨單打包裝箱、回報裝箱內容），現場實況待與工廠確認：揀貨人員（現場原稱打包手）裝箱時**是否清點數量**、清點結果與出貨單數量不符時怎麼處理。此為 [[2026-06-15-生產階段現況校正與as-is-tobe對齊]] 既列的待訪談項（原題「打包手是否驗數量」），需出貨／揀貨現場訪談確認。

## 待解答

- [x] 揀貨裝箱時是否逐箱清點數量？→ 要清點（2026-07-08）
- [x] 現場是否已有裝箱清單／出貨明細之類的習慣可沿用？→ 有，現況 Slack 回報格式（2026-07-08）
- [ ] 點收數量與出貨單數量不符時的處理 → 併入 [[SHP-007-揀貨裝箱回報載體與出貨單狀態顆粒度|SHP-007]]（揀貨錯誤更正流程）

## 決議（2026-07-08，Miles）

**揀貨要清點數量**，回報內容包含：清點數量、裝箱數（幾箱）、箱規（用什麼裝）、怎麼裝的說明，並附照片。現況已有成熟的 Slack 回報慣例可沿用為系統欄位雛形（Miles 提供截圖）：工單／案名／款式／**到貨數量／出貨數量／箱數／箱規**＋成品照、秤重照、裝箱照。注意：現況此回報由品檢完成時一併發出（舊「打包即入庫」實態）；新流程下品檢回報（通過數）與揀貨回報（裝箱內容）拆為兩段，欄位怎麼分家屬 [[SHP-007-揀貨裝箱回報載體與出貨單狀態顆粒度|SHP-007]] 的載體設計。

**落地去處**：[[揀貨人員]] 主要職責補「點收數量」；SHP-007 討論記錄收欄位雛形。
