# PT-017：三主檔「製作天數」的商業用途與交期推算口徑

## 背景

2026-07-23 現況同步發現後端 API（MaterialSpecDetail／ProcessDetail／Binding）與前端 develop 分支已加入「製作天數」（production_days）欄位，落在材料規格、工序、裝訂三主檔上（設備沒有）；main 分支尚未合併，故未寫入 wiki 實體卡欄位表。

## 問題

1. 製作天數的商業用途：是報價時的交期承諾推算輸入？還是排程的工期估算輸入？兩者口徑不同。
2. 一張工單跨多道工序、多筆材料時，製作天數如何合成整體交期：逐段相加？取最大？含平行工序嗎？
3. 為什麼設備層不需要製作天數（設備已有產能速率與換線時間），三主檔的天數與設備產能參數在交期推算中如何分工？
4. main 合併後需回填 [[材料主檔]]、[[工序主檔]]、[[裝訂主檔]] 三卡欄位表。

## 解答（2026-07-23，Miles）

製作天數是 **EC 前台顯示用的交期參考資訊**：取保守值、不考慮訂購數量。與後續工單的有限產能排程無關，排程要另外設計。後端落點為材料規格／工序／裝訂三主檔（後端 migration 0018 已從材料種類層移到材料規格層）。

**誰接住**：欄位與用途說明已回填 [[材料主檔]]、[[工序主檔]]、[[裝訂主檔]] 三卡欄位表；排程設計為 MES 規劃的獨立議題（見 memory `project_mes_architecture_planning`）。

## 觸發解決時機建議

- 已解決。原建議：前端 main 分支合併此功能時；交期推算或排程設計討論前。
