# SHP-005：分批出貨觸發節點

## 問題描述

[[出貨人員]]角色高利害關切：**分批出貨觸發節點設計尚未確認**。

當印件未完全完工但部分入庫時，何時可觸發「分批出貨」？由誰判斷？

- 業務主動申請（依客戶需求）？
- 系統依入庫數量門檻自動建議？
- 出貨人員依倉儲狀況決定？

## 涉及範圍

- 模組：order-management（出貨單）、work-order（齊套）
- 相關卡：[[出貨單]]、[[出貨單狀態]]、[[出貨人員]]、[[齊套邏輯]]、[[印件生產流程]]
- 影響範圍：
  - 出貨單建立流程
  - 多工單齊套邏輯（SHP-002）
  - 倉儲管理與物流排程

## 討論記錄

Phase B 撰寫出貨單實體與狀態機時識別出此議題（既有編號 SHP-005，沿用）。當前 spec：

- SHP-002 多工單出貨齊套：以印件層級統計入庫數為基準
- SHP-003 出貨單作廢回算：可出貨額度重算

但**何時觸發分批出貨**仍未明確。若設計不符合出貨實際操作習慣，可能被繞過。

**現況實務已確認（2026-07-08，Miles）**：分批出貨由**業務**發動——業務告知**印務**他要先出多少（候選方案 A 的實態確認；外部未知已解，audience 改 internal，剩系統設計）。對照生產階段流程重整拍板後的既有機制，系統鏈輪廓為：業務想先出 → 若可出貨數不足，與印務溝通、印務發起提前品檢（[[工序相依性規則]]）→ 通過數入庫（可出貨）→ 業務建部分數量出貨單（即揀貨指令）→ 揀貨 → 出貨。**怎麼在系統中實現待與 [[SHP-007-揀貨裝箱回報載體與出貨單狀態顆粒度|SHP-007]] 同場討論**。

## 待解答

- [x] 分批出貨觸發節點的角色 → 業務發動、告知印務（2026-07-08 現況確認）
- [x] 系統實現：業務在什麼畫面看到「現在可出多少」、與提前品檢的銜接操作 → 見決議
- [x] 客戶端的分批出貨溝通與確認流程 → 業務線下行為，不入系統（見決議）
- [x] 系統是否提供「建議分批出貨」提示 → 不做（見決議）

## 候選方案

### 方案 A：業務 / 客戶主動申請

- 業務依客戶需求申請分批出貨
- 系統檢查入庫足夠後生成出貨單

### 方案 B：系統依入庫門檻自動建議

- 入庫達某比例（如 50%）系統提示「可分批出貨」
- 業務 / 出貨人員決定是否執行

### 方案 C：出貨人員依倉儲狀況彈性決定

- 不固定門檻，出貨人員依現場狀況靈活處理
- 系統允許但不主動建議

## 決議（2026-07-21，Miles）

系統實現拍板（前提：同日品檢模型改為印件層 [[品檢紀錄]]，分次驗、隨驗隨入庫成為常態，「提前品檢」特例機制從正本移除——分批出貨不再需要任何系統特例）：

- **業務端入口＝訂單詳情頁**：每個印件列顯示「可出貨數／已出貨／總量」（可出貨數＝入庫累計−已出貨＋回補，隨品檢即時更新）；**建立出貨單時明細行即時檢核**不得超過可出貨額度。不另開出貨總覽頁（P1 不做）。
- **與品檢的銜接**：業務要先出時，現場（口頭／訊息）請品檢優先驗該印件即可，系統無「提前品檢請求」動作，只看入庫帳。
- **客戶端溝通確認**：業務線下行為，不入系統。
- **「建議分批出貨」提示**：不做，守最精簡。

落地去處：[[出貨單]]（明細額度檢核）、[[齊套邏輯]]（額度即時性）、[[印件生產流程]]（提前品檢特例移除）、[[SHP-007-揀貨裝箱回報載體與出貨單狀態顆粒度|SHP-007]]（併場項同日封存）。
