# PI-004：打樣印件印製維度的生命週期，印件狀態機未建模

## 問題描述

[[印件狀態]] 的印製維度（等待中 → 工單已交付 → 製作中 → 製作完成 → 出貨中 → 已送達）是為**大貨印件**設計的：以出貨單驅動、「已送達」是「所有出貨單皆送達且累計送達數量等於印件數量」的終態、「等待中」是單向初始態無回頭轉換。

但**打樣印件**借用同一條印製維度，其生命週期與大貨印件不同，狀態機未涵蓋兩個關鍵語意：

1. **樣品送樣不是出貨單**：打樣印件做完後是「寄送樣品給客戶評審」，不建出貨單；用印製維度「已送達」承載「樣品送到客戶手上」，是把狀態名套到狀態機未定義的觸發來源。
2. **重打樣（NG-製程問題）是終態回頭**：打樣 NG-製程問題時於同一打樣印件下重做一輪打樣，等於印製維度從一個近終態回到「重新製作」——但印件狀態機的「等待中」是單向初始態、近終態無回頭邊，此轉換在狀態機中不存在。

## 待解答

- [x] 打樣印件的印製維度是否走 [[印件狀態]] 同一條狀態機？→ **是（2026-06-16 Miles 裁決：打樣與大貨共用同一條印製維度狀態機、不另建，打樣印件僅多「打樣結果」欄位記 OK／NG）**。
- [x] 「樣品送達客戶」對應印製維度哪個狀態？→ **已解（2026-07-21 Miles 拍板）：打樣樣品寄送建正式出貨單、走既有分批出貨，印製維度自然走「出貨中 → 已送達」，零特例（觸發來源即出貨單）**。
- [x] 「NG-製程問題重打樣」如何回頭？→ **已解（2026-07-21 Miles 拍板）：新增打樣專屬回頭轉換「已送達 → 等待中」（守衛＝打樣印件且打樣結果填 NG-製程問題，系統建新打樣工單時自動觸發）；打樣結果重置回「待確認」；大貨印件維持無回頭**。

## 結案（answered，2026-07-21）

三子題全數拍板（Miles 2026-07-20 至 07-21）。連帶拍板兩項配套（erp-consultant 輕量審查發現的縫隙）：

1. **舊 NG 樣報廢、累計按週期歸零**：回頭轉換觸發時舊週期樣品報廢（自入庫與送達累計移出、報廢量計入成本），否則第二輪永遠湊不出「累計送達＝印件數量」。正本落 [[齊套邏輯]] 打樣週期歸零特例。
2. **純打樣訂單完成時機**：訂單完成的「全部印件已送達」對打樣印件加守衛「且打樣結果＝OK」——樣品送達後客戶評審中訂單停在出貨段，避免評審前誤完成、NG 回頭破終態。正本落 [[訂單狀態]] 轉換條件。
