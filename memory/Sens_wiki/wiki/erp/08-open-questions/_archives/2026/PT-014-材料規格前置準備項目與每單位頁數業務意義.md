# PT-014：材料規格「前置準備項目」與「每單位頁數」的業務意義

## 背景

2026-07-23 現況同步發現兩個材料計價的新輸入：

1. **前置準備項目**（prep_entries，後端已備、前端 develop 已有、main 未併）：材料規格下的子表，每筆記名稱、寬、高、尺寸單位、裁切數（cutting_count）、適用設備（多選）。看起來是開料／裁切設定，且是材料規格直接關聯設備的新結構。
2. **每單位頁數**（pages_per_unit，main 已併，PR #101）：重量計價尺寸表新增欄位，整數且至少為 1。

wiki [[材料主檔]] 的設備約束欄位（最小／最大長短邊）只做可用性判斷；前置準備項目若也承載設備匹配，兩者的分工需釐清。

## 問題

1. 前置準備項目的業務語意：是「這筆材料上機前要先裁成什麼尺寸、能裁幾張、哪些設備能裁」的開料設定嗎？與 [[數量換算規則]] 的拼模換算是什麼關係？
2. 前置準備項目的「適用設備」與材料規格既有的設備尺寸約束欄位（最小／最大長短邊、起始倍數、最小面積）如何分工？會不會取代後者？
3. 每單位頁數在重量計價中的用途：令重換算？頁數計價的橋接欄位？計算公式歸 [[BOM結構]]，需補公式。
4. main 合併前置準備項目後需回填 [[材料主檔]] 卡（新增子表段）。

## 解答（2026-07-23，Miles ＋ 後端 model 查證）

1. prep_entries 正式名稱為**備料規格**（後端 MaterialSpecPrepEntry：備料名稱、備料尺寸寬／長、尺寸單位、開料數、適用設備 M2M）——即開料設定：這筆材料備成什麼尺寸、能開幾料、哪些設備能用。
2. 與既有設備尺寸約束的分工：材料設定「可以使用的設備」（備料規格的適用設備）；尺寸本身由獨立的**尺寸模組**控制（尺寸範本：一維／二維／三維、單位、出血設定，供商品與工單引用），兩者並存不取代。
3. pages_per_unit 正式名稱為**每台頁數**（後端 db_comment，預設 8），是重量計價尺寸表的台數換算輸入；材料三種成本計算方式（重量計價含重量計／令價計／單張計三子法、面積計價含單獨面積／總面積、數量計價）的後端結構：重量走尺寸表列（規格名／寬長／克重／磅價（元/t）／每台頁數）、面積走面積區間×數量區間價格矩陣、數量走級距單價表。
4. 公式細節下次修訂 [[BOM結構]] 時補。

**誰接住**：備料規格子表、每台頁數、面積子類型名稱（單獨面積／總面積）已回填 [[材料主檔]]；計算公式歸 [[BOM結構]] 下次修訂。

## 觸發解決時機建議

- 已解決。原建議：與維護材料主檔的印務／管理員確認欄位用途時；BOM 計價公式下次修訂時一併補。
