# ORD-015：訂單詳情頁印件清單 120px 縮圖 row 高度副作用驗證

## 問題描述

refine-order-detail-tabs change 將訂單詳情頁印件清單的縮圖欄從 40px × 40px 放大至 **120px × 120px 方形**，並移至表格首欄。

放大後的副作用：
- 每 row 高度 ≥ 128px（含 padding）
- 5 筆印件主表高度 ≈ 640px+（佔 viewport 大部分）
- 跨多訂單對照場景：訂單管理人 / 業務主管在多個訂單詳情頁間切換比對時，垂直 scroll 頻率上升

問題：120px 是否為合適尺寸？是否需要降為折衷尺寸或提供 view 切換？

## 涉及範圍

- 模組：order-management
- 相關卡：[[印件]] / [[訂單]]
- 影響範圍：訂單詳情頁印件清單 Tab 的視覺密度與互動成本
- 對應 spec：openspec/specs/order-management/spec.md § ADDED「訂單詳情頁印件清單表格結構」

## 討論記錄

refine-order-detail-tabs change 規劃階段：
- senior-pm 前期介入提出此風險（120px 對訂單管理人「跨訂單掃描」場景變慢）
- Miles 決議：Pass 1 採 120px、先做先看實際體感
- 驗證計畫：prototype 上線後請 2-3 位業務試用 1 週、依反饋決定是否調整

## 待解答

- [ ] 120px 在 5+ 印件大訂單下的實際體感是否可接受？
- [ ] 訂單管理人跨多訂單對照時的垂直 scroll 是否造成明顯干擾？
- [ ] 若需調整，應採折衷尺寸（80px）還是「精簡 / 詳細」view 切換？

## 候選方案

### 方案 A：維持 120px（當前實作）
- 優點：印件縮圖清晰可辨識細節、減少業務頻繁開檔案
- 缺點：跨訂單掃描體感變慢、5+ 印件大訂單主表高度過長

### 方案 B：降為 80px 折衷
- 優點：row 高度降至 ~88px、5 筆印件約 440px
- 缺點：縮圖細節辨識度下降、可能仍需點開檔案確認

### 方案 C：主表 40px + hover 放大至 200px
- 優點：保留高密度掃描 + 需要時看細節
- 缺點：hover 互動增加、行動裝置無法用（但 ERP 不規劃行動裝置）

### 方案 D：「精簡 / 詳細」view 切換
- 優點：使用者依場景自選（精簡掃描多筆 / 詳細看單筆）
- 缺點：UI 複雜度提升、需追加 view state + toolbar
