# PT-001：師傅報工是否可行動裝置例外

## 問題描述

CLAUDE.md 既定方針：**ERP 系統僅支援電腦版（桌機瀏覽器），不規劃行動裝置版**。

但 [[師傅]] 報工是 **Phase 2 北極星指標（訂單流程完整完成率）達成的關鍵環節**，且現場師傅習慣紙本 / 口頭報工：

- 桌機報工：師傅需離開機台、走到電腦前操作 → 高摩擦
- 行動裝置報工：師傅在機台旁手機 / 平板報工 → 低摩擦

**是否該為師傅報工開放行動裝置例外？**

## 涉及範圍

- 模組：production-task、cross-module（影響全 ERP 行動策略）
- 相關卡：[[師傅]]、[[生產任務]]、[[成功指標]]、[[印件生產流程]]
- 影響範圍：
  - Phase 2 北極星指標達成率
  - 全 ERP 行動策略一致性
  - 開發成本（額外行動裝置 UI / 響應式設計）

## 討論記錄

Phase A 撰寫 03-roles/師傅.md 時識別出此議題。師傅卡 § 預期阻力明確標示：

> 電腦報工的操作習慣改變，老師傅尤其抗拒；事後補輸或完全不輸的風險是產能可見度 95% 目標的最大障礙

若不解此議題，Phase 2 北極星指標難達成。

## 待解答

- [ ] 是否例外開放行動裝置（手機 / 平板）給師傅報工
- [ ] 若開放，範圍多大（僅報工 / 完整工廠平台）
- [ ] 若不開放，替代方案為何（如機台旁設固定平板 / 共用電腦）

## 候選方案

### 方案 A：例外開放行動裝置報工

- 行動：開發師傅報工的響應式 UI（手機 / 平板）
- 優點：降低摩擦、提高報工率
- 缺點：違背「ERP 桌機限定」方針、增加開發成本

### 方案 B：固定平板報工站

- 機台旁設置固定平板 / 共用電腦，師傅就近報工
- 優點：硬體成本可控、不違背桌機方針
- 缺點：仍需走到固定點

### 方案 C：紙本過渡 + 後台補輸

- 師傅紙本報工，由生管 / 訂單管理人後台輸入
- 優點：師傅完全不接觸系統
- 缺點：延遲報工、資料不即時、阻力轉嫁

## 決議（2026-07-21，Miles）

採方案 A 並擴大為**現場回報通道策略**（2026-07-21 生產階段商業現況釐清拍板）：

- **師傅、品檢人員、揀貨人員、出貨人員**四個現場角色使用 **ERP 行動版介面**——範圍寫死於「現場回報」四個介面（報工、品檢記錄、揀貨裝箱回報、出貨／送達確認），不是整套 ERP 行動化。
- **廠務**的轉交回報走 **Slack 表單通道**（webhook 寫回系統），須守四前提：正本在系統（照片回存、Slack 非留痕正本）、通知帶單號預填（禁自由填單號）、保留系統端補登路徑、Slack 帳號與系統使用者對映。
- 原「ERP 僅支援桌機、不規劃行動版」平台限制**修訂**：辦公角色（業務、印務、生管、會計等）維持桌機；現場回報介面例外開放行動版。
