# 問題描述

實體卡憲章化反向拆 [[需求單]] 業務可見欄位時，發現規格欄位修改規則多處未明定、且與雛形系統有結構落差。實體卡欄位表受影響各格已標注引用本 OQ。

## 修改規則未明定

案名、客戶、印件類型、詢問來源、接單公司、發票開立方式、客戶期望交期、報價截止日、流失補充說明、備註——spec 只標必填／選填，未寫可否修改與鎖定時機（對照：收款條件備註與評估印務主管有明確鎖定規則）。

## spec 與雛形系統落差

- 帳務公司欄位：已由 2026-06-12 拍板解決（接單公司即帳務公司、同一概念、業務建單時選擇、廢推導，見 [[需求單]] 欄位表與 wiki/log.md 該筆），**spec Data Model 的「推導唯讀」敘述待同步**——本項只剩 spec 回寫
- 金額三欄（未稅／稅額／含稅總額）：spec 放需求單主表，雛形放印件項目層——層級不一致
- 評估印務主管：spec 標必填，但流程允許未指派即推進——必填性矛盾
- 流失原因選項：雛形有具體七值清單（價格因素／交期無法配合等），spec 只標必填未列舉
- 雛形印件項目有 spec Data Model 未列的業務欄位：難易度（1-10）、是否免審稿、印件檔案備註、已下單數量（散落其他 change 引入，Data Model 未回收）
