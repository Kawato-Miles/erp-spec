# ORD-027：訂單付款狀態是否需新增「部分付款」態

## 問題描述

訂單付款狀態（`Order.payment_status`）enum 目前為四值：**未付款 / 已付款 / 部分退款 / 已退款**，缺少「部分付款」態。

當「已收（已完成收款累計）> 0 但 < 應收（訂單應收總額）」時（B2B 先收訂金、分期未收齊的常態），訂單層付款狀態該如何顯示？是否需新增「部分付款」態？四態之間的完整轉換條件、以及「未付款可否直接跳退款」的流向亦未在 spec 定義。

此問題使「付款狀態」無法在不捏造的前提下畫成完整狀態機 UML（本次 Linear 狀態機補充時暴露）。

## 涉及範圍

- 模組：訂單管理（order-management）/ 款項（Billing & Cash）
- 相關卡：[[付款發票邏輯]]
- 影響範圍：
  - 資料模型：`Order.payment_status` enum 變更（新增值）
  - 狀態機：payment_status 完整轉換 / 推導規則
  - 介面：訂單卡片 / 列表的付款狀態標籤（PaymentStatusBadge）
  - 職責劃分：與請款期次（BillingInstallment）收款維度（未收 / 部分收款 / 已收訖）的資訊重疊
  - 邊界：「同時存在部分收款與退款紀錄」時的顯示優先序

## 討論記錄

2026-06-01 對話中，為「中台 - 訂單管理」project 補狀態機 UML 時，發現 spec 僅定義 payment_status 的 enum 與「依已完成收款累計推導」（order-management spec L2681），未定義「部分退款 / 已退款」判定與「部分收款」呈現。Miles 拍板部分條件並指示此題另開 change，先記 OQ。

## 已拍板條件（Miles 2026-06-01，供後續 change 採用）

- **已付款**：已收 ≥ 應收
- **部分退款**：只要有任一已完成退款紀錄即為「部分退款」（不看退款金額比例、不因後續補收回到已付款）
- **已退款**：已退（已完成退款累計）= 已收，且為終態
- **未確認（待 change 解）**：
  - 部分收款（0 < 已收 < 應收）顯示什麼 —— 取決於是否新增「部分付款」態
  - 「未付款」可否直接跳「部分退款 / 已退款」（與上連動）

## 待解答

- [ ] 是否新增「部分付款」enum 值（0 < 已收 < 應收 時顯示）
- [ ] payment_status 是「即時依收 / 退累計重算的衍生顯示值」還是「事件驅動單向轉換」
- [ ] 「同時有部分收款與退款紀錄」的顯示優先序
- [ ] payment_status 與請款期次收款維度的職責劃分（避免兩處表達同一資訊）

## 候選方案

### 方案 A：新增「部分付款」態（Claude 傾向）
- 優點：對齊 B2B 印刷先收訂金 / 分期收款實務；部分收款不再誤顯示為「未付款」，業務 / 會計不被誤導
- 缺點：enum + 狀態機 + 標籤 UI 變更；需與期次收款維度劃清職責；需定義與退款並存的優先序

### 方案 B：不新增，部分收款停留「未付款」、細節看請款期次
- 優點：訂單層 payment_status 維持粗摘要，避免與期次收款維度重複（DRY）
- 缺點：部分收款顯示「未付款」與實務直覺不符，可能造成業務 / 會計誤判

## 處置

payment_status 狀態機 UML 暫不畫進 Linear「中台 - 訂單管理」project，待此 OQ 經獨立 OpenSpec change 解決後再補。本次 Linear 狀態機補充只涵蓋其餘 8 個 spec 定義完整的狀態機。
