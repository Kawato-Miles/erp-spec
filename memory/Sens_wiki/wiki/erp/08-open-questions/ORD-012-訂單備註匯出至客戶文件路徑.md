# ORD-012：訂單備註匯出至客戶文件的路徑（下游 epic）

## 背景

ceo-reviewer 三視角審查 round 1 指出：新增的三個訂單備註欄位（`order_note` / `delivery_note` / `payment_note`）若只進 DB 不外露給客戶，「降低備註寫不清楚的爭議」這個 ROI **無法兌現**。

Miles 拍板回應：本 change 不做 customer-facing 出口，但承諾**後續報價單／訂單確認單匯出功能會帶出這三個欄位內容**。Phase 1 業務可手動將備註內容貼到 LINE／email 給客戶。

## 問題

後續報價單／訂單確認單匯出 epic 如何引用這三個新欄位？

候選做法：

1. **三類分區呈現**：客戶文件中分「訂單條件 / 交貨條件 / 付款條件」三段顯示
2. **合併單區呈現**：客戶文件中以「補充備註」單區呈現，內部用分隔線區分三類
3. **依文件類型差異呈現**：報價單只帶 paymentNote、訂單確認單帶全部三類、出貨單帶 deliveryNote
4. **客戶可選顯示**：客戶可在 portal 中選擇查看哪些備註類別

## 影響範圍

- 後續匯出 epic 的 spec 範疇
- Invoice / 出貨單 / 報價單 PDF 樣板設計
- 客戶體驗：客戶看到的備註結構是否清楚

## 待釐清

- 下游匯出 epic 的時程（與本 change 的銜接）
- 是否需要客戶端確認簽收備註的機制（reduce dispute）
- 三類分區 vs 合併顯示，客戶閱讀偏好
