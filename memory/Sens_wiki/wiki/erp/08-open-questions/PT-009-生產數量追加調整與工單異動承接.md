# 問題描述

實務上確實會發生**生產數量追加或調整**（客戶加量、損耗補做）。工單與生產任務的數量欄位（每份印件生產數量、目標數量、每份工單需生產數量）若直接放開修改，已報工、已品檢的完成度與入庫帳會跟著浮動；若一律鎖死，又承接不了真實的追加需求。

關鍵問題：**既有的工單異動流程（印務發起 → 生管確認）能不能承接數量追加／調整？**若能，數量欄位本身鎖定、變更一律走異動流程留痕；若不能，要釐清缺什麼（如異動後完成度重算規則、已入庫量如何對待）。

# 待解答

- [ ] 盤點既有工單異動流程的涵蓋範圍：數量變更是否在「需重新審核／不需重新審核」分類內？
- [ ] 數量調整後完成度與入庫數量的重算規則（對照 [[齊套邏輯]] 與 [[數量換算規則]]）
- [ ] 拍板：數量欄位鎖定 + 變更走異動，或開放直接修改 + 留痕

# 候選方案

- 方案 A（傾向）：數量欄位於工單交付後鎖定，追加調整一律走工單異動流程（沿用既有印務發起、生管確認分權），異動確認後觸發完成度重算
- 方案 B：開放直接修改但寫入活動紀錄，完成度即時重算（彈性高、但帳面浮動無審核關卡）
