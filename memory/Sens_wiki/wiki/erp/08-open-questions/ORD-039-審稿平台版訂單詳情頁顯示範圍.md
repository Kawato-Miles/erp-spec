# ORD-039：審稿平台版訂單詳情頁顯示範圍（Tab 閹割 / 動作可見性）

## 問題描述

規劃「審稿模組按訂單檢視待審」新子模組時，訂單節點（母列）可點進訂單詳情，但限訂單層級（不下沉工單 / 生產任務）。然而「**審稿人員在審稿平台點進訂單詳情時，該顯示哪些 Tab、哪些唯讀或隱藏**」目前未定義。

審稿人員為製前角色，理論上不需看到金額 / 發票 / 對帳 / 訂單異動等業務與會計資訊，但確切的閹割清單未定。本 OQ 與 [[ORD-028-業務平台訂單詳情頁Tab閹割範圍|ORD-028]]（業務平台版）平行：平台不同、角色動機相反（業務是訂單主場傾向不閹割；審稿是製前角色傾向窄）。

中台（審稿主管）查看訂單詳情沿用既有完整版，不在本 OQ 範圍。

## 涉及範圍

- 模組：審稿平台（reviewer）/ 訂單管理（order-management）
- 相關卡：[[審稿人員]]、[[訂單]]
- 影響範圍：審稿平台版訂單詳情頁前端 Tab + 動作可見性 gating；與 ORD-028（業務平台版）平行追蹤

## 討論記錄

2026-06-09 規劃審稿模組「按訂單檢視待審」子模組（審稿人員與審稿主管都要、訂單→印件兩層、可視範圍放寬為審稿人員可看整張訂單全貌、訂單節點可點進詳情限訂單層級）時識別此缺口。

## 待解答

- [ ] 審稿平台版訂單詳情頁顯示哪些 Tab（資訊 / 印件清單 / 審稿紀錄 / 活動紀錄 等）
- [ ] 金額及付款 / 發票 / 對帳 / 訂單異動 / 帳務等業務與會計資訊是否一律對審稿人員隱藏
- [ ] 是否比照印件詳情頁業務版的閹割模式（僅資訊 / 審稿紀錄 / 活動紀錄三 Tab）

## 候選方案

### 方案 A：比照印件詳情頁業務版閹割
僅顯示「資訊 / 印件清單 / 審稿紀錄 / 活動紀錄」，隱藏所有金額 / 帳務 / 異動 / 工單 / QC / 出貨。
- 優點：審稿人員製前角色不涉業務與會計，閹割徹底；與印件詳情頁業務版邏輯一致
- 缺點：須確認審稿是否偶爾需要某些訂單資訊（如交期、特殊備註）

### 方案 B：另立審稿平台專屬可見集
- 優點：精準匹配審稿職責所需
- 缺點：又一套可見性規則要維護

## 處置

本次規劃先標訂單詳情入口「限訂單層級」，具體 Tab 閹割清單待此 OQ 於 propose 階段解決；與 ORD-028 並行追蹤。

## 決議與理由

**決議**：審稿平台版訂單詳情頁採精簡閹割——僅顯示「資訊（限訂單編號 / 客戶 / 交期 / 案名，隱藏金額組成 / 付款狀態）」「印件清單（含審稿狀態）」「活動紀錄」三個頁籤；隱藏金額 / 發票 / 對帳 / 訂單異動 / 帳務 / 工單 / QC / 出貨。

**與顧問建議的差異**：顧問建議的四頁籤含「審稿紀錄」頁籤，但訂單詳情頁目前並無「審稿紀錄」頁籤，**本次先不新增該頁籤**（審稿紀錄維持在印件層查看），故落地為三頁籤。

**理由**：審稿角色為製前角色，金額 / 對帳 / 異動屬會計與業務職權、超出審稿職責邊界（角色權責依據）。訂單詳情入口的目的是讓審稿員理解稿件所屬訂單的脈絡（客戶、交期、同訂單其他印件），不需營收與帳務資訊。
