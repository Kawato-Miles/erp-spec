# AR-14：審稿關卡逾期訂單數的「距交期安全緩衝」閾值

## 問題描述

「審稿訂單脈絡查詢視圖」的指標②「審稿關卡逾期訂單數」採「距客戶交期 < 安全緩衝 且 仍在待審母集合」的訂單數口徑（序列協作 Phase 3 顧問建議、Phase 4 採納，改用距交期口徑以避免依賴指標①停留天數口徑）。此口徑用既有 Order.客戶交期、零口徑依賴。但「安全緩衝」的具體閾值（距交期幾個工作天即列入逾期警示）尚未定值。

## 涉及範圍

- 模組：審稿（prepress-review）查詢視圖指標
- 相關卡：[[訂單]]（客戶交期欄位）
- 影響範圍：審稿訂單脈絡查詢視圖的逾期訂單計數與警示門檻

## 討論記錄

2026-06-09 序列協作 Phase 4：指標①「訂單審稿停留天數」採牆鐘時長口徑（Miles 拍板）；指標②逾期定義改用「距交期口徑」避免依賴指標①。閾值待 CEO / Miles 從營運現場定值。

## 待解答

- [ ] 距客戶交期幾個工作天內、且仍在待審母集合，即列入「逾期訂單」警示？
- [ ] 此閾值是否依印件難易度 / 訂單類型分級，還是全公司單一值？

## 候選方案

### 方案 A：單一固定值（如距交期 ≤ 2 工作天）
- 優點：簡單、易理解、prototype 即可實作
- 缺點：未反映難件需更長審稿前置期

### 方案 B：依難易度分級
- 優點：貼合實務（難件預留更長前置期）
- 缺點：需先有「難易度 → 前置期」對照，複雜度高
