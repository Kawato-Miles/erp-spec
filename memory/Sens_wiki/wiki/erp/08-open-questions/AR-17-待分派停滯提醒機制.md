# AR-17 待分派停滯提醒機制

## 問題

線下單印件長時間停在「待分派」（訂單回簽後訂單管理人遲未分派、或稿件已上傳但無人分派）時，**是否需要逾期提醒？閾值多久？提醒誰？**

已確定的部分：稿件上傳而尚未分派時，系統即時通知訂單管理人（[[審稿分配規則]] 已定）。本 OQ 問的是「通知後仍未處理」的停滯升級機制。

## 影響

- 待分派是新增狀態，若無停滯監控，回簽後的印件可能安靜地卡住、逼近交期才被發現
- 與 [[AR-14-審稿關卡逾期訂單距交期緩衝閾值]] 相關：AR-14 談審稿關卡整體逾期閾值，本卡的待分派停滯屬其上游環節，裁決時可一併定義

## 選項

| 選項 | 做法 |
|------|------|
| A | 不另設提醒，靠訂單管理人的訂單進度追蹤日常巡檢 |
| B | 停滯 N 天（或距交期 N 天）升級提醒訂單管理人與業務 |
| C | 併入 AR-14 的審稿關卡逾期機制一起設計 |
