# ORD-011：備註模板含變數佔位符（[日期] [金額]）的處理（Phase 2 議題）

## 背景

附件 28 條 seed 模板中 4 條含變數佔位符：

- 急件處理：「最晚需要於 [日期] 過審稿，可於 [日期] 完成寄出」
- 回簽截止日：「[日期] 下午17:00前回簽報價單並提供檔案」
- 付款截止日：「[日期] 下午17:00前付款全額」
- 指定日期付款全額：「此專案款項：[金額]元（含稅）。請於 [日期] 前完成匯款」

Phase 1 業務插入後手動 search-replace `[日期]` / `[金額]`。

## 問題

未來如何處理變數佔位符？

候選做法：

1. **手動 search-replace（Phase 1 做法）**：業務插入後自己改文字
2. **插入時 Toast 提示**：偵測 `[xxx]` pattern，Toast「請替換 N 個佔位符」（軟性提醒）
3. **插入時跳變數填入 Dialog**：偵測到含 `[日期]` 的模板，跳變數填入表單，業務填好再插
4. **textarea 內 inline highlight**：插入後 `[xxx]` 用顏色 / 底線標出，hover 顯示「待填入」
5. **整合既有訂單欄位**：自動帶入 expected_delivery_date / payment_due_date 等既有訂單欄位（如果有對應）

## 影響範圍

- NoteTemplatePopover 元件複雜度
- UX 流程：是否中斷使用者操作
- 與訂單既有日期欄位的整合機會（自動帶入）

## 待釐清

- 變數佔位符使用頻率（28 條中只 4 條含變數，4/28 = 14%）
- 業務手動 search-replace 的錯誤率（漏改 vs 改對）
- 自動帶入 vs 手動填入的取捨（自動可能帶錯日期）
