# 問題描述

實體卡憲章化反向拆 [[工單]]／[[生產任務]]／[[任務]] 業務可見欄位時，發現生產規格兩份 spec 的欄位修改規則多處未明定、且有內部矛盾與 spec / 雛形系統不一致。實體卡欄位表「可否修改」欄受影響各格已標注引用本 OQ。

## 部分拍板（2026-06-12，Miles）

- **工廠類別不可改**：綁在 BOM 上——工序／材料決定負責工廠（內部或外部），工單只能依 BOM 內容建生產任務。已落地 [[生產任務]] 欄位表（唯讀）。
- **狀態枚舉與色彩模型統一**：以涵蓋較完整的版本為準修改另一側；逐項對照於 spec 同步時執行。
- **數量欄位**：實務確有追加、調整數量需求，是否由既有工單異動流程承接拆出 [[PT-009-生產數量追加調整與工單異動承接|PT-008]] 獨立追蹤。

## 仍待解（修改規則未明定）

- 工單：工單類型（建立草稿後可否改）、負責人改派時機
- 任務：目標數量與預計執行日的填寫者與鎖定時機

## spec 內部矛盾（待 spec 回寫時清理）

- 工廠類別：已拍板唯讀（見部分拍板），spec 殘留的「可改」敘述待清
- 任務類型與範疇兩欄的正本位置待整理

## spec 與雛形系統不一致

- 生產任務狀態枚舉：spec 8 值，雛形多「已交付／已取消／異動」3 值
- 色彩模型：spec 用「單黑／CMYK 開關＋倍率」，雛形用「正面色數／背面色數」數字模型
- 任務層狀態：spec 單一狀態欄，雛形拆「交付狀態＋異動狀態」兩欄再衍生
- 雛形有、spec 無的業務欄位：工單的預計交期與主管欄；任務與生產任務的編號與名稱欄
- 合批群組功能尚未定案（spec 自註待裁決）
