# 問題描述

實體卡憲章化反向拆 [[工單]]／[[生產任務]]／[[任務]] 業務可見欄位時，發現生產規格兩份 spec 的欄位修改規則多處未明定、且有內部矛盾與 spec / 雛形系統不一致。實體卡欄位表「可否修改」欄受影響各格已標注引用本 OQ。

## 修改規則未明定（spec 只標型別，沒寫誰能改、何時鎖）

- 工單：工單類型（建立草稿時選定，之後可否改未明定）、負責人改派時機、每份印件生產數量與目標數量的鎖定時機
- 生產任務：目標數量、每份工單需生產數量的鎖定時機
- 任務：目標數量與預計執行日的填寫者與鎖定時機

## spec 內部矛盾

- 工廠類別可否修改：生產任務 Data Model 表標可改，但工單與生產任務條文皆寫「工廠由工序自動決定（唯讀）」
- 任務類型與範疇兩欄只在補充表出現，主 Data Model 表未列

## spec 與雛形系統不一致

- 生產任務狀態枚舉：spec 8 值，雛形多「已交付／已取消／異動」3 值
- 色彩模型：spec 用「單黑／CMYK 開關＋倍率」，雛形用「正面色數／背面色數」數字模型
- 任務層狀態：spec 單一狀態欄，雛形拆「交付狀態＋異動狀態」兩欄再衍生
- 雛形有、spec 無的業務欄位：工單的預計交期與主管欄；任務與生產任務的編號與名稱欄
- 合批群組功能尚未定案（spec 自註待裁決）
