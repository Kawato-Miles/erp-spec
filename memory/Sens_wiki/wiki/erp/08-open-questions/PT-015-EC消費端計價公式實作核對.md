# PT-015：EC 消費端計價公式實作核對

## 背景

2026-07-23 全面梳理後端 sens-print-core 確認：後端只承載計價設定與 options 發布（主檔 → S3 JSON → 消費端），本身沒有成本計算引擎；實際售價計算在消費端平台（EC 前台，程式不在已盤點的兩個 repo）。[[BOM結構]] 的材料計價公式依資料模型參數與紙業慣例整理（重量計噸價公式、令價計令重磅數換算 0.000711、單張計直取），與模型一致但未經消費端程式碼直接對證。

## 問題

1. EC 消費端對重量計／令價計的實際計算式是否與 [[BOM結構]] 記載一致（含 0.000711 常數、一令 500 張）？
2. 「每台頁數」（預設 8）在消費端公式中怎麼用：書冊頁數 → 台數 → 用紙張數的換算鏈是否如推定？
3. 數位印刷 399mm 小圖／大圖分界、平版每色價 × 色數、大圖才數折扣乘數的套用順序，消費端實作為何？
4. 後端 weight_entry 價格欄的 db 註解寫「磅價（元/t）」，單位標注與令價計（元/磅）／重量計（元/t）的雙語意混用，建議後端修註解。

## 觸發解決時機建議

- 取得 EC 前台（options JSON 消費端）程式碼或其計價規格時
- 工單成本計算引擎設計前（沿用主檔計價邏輯的實作依據）
