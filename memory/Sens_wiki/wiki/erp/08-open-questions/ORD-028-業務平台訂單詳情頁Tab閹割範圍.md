# ORD-028：業務平台訂單詳情頁 Tab 閹割 / 動作可見性範圍

## 問題描述

sales-platform spec 明確定義了「**印件詳情頁**」業務 / 諮詢版的 Tab 閹割（僅顯示資訊 / 審稿紀錄 / 活動紀錄三個 Tab、隱藏工單 / QC / 轉交 / 出貨），但**未定義「訂單詳情頁」業務平台版的 Tab 閹割 / 動作可見性範圍**。

業務 / 諮詢於業務平台查看自己負責的訂單詳情時，哪些 Tab（金額及付款狀態 / 發票 / 對帳檢視 / 訂單異動 / 附件 等）或動作（開立發票 / 折讓 / 帳務公司 / 對帳 CSV 匯出 等中台 / 會計專屬功能）應隱藏或唯讀？

## 涉及範圍

- 模組：業務平台（sales-platform 容器）/ 訂單管理（order-management）
- 相關卡：—
- 影響範圍：業務平台訂單詳情頁前端（FE-259 範疇）+ 後端動作權限 gating（BE-168）

## 討論記錄

2026-06-01 以 linear-delivery skill 交付業務平台 - 訂單管理時，評審 sub-agent 識別此缺口。對照印件詳情頁閹割（業務不介入生產層）的動機，訂單詳情頁的閹割動機可能不同。

## 待解答

- [ ] 業務平台訂單詳情頁是否需要 Tab 閹割（業務是訂單負責人、訂單詳情為其主場，可能不閹割或僅隱少數動作）
- [ ] 若閹割，明確哪些 Tab / 動作對業務 / 諮詢隱藏或唯讀（特別是帳務 / 發票 / 對帳這類會計 / 中台專屬功能）
- [ ] 諮詢與業務於訂單詳情頁的可見性是否一致

## 候選方案

### 方案 A：不閹割訂單詳情，僅隱藏中台 / 會計專屬動作
- 優點：業務管自己訂單需完整資訊；符合「訂單詳情是業務主場」
- 缺點：須逐一界定「中台 / 會計專屬動作」清單

### 方案 B：比照印件詳情頁，閹割帳務相關 Tab
- 優點：與印件詳情頁閹割模式一致
- 缺點：業務處理訂單可能需要看金額 / 發票狀態，過度閹割反而卡作業

## 處置

本批（業務平台訂單管理交付）先沿用中台訂單詳情 + 依角色隱藏中台專屬動作；明確 Tab 閹割清單待此 OQ 解決後補。Linear 交付文件以「另案處理」標示，不捏造閹割清單。
