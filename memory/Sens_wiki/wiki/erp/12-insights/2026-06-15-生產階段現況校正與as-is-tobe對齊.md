# 生產階段現況校正與 as-is／to-be 對齊（第二輪結論，供第三輪接續）

> 本卡是生產階段系統設計第二輪（釐清與校正現況）的固化結論，供第三輪（出貨模組 G2 細部設計）接續。設計大原則：以業界典範最佳實踐為骨幹、依公司現況微調、不臆測、不過度設計、以可拋棄的最小可行性設計但把完整 Roadmap 處理到「可進 SDD」（見 [[痛點]] 問題 6 與 memory feedback）。

### 八項關鍵差異

| 面向 | 現況 as-is | 導入後 to-be | 對高層的意義 |
|------|-----------|-------------|-------------|
| 派工載體 | 印務印紙本工單、交廠務實體派送；異動寫紙本不回寫系統 | 印件可製作時系統自動產生草稿工單、分派單一印務；生管以工作包（跨工單、一次上機）系統派工 | 消除掉單與照舊版規格生產，異動全程留痕 |
| 報工方式 | 產線／師傅在 LINE 派單串文字＋照片，只記報工數量一個數字；外包無回報 | 師傅／供應商自助介面報工（良品／不良品／工時／設備／停機／耗料）；首筆報工自動向上推進 | 報工事實成為折損率／良率／成本的唯一底稿 |
| 狀態追溯與擱置 | 擱置靠「印務有沒有在群組回覆」隱性判定；寄倉／分批缺狀態 | 顯式狀態機＋向上反映鏈，狀態與責任人綁定 | 進度追得到、責任綁定可建績效基礎 |
| 跨平台同步（最大人力成本） | 同筆訂單在 RAGIC／官網／中台多平台重複維護 | 單一 ERP 正本、四層單據狀態自動向上推進 | 直接砍現場明指的最大人力成本 |
| 成本歸集 | 無實際成本；運費箱數事後人工回填運費總表 | 計價快照（預計）vs 報工回填（實際）比對成本差異 | 從無估準度到可量化成本達成率與毛利率（P2） |
| QC 流轉與完成度 | 品檢拿紙本工單、群組回報通過／不通過 | QC 併生產任務（type=qc），通過數驅動入庫；缺口補做走異動 | 品檢通過數成為入庫與可出貨上限唯一依據 |
| 出貨驗收與放行 | 出貨等前端口頭／Slack 通知；超商單手寫工單號 | 出貨額度齊套管控、出貨單狀態機系統管理、送達後自動收尾 | 放行從人工通知變系統化額度與狀態守衛 |
| 齊套與數量換算 | 多工序完成數、放損、拼模靠人腦與紙本 | 系統以最慢工序算齊套、三層換算無條件捨去、倍數可覆寫 | 避免遺漏或超做、可出貨量精確 |

廠務說明：現況與導入後皆保留廠務「場內生產任務轉交」泳道（依 [[廠務]] 職責 2：上一段做完把印件送往下一站、一路到品檢）；to-be 廠務依 [[轉交單]] 物理搬運，單據由印務操作。

查核剔除兩個假缺口（示範不捏造）：印刷成本歸屬（M2 已裁決歸工序、[[工序主檔]] 已補）、工序計價第八種「工序總價」（[[工序主檔]] 已補入）。確認為真的缺口：

| 缺口 | 嚴重度 | 預計解法（Raw 權威調研） | 待補 |
|------|-------|-----------------------|------|
| 成本／實績地基未就位（[[成本差異]]／[[在製品]]／[[報工紀錄]] draft） | 高 | ISA-95 SegmentResponse 統一承載＋Dynamics365 良品/不良品/原因同動作＋Odoo 損失三分類 KPI 衍生 | 5 項現場量測決策（報廢分類／停機分類／設備可用時數／效率值／折損去向）P2 啟用前 |

## 四、中國線校正：派單／貨運單／生產任務三實體分離（Miles 2026-06-15 裁決）

依既有 dev dashboard 派單／貨運單實況（raw [[2026-06-15-claude-self-capture-中國派單與貨運單平台現況]]），中國線非紙本留言、已系統化。新 ERP/MES 採三實體分離、不合併：

- [[派單]]（外發委外單）：獨立實體，綁印件／工單、指派外部供應商，持 [[派單狀態]]（大陸處理狀態 14 值：未送大陸→已發稿→打樣製作／寄送／到貨／重打樣／合格→大貨製作整批或分批→大陸補印中→已送集運商→運回台灣中→補印貨回台灣→海外直發／交付），記外部廠商當地進程。場內與加工廠不需派單；廠商類型路由（外包廠／中國廠才產生派單）。台灣外包派單概念相同、尚未實作。
- [[貨運單]]（中國→台灣跨境物流）：獨立實體，記承運商／運費（RMB＋NT$）／收貨點收／台灣實秤與重量差異／運費關稅按重量佔比分攤回 [[生產任務]]／關稅單據／認列狀態。成本回生產任務，與台灣對客戶 [[出貨單]] 分離。
- [[生產任務]]：內部單位，外發時外部詳細進程歸派單；[[生產任務狀態]] 保留外發在途粗狀態（已送集運商／運送中），由印務依派單狀況手動同步回工單、回場後人工推進。兩實體不重疊——派單記外發委外單的詳細 14 值，生產任務記工單視角的粗進度。

連帶：派單與貨運單為待新增實體；EC 訂單日後整合進 ERP，以工單屬性（台灣／中國）路由。

|---------|----------------------|------------|

| 派工／排程分層 | 兩級排程＋瓶頸有限產能（Dynamics／Odoo／PrintVis） | P1 設備佇列（天）、P3 有限產能；工作包＝一次上機 |
| 報工／回報 | SegmentResponse 統一承載＋良品/不良品/原因同動作＋損失三分類（ISA-95／Dynamics／Odoo） | 報工紀錄缺口補欄位 |
| QC／品檢 | 品質控制點＋四型檢查＋批次放行（Odoo／MES 分批） | QC 併生產任務、通過數驅動入庫；不通過原因見 [[QC-003-QC不通過原因固定選項\|QC-003]] |
| 在製品 WIP | 帳務科目累積 或 中間地點顯式（Dynamics／Odoo） | 在製品＝投入減產出 |
| 成本／折損／KPI | 估價-訂單-實際三層比對＋OEE computed（PrintVis／EFI／Odoo） | 計價快照 vs 報工回填；KPI 衍生不另存；P2/P3 |
| BOM／設備計價 | 資源以能力＋月曆＋效率%＋瓶頸旗標建模（Dynamics／Odoo） | 三主檔；印刷計價歸工序（M2）、設備只記開機＋折舊 |
| 分批出貨／部分驗收 | 先打包入庫後建單、部分出貨標配（MES 分批） | 沿用齊套額度、不引入拆批單據 |
| 拼版／併單 | 承印物與版面內容分離＋自動拼版前移報價（PrintVis／Avanti） | P1 模數比例過渡、P3 補成品尺寸＋多成品拼版 |
| 中國外派／跨境收貨 | 八份調研無直接對應（業界找不到） | 沿用既有派單／貨運單實作、收斂進 ERP |

## 六、待第三輪解的設計問題

出貨模組 G2 動工前三決策：(1) 出貨單狀態顆粒度（EC 未處理／打包中／待出貨／已出貨 vs wiki 未處理→運送中→已送達／異常，建議融合）；(2) 台灣外包派單是否與 G2 一起設計通用派單（建議一次設計、分階段實作）；(3) 台灣線出貨運費是否進出貨單欄位（建議進，解運費總表人工回填）。

外發銜接：中國／台灣外包回場後生產任務的報工與向上反映如何與派單／貨運單銜接，見 [[PT-008-外發回廠流轉的系統設計範圍|PT-008]]。

待確認問題分兩類：業界找不到（中國跨境收貨關稅、OEE 統一三因子公式、小批量異形併單規模、派案平台 EC→ERP 轉換）；公司特殊微調（M1 BOM 呈現、M3/M4 非紙 BOM、M5 EC vs 真實工單、5 項現場量測、打包手驗數量、分批觸發節點、QC-003 選項）。
