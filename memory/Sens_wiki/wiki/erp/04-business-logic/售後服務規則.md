# 售後服務規則

> 本卡承載訂單已完成後售後服務單的五步流程、決議分支與特殊規則的完整定義。[[訂單異動規則]]（來源售後服務單標記、訂單完成後金額異動路徑分流）引用本卡，不重述。屬款項與收款領域。

## 這張卡要回答的問題

- 訂單完成後客戶反映問題，從受理到結案怎麼走？
- 售後服務單本身要不要主管核可？
- 退款、補印免費、補印收費、不處理四種決議各走什麼路？
- 一張訂單能同時有幾張未結案的售後服務單？

## 營運背景

印刷廠出貨後客戶可能反映瑕疵、規格不符、物流問題等。這些售後事件需要一個追蹤容器，記錄客訴內容、討論結果、處理方式，避免事情斷在某個環節沒人收尾。公司的做法是：售後服務單本身只是容器與追蹤器，真正的金額動作（退款、補收）沿用既有的異動單流程，生產動作（補印）沿用既有的審稿與工單流程——不為售後另開新流程。

## 規則與營運動機

### 五步售後流程

| 項目 | 內容 |
|------|------|
| 意圖 | 從客訴受理到結案有完整的追蹤流程，每一步都有明確的負責人與動作 |
| 規則 | （一）受理：業務建立售後服務單，填客訴內容、事件分類（印件瑕疵／規格不符／物流問題／工法限制／交期延誤／其他）、責任歸屬（公司認賠／客戶承擔／共同分擔），狀態為受理中。（二）討論（系統外）：業務在公司通訊軟體與業務主管討論處理方式，討論完把討論串連結貼回售後單。（三）決議：業務填入處理方式（不處理／退款／補印／退款＋補印）並送出，狀態推進為處理中。（四）執行下游動作：依處理方式分流（見下方各決議規則）。（五）結案：業務確認客戶滿意或所有下游動作完成後手動點結案，狀態推進為已結案 |
| 營運動機 | 五步結構讓每張售後服務單從頭到尾都有跡可查。討論放在系統外（公司通訊軟體）是因為印刷業實務中主管判斷常需看照片和即時溝通，系統只記結果 |

### 無核可關卡

| 項目 | 內容 |
|------|------|
| 意圖 | 售後服務單本身不設核可流程，把關擺在下游 |
| 規則 | 售後服務單本身無核可關卡。業務與主管在公司通訊軟體線下討論決議，系統僅記錄結果。真正需要核可的是：售後單內的退款異動單仍走業務主管審核（既有異動單狀態機不變，見 [[訂單異動規則#退款送審]]）|
| 營運動機 | 售後事件的判斷（該不該退、退多少）靠主管線下拍板，不需在系統上再跑一次核可流程空轉。金額把關已在異動單層級做了 |

### 一訂單一未結案售後單

| 項目 | 內容 |
|------|------|
| 意圖 | 避免同一訂單同時有多個進行中的售後事件互相干擾 |
| 規則 | 一個訂單最多一張未結案的售後服務單。結案後可建新的售後單 |
| 營運動機 | 多張未結案售後單會讓退款金額、補印印件難以追蹤，逼著業務處理完一件再開下一件 |

### 決議分流：不處理

| 項目 | 內容 |
|------|------|
| 規則 | 業務選「不處理」→ 售後單不建任何下游動作（無異動單、無款項、無印件）→ 業務直接結案。訂單應收、發票、收款均不變動 |

### 決議分流：退款

| 項目 | 內容 |
|------|------|
| 規則 | 業務在售後單內建關聯退款異動單（金額為負、帶售後來源標記）→ 異動單走業務主管審核（見 [[訂單異動規則#退款送審]]）→ 業務在發票區建退款款項 ＋ 視需要開折讓單 → 業務確認客戶滿意後結案 |

### 決議分流：補印免費

| 項目 | 內容 |
|------|------|
| 規則 | 業務在售後單內建補印印件 → 印件走原審稿與工單流程 → 不建異動單（免費補印）。補印印件出貨完成不自動結案售後單（需業務確認客戶滿意後手動結案）|

### 決議分流：補印收費

| 項目 | 內容 |
|------|------|
| 規則 | 業務在售後單內建關聯補收異動單（金額為正）＋ 建補印印件。異動單走補收流程（見 [[訂單異動規則#補收免審]]），印件走原審稿與工單流程 |

### 決議分流：退款＋補印

| 項目 | 內容 |
|------|------|
| 規則 | 同時走退款與補印路徑（上述兩條）|

### 訂單狀態不回退

| 項目 | 內容 |
|------|------|
| 規則 | 售後處理期間訂單狀態永遠保持「已完成」不回退。售後處理中的狀態以介面徽章呈現（非訂單主狀態）|
| 營運動機 | 訂單完成是終態，售後是完成後的善後，不應把已結算的訂單拉回未完成狀態 |

## 具體例子

例子 1（退款場景，正例）：訂單 #ORD-2026-0470 已完成後客戶反映 5000 元瑕疵。業務建售後單（印件瑕疵、公司認賠）→ 與主管在公司通訊軟體討論決議退款 → 售後單內建退款異動單 −5000 元（帶售後來源標記）→ 主管核可 → 業務建退款款項 −5000 元 ＋ 開折讓單 → 確認客戶滿意後結案。

例子 2（補印免費場景，正例）：訂單 #ORD-2026-0490 已完成後客戶反映規格不符要求補印 100 份、公司認賠。業務建售後單（規格不符、公司認賠、補印）→ 售後單內建補印印件 → 印件走審稿、工單、出貨 → 出貨完成但售後單不自動結案 → 業務確認客戶滿意後手動結案。全程不建異動單，應收不變。

例子 3（不處理場景，正例）：訂單 #ORD-2026-0505 已完成後客戶反映輕微色差但可接受。業務建售後單（印件瑕疵、公司認賠、不處理）→ 不建任何下游動作 → 直接結案。對帳不受影響。

## 一定要成立的規則

1. 售後服務單內的退款異動單走與訂單期間相同的退款審核規則（業務主管核可，見 [[訂單異動規則#退款送審]]）。
2. 一訂單最多一張未結案售後服務單。
3. 訂單狀態保持「已完成」不回退。
4. 補印印件出貨完成不自動結案售後單（需業務手動確認客戶滿意）。
