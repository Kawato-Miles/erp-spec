# 報價印件填寫原則

> 本卡承載報價單建立印件時的拆分與合併原則完整定義。[[印件生產流程]]、[[齊套邏輯]] 等卡引用本卡，不重述。屬訂單與交付領域。

## 營運背景

印刷廠報價階段，業務要把客戶要的東西拆成印件。拆法直接決定後面工單怎麼分、出貨怎麼出、帳怎麼算。拆錯了會導致：同批商品被分開出貨（客戶收不齊）、或是各別出貨的東西擠在同一個印件（出貨時拆不開）。

## 規則與營運動機

### 同批出貨合併、獨立出貨拆分

| 項目 | 內容 |
|------|------|
| 意圖 | 報價階段就把出貨維度想清楚，避免到出貨階段才發現拆錯 |
| 規則 | 同批商品一起出貨 → 合併為一個印件。各別獨立出貨 → 拆為多個印件。打樣與大貨 → 分開建立（各自走獨立審稿與工單流程）|
| 營運動機 | 印件是出貨的最小管理單位，合併或拆分直接影響後續齊套性計算（見 [[齊套邏輯]]）與出貨明細的防呆邏輯 |

### 一式含多工單的數量計算

| 項目 | 內容 |
|------|------|
| 意圖 | 讓「一式」這種報價單位正確對應到工單層的生產數量 |
| 規則 | 報價為「1 式」含多種組件（如 1 式 ＝ 1000 個專輯 ＋ 300 本歌詞本）時，印件數量為 1，拆成工單 A（專輯，每份印件生產數量 ＝ 1000）與工單 B（歌詞本，每份印件生產數量 ＝ 300），印件完成數依齊套性邏輯計算 |
| 營運動機 | 專輯和歌詞本的製作工序不同（分開工單），但出貨要一起交（同一印件），拆法正確才能讓齊套性邏輯算出正確的印件完成數 |

## 具體例子

例子 1（同批合併，正例）：客戶要 5000 張 DM 和 5000 張折價券，同時出貨。業務報價時合併為一個印件，拆成兩張工單（DM 工單 + 折價券工單），出貨時兩樣東西一起出。

例子 2（獨立出貨拆分，正例）：客戶要 3000 張名片和 2000 份目錄，名片寄台北倉、目錄寄高雄。業務報價時拆成兩個印件，各自走獨立出貨流程。
