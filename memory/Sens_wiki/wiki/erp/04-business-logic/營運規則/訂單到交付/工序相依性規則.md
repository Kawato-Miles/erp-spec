## 現況

印刷品的工序有先後：下一道吃上一道的產出（印好才能上光、上光才能裁、裁完才能裝訂），中間還可能插著轉交（半成品送到下一站）。生管派工如果不看前置，就會發生「下一站開工了、上一站東西還沒到」的空等或錯做。本卡定義生產任務之間的相依性規則：什麼條件擋住派工、什麼情況可以破例提前。品檢不是工序間的前置——工序之間不設品檢節點，品檢收斂為印件入庫前的最終品檢（正本見 [[品檢紀錄]]）。

## 規則與營運動機

### 工序相依（前置有到料才可開工，數量級滾動）

| 項目 | 內容 |
|------|------|
| 規則 | 每個生產任務可設前置任務清單；預設依印務規劃時的排序自動帶「上一道是前置」的線性鏈，並行匯流情境（如封面與內頁各自做完才能裝訂）由印務手動設定多個前置。前置判定為**數量級**：需轉交的前置看「已送達量」、不需轉交的前置看「已報工量」，**前置到料量大於 0 即可派工開工**、到多少做多少（師傅介面看「已到料／目標量」滾動累積）；多個前置時可做量＝各前置到料量取最小。前置到料量為 0 時阻擋派工並提示哪個前置未到料 |
| 營運動機 | 工序常跨天分批完成（做 1000 個、每天 200），做完的部分先流到下一站接著做，下一站不必空等上一站全部完成——分批流動是常態不是破例（2026-07-20 拍板）。線性自動帶讓印務不必逐筆設；並行匯流交給手動，避免系統猜錯。阻擋擺在生管派工這一刻——派下去就是現場開工，這是最後也最有效的把關點 |

### 跨工單前置（2026-07-22 拍板，MES 設計需求）

| 規則 | 前置任務允許指向**別張工單的生產任務**（典型場景：成套商品的組裝工序，前置為 A 工單與 B 工單的完成量，見 [[印件拆分規範]]）。放行機制沿用數量級：組裝可做量＝各上游工單到料量換算後取最小。MES／ERP 設計時須規劃跨工單前置的設定與連動功能（現況未實作） |
| --- | --- |
| 營運動機 | 業界的跨工單相依即「上游工單產出＝下游工序投料」，靠料到量放行、不靠狀態機串接；既有數量級放行機制天然適用，只需把前置指向範圍從同工單放寬到跨工單。異動連鎖：上游工單異動／棄用時，下游跨工單前置同樣觸發「前置受影響」警示（見 [[PT-007-前置受影響警示通知對象與形式|PT-007]]，範圍含跨工單） |

### 需轉交才可製作（預設需轉交）

| 項目   | 內容                                              |
| ---- | ----------------------------------------------- |
| 規則   | 生產任務**預設「需轉交」**：上一道的產出須轉交到位（轉交單已送達的數量）才計入下一道的到料量。印務在製程規劃時只標例外「**不需轉交**」（同產線連續兩道、沒有實體移動），不需轉交者到料量直接看上一道已報工量。轉交單由系統於廠務確認搬運時自動生成，正本見 [[轉交單]]、[[轉交單狀態]]（2026-07-20 拍板） |
| 營運動機 | 大部分工序做完都要實體移動，逐筆設「需轉交」是重複負擔——預設反轉，只標少數例外。跨站／跨廠的工序，貨沒到場開不了工；把「貨到了沒」做成到料判定，取代現場口頭確認與紙本工單跟貨 |

### 分次驗、隨驗隨入庫（分批出貨的常態支援）

分批出貨不需特例：品檢採分次驗、隨驗隨入庫，做多少驗多少、通過即入庫（可出貨），本來就是常態（正本見 [[品檢紀錄]]）。業務需要先出一批時，由品檢現場優先驗該印件即可，不需任何破例流程（原「提前品檢」破例機制 2026-07-21 廢止）。

保留一項現場彈性（印務發起）：**緊急品檢**（印件大遲到、非上班時間的品檢需求）可先請品檢開驗、事後補上工單並確保單號可對應——「工單＝唯一標準」的事後補單彈性。

### 前置受影響警示

| 項目 | 內容 |
|------|------|
| 規則 | 前置任務延遲、作廢或報廢時，系統對受影響的下游任務標示「前置受影響」並發出警示（通知對象與形式待 [[PT-007-前置受影響警示通知對象與形式|PT-007]] 拍板） |
| 營運動機 | 阻擋只在派工那一刻起作用；上游出事時下游的排程與承諾交期已經受影響，要主動讓人知道、不能等到派工才發現 |

## 具體例子

- 例子 1（並行匯流，數量級）：專輯袖套案，封面與內頁各自印刷加工，裝訂任務的前置設為封面加工＋內頁加工——封面到料 300、內頁到料 200，裝訂可做量＝200；內頁到料 0 時裝訂不能派工。
- 例子 2（分次驗支援臨時分批出貨）：名片 1000 份做到 600，客戶來電急要一批。品檢分次驗、隨驗隨入庫本是常態——品檢人員現場優先驗這 600、通過 580 入庫；業務回覆客戶「可先出 580」並建出貨單。剩 400 做完再驗，累計達標才算整體完成。
- 例子 3（滾動轉交，跨天分批）：貼紙 1000 張，印刷每天做 200、裁切為下一道（預設需轉交）。第一天印刷報工 200 → 廠務依待搬清單送裁切站並確認送達（系統生成轉交單）→ 裁切到料 200、隔天即可開工做這 200；此後每天滾動，印刷第五天全部完成時裁切已做 600，不必空等。

## 邊界

- 前置未完成時排程日期怎麼順延 → 排程初期人工，系統連動屬後續補強，不在本卡
- 做齊多少、能出多少的計算 → [[齊套邏輯]]（規則正本）
- 轉交單自身的生成與狀態 → [[轉交單]]、[[轉交單狀態]]（正本）
- 品檢的驗收記錄與不符合處置 → 屬品檢規則（正本見 [[品檢紀錄]]），本卡不展開。品檢缺口的處置為兩個獨立決策的組合：「缺口補不補」（補做，或不補、由業務下修訂單數量收尾）×「瑕疵品去向」（報廢，或客戶照收），可自由組合；決策由印務與業務討論（必要時業務向客戶確認），「放行」操作與最終責任在印務（工單主責）；順序為業務先下修訂單、印務再處理工廠端資料（2026-07-22 拍板），見 [[印務]]
