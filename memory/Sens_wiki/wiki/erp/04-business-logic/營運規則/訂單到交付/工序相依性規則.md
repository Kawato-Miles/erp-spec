## 現況

印刷品的工序有先後：下一道吃上一道的產出（印好才能上光、上光才能裁、裁完才能裝訂），中間還可能插著轉交（半成品送到下一站）。生管派工如果不看前置，就會發生「下一站開工了、上一站東西還沒到」的空等或錯做。本卡定義生產任務之間的相依性規則：什麼條件擋住派工、什麼情況可以破例提前。品檢不是工序間的前置——工序之間不設品檢節點，品檢收斂為印件入庫前的最終品檢（正本見 [[生產任務狀態]] § 品檢型任務）。

## 規則與營運動機

### 工序相依（前置完成才可派工）

| 項目 | 內容 |
|------|------|
| 規則 | 每個生產任務可設前置任務清單；生管派工前系統檢查全部前置是否完成（多個前置須全部滿足），未完成則阻擋派工並提示哪個前置未完。預設依印務規劃時的排序自動帶「上一道是前置」的線性鏈；並行匯流情境（如封面與內頁各自做完才能裝訂）由印務手動設定多個前置 |
| 營運動機 | 線性是大多數工單的實況，自動帶讓印務不必逐筆設；並行匯流交給手動，避免系統猜錯。阻擋擺在生管派工這一刻——派下去就是現場開工，這是最後也最有效的把關點 |

### 需轉交才可製作

| 項目   | 內容                                              |
| ---- | ----------------------------------------------- |
| 規則   | 生產任務可標記「需轉交」：上一道完成後，半成品須轉交到位（轉交單已送達）才能派下一道      |
| 營運動機 | 跨站／跨廠的工序，貨沒到場開不了工；把「貨到了沒」做成派工前置，取代現場口頭確認與紙本工單跟貨 |

### 提前破例（提前品檢／提前轉交）

| 項目 | 內容 |
|------|------|
| 規則 | 正常流是印件的生產任務全部做完才做最終品檢、做完才轉交。當商業需要提前（最常見：客戶問「現在可以先出多少」的臨時分批出貨），流程為：**業務／諮詢收到商業需求 → 與印務溝通 → 由印務發起提前品檢（最終品檢提前開驗）或提前轉交**；條件＝對應生產任務已有報工量。提前品檢驗已完成的部分、通過數量即計入入庫（可出貨）；生產繼續做、之後再驗後續批，品檢任務累計達標才算整體完成 |
| 營運動機 | 臨時分批吃的是報工狀況——已做多少、驗了多少、能出多少。用「提前破例」而不是把整批拆單：不增加單據複雜度，沿用既有的分批驗收累計與通過數入庫機制；發起權收在印務（生產規劃的主責者），業務不直接動生產 |

四句話看懂：什麼時候用＝客戶要先出一部分／下游等不及；誰發起＝印務（業務溝通後）；條件＝該任務已有報工量；效果＝提前驗的通過數先入庫先出，整體完成判定不變。

另一種破例同屬印務發起：**緊急品檢**（印件大遲到、非上班時間的品檢需求）可先請品檢開驗、事後補上工單並確保單號可對應——「工單＝唯一標準」的事後補單彈性。

### 前置受影響警示

| 項目 | 內容 |
|------|------|
| 規則 | 前置任務延遲、作廢或報廢時，系統對受影響的下游任務標示「前置受影響」並發出警示（通知對象與形式待 [[PT-007-前置受影響警示通知對象與形式|PT-007]] 拍板） |
| 營運動機 | 阻擋只在派工那一刻起作用；上游出事時下游的排程與承諾交期已經受影響，要主動讓人知道、不能等到派工才發現 |

## 具體例子

- 例子 1（並行匯流）：專輯袖套案，封面與內頁各自印刷加工，裝訂任務的前置設為「封面加工完成＋內頁加工完成」——封面做完但內頁還沒做完，裝訂不能派工。
- 例子 2（提前破例，臨時分批）：名片 1000 份做到 600，客戶來電急要一批。業務與印務溝通後，印務發起提前品檢；品檢人員驗 600、通過 580 入庫；業務回覆客戶「可先出 580」並建出貨單。剩 400 做完再驗，品檢任務累計達標才算完成。
- 例子 3（需轉交才可製作）：印刷在 A 廠、裱貼在 B 廠，裱貼任務標記需轉交——印刷完成後轉交單建立、送達 B 廠（轉交單已送達），生管才能派裱貼。

## 邊界

- 前置未完成時排程日期怎麼順延 → 排程初期人工，系統連動屬後續補強，不在本卡
- 做齊多少、能出多少的計算 → [[齊套邏輯]]（規則正本）
- 轉交單自身的建立與狀態 → [[生產任務狀態]] 與轉交單設計（收斂中）
- 品檢的驗收記錄與不符合處置 → 屬品檢規則，本卡只管「提前品檢作為破例」；異常的處置決策者＝印務（與前端討論後決定補印／重做／放行，可決定放行），見 [[印務]]
