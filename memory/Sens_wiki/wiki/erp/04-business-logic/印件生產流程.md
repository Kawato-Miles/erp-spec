# 印件生產流程

> 核心：印件從審稿通過 → 業務確認可製作 → 工單草稿 → 印務 → 生管 → 師傅報工 → QC → 入庫 的端到端流程。

→ 視覺化見 `../../../decks/print-item-production-flow.svg` 與 `../../../decks/print-item-production-swimlane.svg`

## 零、單據層級結構

印刷業 ERP 的生產單據遵循以下層級關係，上層單據建立後才能產生下層單據：

```
需求單 → 訂單（含 N 個印件） → 工單（含 N 個任務） → 任務（紙本工單，含 N 個生產任務）
```

一張訂單可拆成多張工單，一張工單可拆成多個任務，一個任務可含多個生產任務。這個層級決定了數量換算的方向（從生產任務往上逐層匯算至印件，見 [[數量換算規則]]）與齊套性的計算基礎（見 [[齊套邏輯]]）。

## 一、工單分派順序

工單分派 SHALL 依以下順序執行：

```
審稿通過 → 業務確認可製作 → 建工單草稿 → 分派給印務 → 印務填寫內容 → 交付任務 → 生管分派
```

B2C 訂單印件合格後系統自動確認可製作（跳過業務手動步驟）；B2B 線下單由業務手動確認。

## 二、分派限制

- **一張訂單可分派給多位印務**：訂單包含多張工單時，可分別分派
- **一張工單僅對應一位印務**：阻擋同工單派給第二位印務

## 三、工單建立方式（B2C / B2B 分流）

| 訂單來源 | 確認可製作後建工單方式 | 生產任務帶入 |
|---------|---------------------|------------|
| **B2C（線上單）** | 系統自動建工單 | 系統依商品主檔工序定義自動帶入生產任務（工單 + 生產任務一次建齊） |
| **B2B（線下單）** | 系統建空工單草稿 | 印務主管後續手動拆分生產任務 |

依印件類型細分：

| 印件類型 | 生產任務帶入方式 |
|---------|------------------|
| **EC 自動產品** | 系統自動帶入生產任務，無須印務手動建立 |
| **半客製產品** | 參照既有範本帶入，允許調整 |
| **全客製產品** | 印務手動建立生產任務 |

## 四、預計生產數量下限

- 預計生產數量由[[印務]]決定 buffer
- 預計生產數量 **不可小於** 印件所需數量
- 含 buffer 範例：印件所需 1000 → 印務設定 1100（10% buffer）→ 允許

## 五、入庫與出貨數量規則

- **入庫數量僅在 QC 通過後才計入**（QC 未通過不計入庫）
- **出貨數量不得超過入庫數量**
- 範例：印件入庫 500，使用者嘗試出貨 600 → 系統阻擋，提示「出貨數量超過入庫數量」

## 六、出貨單作廢回算

出貨單作廢時系統重新計算可出貨額度（計算公式與 Scenario 見各模組 spec：order-management / work-order / shipment）。

## 七、QC 數量限制

```
可 QC 數量 ≤ 報工數量 - 其他 QC 單已申請數量
```

範例：報工 1000、已有 QC 申請 700 → 新 QC 單最多可申請 300。

## 七B、分批出貨累計控制

分批出貨時系統進行累計控制：

- **累計限制（警告非阻擋）**：跨所有出貨單的出貨明細數量總和超過目標生產數量時，系統顯示警告但不阻擋——印刷業實務中，因良率 buffer 實際可出貨數量可能超過原需求數量，阻擋反而會造成麻煩
- **累計送達判斷**：跨所有出貨單的送達數量總和達到或超過目標生產數量時，印件狀態推進為「已送達」
- 範例：印件目標 1000，第一批出貨 600 已送達、第二批出貨 400 已送達，合計送達 1000 → 印件推進為「已送達」
- 範例（警告）：印件目標 1000，已出貨累計 900，新建出貨明細 200（累計將達 1100）→ 系統顯示警告「累計出貨數量超過目標生產數量」，但允許建立
