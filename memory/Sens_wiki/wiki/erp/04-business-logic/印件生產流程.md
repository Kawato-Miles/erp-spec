# 印件生產流程

> 核心：印件從審稿通過 → 業務確認可製作 → 工單草稿 → 印務 → 生管 → 師傅報工 → QC → 入庫 的端到端流程。

→ 視覺化見 `../../../decks/print-item-production-flow.svg` 與 `../../../decks/print-item-production-swimlane.svg`

## 一、工單分派順序

工單分派 SHALL 依以下順序執行：

```
審稿通過 → 業務確認可製作 → 建工單草稿 → 分派給印務 → 印務填寫內容 → 交付任務 → 生管分派
```

B2C 訂單印件合格後系統自動確認可製作（跳過業務手動步驟）；B2B 線下單由業務手動確認。

## 二、分派限制

- **一張訂單可分派給多位印務**：訂單包含多張工單時，可分別分派
- **一張工單僅對應一位印務**：阻擋同工單派給第二位印務

## 三、工單建立方式（B2C / B2B 分流）

| 訂單來源 | 確認可製作後建工單方式 | 生產任務帶入 |
|---------|---------------------|------------|
| **B2C（線上單）** | 系統自動建工單 | 系統依商品主檔工序定義自動帶入生產任務（工單 + 生產任務一次建齊） |
| **B2B（線下單）** | 系統建空工單草稿 | 印務主管後續手動拆分生產任務 |

依印件類型細分：

| 印件類型 | 生產任務帶入方式 |
|---------|------------------|
| **EC 自動產品** | 系統自動帶入生產任務，無須印務手動建立 |
| **半客製產品** | 參照既有範本帶入，允許調整 |
| **全客製產品** | 印務手動建立生產任務 |

## 四、預計生產數量下限

- 預計生產數量由[[印務]]決定 buffer
- 預計生產數量 **不可小於** 印件所需數量
- 含 buffer 範例：印件所需 1000 → 印務設定 1100（10% buffer）→ 允許

## 五、入庫與出貨數量規則

- **入庫數量僅在 QC 通過後才計入**（QC 未通過不計入庫）
- **出貨數量不得超過入庫數量**
- 範例：印件入庫 500，使用者嘗試出貨 600 → 系統阻擋，提示「出貨數量超過入庫數量」

## 六、出貨單作廢回算

出貨單作廢時系統重新計算可出貨額度。計算公式與 Scenario 見 OpenSpec [business-processes/spec.md § 出貨單作廢回算](../../../../openspec/specs/business-processes/spec.md)（實作規格正本）。

## 七、QC 數量限制

```
可 QC 數量 ≤ 報工數量 - 其他 QC 單已申請數量
```

範例：報工 1000、已有 QC 申請 700 → 新 QC 單最多可申請 300。

## 八、相關角色責任分配

| 階段 | 主要角色 |
|------|---------|
| 審稿 → 建工單草稿 | [[印務主管]] |
| 工單分派 | [[印務主管]] → [[印務]] |
| 印務填寫內容 / 建立生產任務 | [[印務]] |
| 任務分派 | [[生管]] |
| 執行生產任務 / 報工 | [[師傅]] / [[中國廠商]] / [[外包廠商]] |
| QC 檢驗 | [[品檢人員]] |
| 入庫管理 / 出貨 | [[出貨]] |
