# 齊套邏輯（Kitting Logic）

> 核心原則：**取最小值作為整體結果**（多工單共同構成印件時，由「最少完成」的工單決定印件完成數）。對應 [[glossary-erp#系統行為術語|min() 聚合邏輯]] 或業務用語「最少工單原則」。

## 一、為什麼需要齊套邏輯

一個印件可能由多個工單共同構成（例：專輯 = 工單 A 專輯內頁 + 工單 B 歌詞本）。當任一工單未完成，整個印件即不可出貨。

→ 視覺化見 `../../../decks/kitting-rule-architecture.svg`

## 二、四層計算精確流程

系統依四層由下而上計算印件完成數，每層結果向上傳遞：

| 層級 | 計算 | 邏輯 |
|------|------|------|
| **層級 1 生產任務層** | `pt_qc_passed = sum(已完成報工紀錄的通過數量)`（QC 生產任務 `type = qc` 或品檢生產任務 `type = inspection` 的報工通過量加總） | 加總該生產任務所有報工紀錄的通過數量（獨立 QCRecord 已廢止，改用生產任務框架） |
| **層級 2 任務層** | `pt_completion_ratio = floor(pt_qc_passed / pt.quantity_per_work_order)` → `task_completion = min(所有 completion_ratio)` | 篩選 `affects_product = TRUE` 的生產任務，取最小值（齊套性邏輯）|
| **層級 3 工單層** | `wo_completion = min(該工單下所有任務的 task_completion)` | 取最小值 |
| **層級 4 印件層** | `wo_completion_ratio = floor(wo_completion / wo.quantity_per_print_item)` → `pi_completion = min(所有工單的 wo_completion_ratio)` | 取最小值，若 ≥ 目標生產數量則狀態推進「製作完成」|

## 三、典型情境

### 情境 1：單一工單多生產任務（層級 2 齊套）

- 任務含生產任務 A：QC 通過 2100，`quantity_per_work_order = 2`，`affects_product = TRUE`
- 任務含生產任務 B：QC 通過 900，`quantity_per_work_order = 1`，`affects_product = TRUE`
- A 的 completion_ratio = floor(2100/2) = **1050**
- B 的 completion_ratio = floor(900/1) = **900**
- `task_completion = min(1050, 900) = 900`（B 為制約因素）

### 情境 2：多工單一印件（層級 4 齊套）

- 印件含工單 A：`quantity_per_print_item = 1000`，`wo_completion = 2100`
- 印件含工單 B：`quantity_per_print_item = 300`，`wo_completion = 700`
- A 的 wo_completion_ratio = floor(2100/1000) = **2**
- B 的 wo_completion_ratio = floor(700/300) = **2**
- `pi_completion = min(2, 2) = 2`

### 情境 3：多工單出貨齊套性檢查（SHP-002）

- 印件含工單 A（入庫 500 份）與工單 B（入庫 300 份）
- 每份印件皆需工單 A 與工單 B 各一份
- 系統以最小入庫數 **300** 為該印件可出貨上限

## 四、邊界防呆

| 邊界情況 | 系統行為 |
|---------|---------|
| 一個任務內所有生產任務的 `affects_product` 皆為 FALSE | 阻擋儲存，提示「至少需有一個生產任務影響成品」 |
| `quantity_per_work_order ≤ 0` | ERROR 阻擋儲存 |
| `quantity_per_work_order > 10000` | WARNING 提醒印務確認 |
| `pi_completion = 0` | UI 顯示「已完成 X / Y（缺口 Z）」 |

| 欄位 | 層級 | 說明 |
|------|------|------|
| `quantity_per_print_item` | WorkOrder | 完成 1 份印件需要多少份此工單 |
| `quantity_per_work_order` | ProductionTask | 完成 1 份工單需要多少份此生產任務 |
