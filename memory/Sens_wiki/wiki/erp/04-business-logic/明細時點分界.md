# 明細時點分界

> 一筆 [[訂單]] 的明細金額（印件單價／下單數量／運費急件等其他費用），「到什麼時候業務還能在訂單裡直接改、過了哪條線就鎖死改走別條路」的分界正本。本卡只講「分界在哪、兩階段各能做什麼」；過了線之後退款／補收各自怎麼走、怎麼認列，正本在 [[訂單異動規則]]，本卡只接到它的入口、不重述規則。屬款項與收款這塊。

## 這張卡要回答的問題

- 一筆訂單還沒完成前，業務想改印件單價、改下單數量、加運費，能不能直接改？要不要主管核可？
- 改價會不會自動退回業務主管重新審核？
- 哪一條線一過，明細就鎖死、不准在訂單裡直接改？
- 鎖死之後還要改金額，要走哪條路（退款？補印？向客戶加收？）？
- 「調降」（少收客戶錢）算不算特別，要不要比照退款送主管核可？

## 營運背景

印刷廠一筆訂單從報價到交貨，金額常一路微調：客戶加印、追加數量、改規格連動售價、報價當初打錯要修、議價砍價（增、減都有），業務跟 [[諮詢]] 當場就會動明細。這些調整若每筆都要開單據、等主管核可，會把當場能談妥的事拖成兩段流程，客戶在等、出貨在等。所以公司的做法是：訂單還在進行中（還沒結案）時，明細金額由業務／諮詢直接改，連「調降」（少收錢）也當場可決定，主管不擋、但事後看得到（弱把關、留痕）；等訂單一旦結案（進入終態），帳已經要拿去對了，明細就鎖死，之後要再動金額得走有跡可循的善後路徑（退款走售後服務單、向客戶加收走新訂單），不再回頭改已結算的原單。

這條分界要切在哪？路 C 拍板切在「訂單完成／已取消」這兩個終態：訂單一進到這兩個狀態的任一個，明細就鎖。切在終態而不是切在「回簽」或「製作開始」這些中間點，是因為製作中、出貨中客戶照樣會加印改量，過早鎖死反而擋掉正常生意（業務訪談 confirm，見 [[ORD-030-完成前明細直接增減真實業務流程待訪談\|ORD-030]]）。

主管讀本卡要能驗證的是：把「明細鎖定」這條線切在訂單結案（終態），結案前全放手讓業務直接改、結案後一律走善後路徑，符不符合公司「給彈性、最後靠對帳兜底」的實際做法。

## 規則與營運動機

> 兩階段以「訂單完成／已取消」雙終態為界。階段一（未達終態）放手直接改；階段二（已達終態）鎖定走善後路徑。各狀態下哪些欄位可改的精確清單在 OpenSpec，本卡只講分界與動機。

### 階段一：訂單未進入終態，明細可直接增減

| 項目 | 內容 |
|------|------|
| 觸發條件 | 訂單狀態還沒進到「訂單完成」也沒進到「已取消」（草稿／待業務主管審核／審核通過／報價待回簽／已回簽／製作中／出貨中等都算） |
| 規則 | [[業務]]／[[諮詢]] 可直接增減明細：印件單價、印件下單數量、其他費用（運費／急件費等，含新增與刪除），增、減都可以（含調降＝少收客戶錢）。改完訂單應收當下重算（印件費＝每個印件下單數量 × 報價單價，逐件加總，再加其他費用），不需業務主管核可、不會把訂單退回主管重新審核。調降屬弱把關：主管事後在活動紀錄看得到、不擋操作；改動寫一筆留痕 |
| 營運動機 | 進行中的訂單客戶常加印改量議價，當場能決定的事不該拖成送審；調降雖是少收錢，但發生情境多是報價打錯／議價，把關擺在「主管事後看得到」這層就夠，不必每筆擋下來。服從營運原則「彈性優先、事後對帳兜底」——給業務當場處理的彈性，最後靠對帳把對不上的挑出來（見 [[對帳一致性]]） |

四句話看懂：什麼時候會用到＝訂單還沒結案；誰能改＝業務／諮詢；要不要主管審＝不用（調降也只是事後看得到）；要守的規則＝改完應收即時重算、留痕。

### 階段二：訂單進入終態，明細鎖定走善後路徑

| 項目 | 內容 |
|------|------|
| 觸發條件 | 訂單狀態進到「訂單完成」或「已取消」其中之一（雙終態任一） |
| 規則 | 明細金額（印件單價／下單數量／其他費用）一律鎖定，不能在訂單裡直接改。之後要再動金額，依方向走善後路徑：（一）要退錢給客戶、或要替客戶補印一批貨 → 開 [[售後服務]] 單（單內放退款負項異動單＋補印印件）；（二）要向客戶加收錢 → 另開一張新訂單收費。兩條路徑的退款／補收規則正本見 [[訂單異動規則#訂單完成後金額異動路徑分流]] |
| 營運動機 | 訂單一結案，帳就要拿去對（應收／發票／收款三方對帳），這時若還能回頭改明細，已對好的帳會被打亂、也查不出是誰何時改的。鎖死明細、改走有單據可循的善後路徑，退錢與補印歸到售後服務單留客訴脈絡，向客戶加收歸到新訂單獨立報價開票，帳就不會被反覆拉開 |

四句話看懂：什麼時候會用到＝訂單已結案（完成或取消）；誰能改＝沒人能在訂單裡直接改；要走哪條路＝退款／補印走售後服務單、加收走新訂單；要守的規則＝鎖定點是雙終態、過線一律走善後路徑。

## 具體例子

例子 1（階段一，製作中加印，正例）：訂單 #ORD-2026-0512 製作中（未達終態），客戶把名片從 500 張加到 800 張。業務直接把該印件下單數量改成 800，訂單應收當場依新數量重算上去，不必開單、不必等主管核可，整批貨照常生產。

例子 2（階段一，議價調降，正例）：訂單 #ORD-2026-0520 審核通過後，客戶議價砍掉 2000 元。業務直接把某印件報價單價往下調，應收當場下降 2000；這筆調降不必退回主管重審，但系統寫一筆留痕、主管事後在活動紀錄看得到。

例子 3（階段一，報價打錯修正，正例）：訂單 #ORD-2026-0525 已回簽，業務發現某印件單價當初少打一個 0。業務直接改正單價，應收重算，不退回主管重審（已回簽仍未達終態，明細未鎖）。

例子 4（階段二，完成後退款，正例）：訂單 #ORD-2026-0470 已是「訂單完成」終態，客戶反映外觀瑕疵要退費並重印。明細已鎖、不能在訂單裡改，業務改開一張售後服務單，單內建退款負項異動單並掛補印印件（規則見 [[訂單異動規則]]）。

例子 5（階段二，完成後加收，正例）：訂單 #ORD-2026-0470 已完成，客戶滿意後當場追加一批新名片。這是向客戶加收的新生意，業務另開一張新訂單獨立報價收費，不在原訂單裡改明細、也不在售後單內建補收異動單。

例子 6（階段二，已取消同屬鎖定終態，邊界正例）：諮詢訂單 #SSP-20260417-CR01 諮詢取消後落在「已取消」終態。已取消與訂單完成同屬鎖定終態，該訂單明細一律唯讀，之後任何金額調整都不在訂單裡直接改。

## 一定要成立的規則

> 系統一定要守住的分界鐵則。

1. 明細（印件單價／下單數量／其他費用）可否在訂單裡直接改，只看一條線：訂單狀態是否已進入「訂單完成／已取消」雙終態——未進入＝可直接改、已進入＝鎖定。
2. 階段一改任何明細金額，訂單應收當下重算（印件費＝每個印件下單數量 × 報價單價逐件加總 ＋ 其他費用）；不退回業務主管重審。
3. 階段二之後的金額異動，退款／補印一律走 [[售後服務]] 單、向客戶加收一律走新訂單，不在原訂單或售後單內直接改明細金額。

## 邊界（不適用 / 範疇外）

- 各狀態下「哪些欄位可改／唯讀」的精確清單與資料模型 → OpenSpec [order-management/spec.md](../../../../openspec/specs/order-management/spec.md)（本卡只講分界與動機，不重述欄位與狀態列舉）。
- 訂單各狀態自身的定義、轉換條件、終態判定 → [[訂單狀態]]（本卡只引用「訂單完成／已取消」為鎖定分界，不重畫狀態圖）。
- 過了分界之後退款免不免審、補收怎麼認列、應收怎麼算、對帳怎麼核銷 → [[訂單異動規則]]（金額異動規則正本）。
- 完成前已收款後減量退款的免送審軟兜底（款項紀錄直接建退款）→ [[訂單異動規則#款項紀錄直接建退款]]（本卡只負責「明細可直接調降」這一段，退款動作的規則在該卡）。
- 純規格調整（不影響金額，如改紙張磅數）走通知＋活動紀錄不建單 → [[訂單異動規則#零金額不建單]]。
