## 概述

發票（InvoiceStatus）從開立到可能作廢的狀態機，發票實體的欄位見 [[帳務]]。台灣的統一發票制度下，發票開出去就是正式文件——不能刪除、不能修改，只能作廢另開，所以狀態只有兩態、設計極簡。發票經電子發票加值服務（藍新）開立；現階段開立即生效、不設草稿，未來若需要草稿流程再增補狀態。

開票時機、發票與收款透過請款期次中介的對應規則，正本在 [[付款發票邏輯]]，本卡只定義狀態與轉換、不複述規則。

## 狀態列舉（正本）

> 本段是發票狀態的唯一正本。狀態的新增與修改是商業決策，直接在此卡維護。

| 狀態 | 說明 | 對應營運需求 |
|------|------|------------|
| 開立 | 初始；發票開立成功、正式生效，可被折讓單引用 | 有效發票計入對帳的發票淨額 |
| 作廢 | 終態；開錯（金額、抬頭、統編）時作廢並留原因，同訂單需要時另開新發票 | 失效發票退出三方對帳，作廢原因留稽核 |

## 狀態機圖（UML）

依 UML 狀態機圖記法繪製：實心圓為初始點、雙圈為終止點。「開立」是持續有效態（多數發票停留於此），只有開錯才走作廢。

```mermaid
stateDiagram-v2
    direction LR

    [*] --> s_issued : 業務／諮詢人員開立發票
    state "開立" as s_issued
    state "作廢" as s_void

    s_issued --> s_void : 業務／諮詢人員作廢 [必填作廢原因]
    s_void --> [*]
```

## 轉換條件與觸發事件

| 轉換 | 觸發事件 | 條件 |
|------|---------|------|
| （建立）→ 開立 | 業務／諮詢人員開立發票 | 開立即生效，不設草稿 |
| 開立 → 作廢 | 業務／諮詢人員執行「作廢」並填入原因 | 作廢後流水號不重用；同訂單重開時新發票流水號順延 |

## 關鍵轉換的營運動機

- 開立 → 作廢 → 動機：發票開錯（金額、抬頭、統編）依法不能直接修改，必須作廢另開，作廢原因留存供稽核 → 例子：ORD-2026-0512 的發票 INV-01 統編打錯，業務作廢 INV-01 並重開 INV-02。
- 流水號不重用 → 動機：每張發票序號唯一可追溯，作廢後重用同號會讓會計帳混亂。
- 作廢退出對帳 → 動機：對帳只加總「開立」狀態的發票，確保發票淨額反映實際有效的發票。

## 與其他狀態機的關係

- 發票與收款的對應透過 [[分期請款狀態|請款期次]] 中介（期次與發票一對一、期次與收款多對多），不直接掛收款紀錄。
- 折讓單引用「開立」狀態的發票進行折讓，見 [[折讓單狀態]]；發票作廢不影響訂單或異動單的狀態。
