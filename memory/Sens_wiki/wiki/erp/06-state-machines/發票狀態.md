# 發票狀態（InvoiceStatus）

> [[發票]] 從開立到可能作廢的狀態機。只有兩態，設計極簡。流水號不重用、作廢後不參與三方對帳。開票時機、發票與收款的對應透過請款期次中介等規則正本在 [[付款發票邏輯]]，本卡只定義狀態與轉換、不複述規則。

## 這張卡要回答的問題

> 每個問題在正文都有對應結論；判斷「卡完整了沒」就看每個問題是否都答到了。

## 狀態列舉（正本）

> 本段是發票狀態的正本定義。狀態的新增與修改是商業決策，直接在此卡維護。轉換條件（Given/When/Then）留在 OpenSpec [order-billing/spec.md](../../../../openspec/specs/order-billing/spec.md) 行為規格中。

| 狀態 | 說明 | 對應營運需求 |
|------|------|------------|
| 開立 | 藍新開立成功；可被折讓單引用；與收款的對應透過請款期次中介 | 發票已正式生效，計入對帳淨額 |
| 作廢 | 業務作廢（含作廢原因）；終態 | 發票失效，不再參與三方對帳 |

> 開票時機、發票與期次 / 收款的關係見 [[付款發票邏輯]]，本卡不重述。

## 狀態轉換

發票（業務手動作廢）：

```
開立 ─業務作廢（填原因）─▶ 作廢
```

## 轉換條件與觸發事件

| 轉換 | 觸發事件 | 條件 |
|------|---------|------|
| 開立 → 作廢 | 業務 / 諮詢人員點「作廢」並填入原因 | 人工；作廢後流水號不重用 |

## 營運背景

發票在台灣有法規要求（統一發票使用辦法），開出去就是正式文件、不能刪除只能作廢。作廢後同訂單若需重開，新發票的流水號順延（如 `O-25050601-INV-01` 作廢後，重開為 `O-25050601-INV-02`），確保每張發票流水號唯一、可追溯。

狀態只設兩態（開立 / 作廢），不設草稿——因為目前模擬階段（Mockup）開立即生效，真實串接藍新 API 若需要草稿流程，屆時另開變更補狀態。

## 關鍵狀態 / 轉換的營運動機

- 開立 → 作廢 → 動機：發票開錯（金額錯、抬頭錯、統編錯）必須作廢重開，不能直接修改已開出的發票（法規要求），作廢後留原因供稽核 → 例子：#ORD-2026-0512 的發票 INV-01 統編打錯，業務作廢 INV-01 並重開 INV-02。
- 流水號不重用 → 動機：確保發票序號唯一可追溯，避免作廢後重用同號碼導致會計帳混亂。
- 作廢不參與三方對帳 → 動機：對帳時只加總「開立」狀態的發票金額，作廢的排除在外，確保對帳數字反映實際有效發票。

## 與其他狀態機 / 實體的關係

- 發票與收款的對應透過 [[分期請款狀態|請款期次]] 中介（期次與發票一對一、期次與收款多對多），不直接掛收款紀錄。
- 折讓單引用有效（開立）的發票進行折讓，見 [[折讓單狀態]]。
- 作廢的發票不影響訂單狀態或異動單狀態。
