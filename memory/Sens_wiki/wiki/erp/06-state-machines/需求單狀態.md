# 需求單狀態（QuoteStatus）

> [[需求單]] 的狀態機。**業務獨立完成，沒有業務主管 gate**。業務主管審核發生在訂單階段。

## 狀態清單

```
需求確認中 → 待評估成本 → 已評估成本 → 議價中 → 成交 / 流失
```

## 狀態說明

| 狀態 | 說明 | 主要角色 |
|------|------|---------|
| 需求確認中 | 業務蒐集客戶需求 | [[業務]] / [[諮詢]] |
| 待評估成本 | 業務送出，等待印務主管評估 | 等待 [[印務主管]] |
| 已評估成本 | 印務主管完成評估，業務可報價 | [[業務]] |
| 議價中 | 業務與客戶談價 | [[業務]] |
| 成交 | 進入訂單階段 → 生成 [[訂單]] | — |
| 流失 | 客戶不接受報價 | — |

## 主要轉移觸發

| From → To | 觸發 |
|-----------|------|
| 需求確認中 → 待評估成本 | 業務提交需求確認 |
| 待評估成本 → 已評估成本 | 印務主管完成成本評估 |
| 已評估成本 → 議價中 | 業務點擊「進入議價」|
| 議價中 → 成交 | 客戶接受報價 |
| 議價中 → 流失 | 業務判斷流失 |
| 已評估成本 → 待評估成本 | 業務申請重新評估 |

## 自動推進

**無**。所有轉移由業務或印務主管手動觸發。

## 與訂單狀態的連結

成交 → 自動建立 [[訂單]]，初始狀態依訂單類型：
- 線下：報價待回簽
- 諮詢：建立
- 線上（從 EC）：等待付款

詳見 [[訂單狀態]]。
