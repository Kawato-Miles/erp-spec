# 諮詢單狀態（ConsultationRequestStatus）

> [[諮詢單]] 的狀態機。surveycake webhook 自動建單，**雙出口**：轉需求單 / 諮詢訂單。

## 狀態清單

> v2 簡化（resolve-consultation-request-gaps-cr-1-cr-2，2026-05-22）：移除「諮詢中」過渡狀態，諮詢人員於「待諮詢」狀態下直接選擇分支動作。狀態 enum 與 consultation-request spec 一致：`待諮詢 / 已轉需求單 / 完成諮詢 / 已取消`。

```
待諮詢 → 已轉需求單（轉做大貨）
待諮詢 → 完成諮詢（不做大貨；或諮詢來源需求單後續流失自動觸發）
待諮詢 → 已取消（待諮詢階段取消預約退費）
```

## 狀態說明

| 狀態 | 說明 |
|------|------|
| 待諮詢 | webhook 自動建立後，等待諮詢人員自我認領（認領後 status 維持「待諮詢」，僅標示已分派）|
| 已轉需求單 | 諮詢人員選擇「轉做大貨」→ 建 [[需求單]]；Payment 維持綁諮詢單、待需求單結局明確時才轉移 |
| 完成諮詢 | 諮詢人員選擇「完成不做大貨」→ 建諮詢[[訂單]]（含諮詢費 2000，訂單完成終態）；或諮詢來源需求單後續流失自動觸發 |
| 已取消 | 待諮詢階段取消預約 → 建諮詢訂單（已取消終態）+ 半額退 1000（收 2000 退 1000）+ 自動建退款 OA（已核可、approved_by=system）+ 退款 Payment（處理中）；`cancel_reason_category` 六選一必填；不自動開發票 / 折讓 / 待開發票 |

## 主要轉移觸發

| From → To | 觸發 |
|-----------|------|
| - → 待諮詢 | surveycake webhook 自動建單 |
| 待諮詢 → 已轉需求單 | 諮詢人員點「轉需求單」，建 QuoteRequest（MUST NOT 建訂單）|
| 待諮詢 → 完成諮詢 | 諮詢人員點「完成不做大貨」，建諮詢訂單（訂單完成終態）|
| 待諮詢 → 已取消 | 諮詢人員 / 業務主管取消預約，選定 cancel_reason_category → 建諮詢訂單（已取消終態）+ 半額退費 |

## 自動推進

**無**。但有自動建單觸發（surveycake webhook → 待諮詢）。

## 關聯狀態機

- → [[需求單狀態]]（轉需求單時）
- → [[訂單狀態]]（諮詢訂單時，order_type = 諮詢）
