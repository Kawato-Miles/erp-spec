# 工單狀態（WorkOrderStatus）

> [[工單]] 的狀態機。**含可逆路徑**：收回（Withdraw）+ 異動（Modification）。

## 狀態清單

```
草稿 → 製程確認中 ↔ 重新確認製程 → 製程審核完成 → 工單已交付 → 製作中 → 已完成
                                                  ↑
                                            異動（任務層 bubble-up 驅動）
```

## 主要轉移觸發

| From → To | 觸發 | 推進方式 |
|-----------|------|---------|
| 草稿 → 製程確認中 | 印務填寫完成後送審 | 手動 |
| 製程確認中 → 製程審核完成 | 印務主管審核通過 | 手動 |
| 製程確認中 → 重新確認製程 | 印務主管退回 | 手動 |
| 製程審核完成 → 工單已交付 | 首個任務交付 | **自動** |
| 工單已交付 → 製作中 | 首個生產任務進入製作中 | **自動向上傳遞** |
| 製作中 → 已完成 | 累計 QC 入庫 ≥ wo_target_qty | **自動**（齊套達標）|

## 可逆路徑

### 收回（Withdraw）

| 條件 | 行為 |
|------|------|
| 製程確認中 → 草稿 | 印務主動收回 |
| 製程審核完成 → 草稿 | 印務主動收回（重做）|

### 異動（Modification）

由任務層 bubble-up 驅動：

| 條件 | 行為 |
|------|------|
| 任務進入異動 | 工單跟隨進入「異動」狀態 |
| 異動完成 | 工單回到原狀態 |

異動期間**完成度計算持續運作**（不暫停）。

## 自動推進

工單狀態大量依靠**向上傳遞**：

```
生產任務製作中 → 任務製作中 → 工單製作中
```

詳見 [[印件狀態]] § 三、四。

## 自動完成（齊套達標）

```
工單累計入庫 ≥ wo_target_qty → 自動推進至「已完成」
```

齊套邏輯詳見 [[齊套邏輯]]。

## 一張工單僅對應一位印務

- 系統阻擋分派給第二位印務
- 異動時可走「重新確認製程」流程改派
