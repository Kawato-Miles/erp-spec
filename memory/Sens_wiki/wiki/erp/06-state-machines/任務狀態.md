# 任務狀態（TaskStatus / WorkGroupStatus）

> [[任務]] 的狀態機。**完全由[[生產任務狀態]]向上傳遞驅動**。

## 狀態清單

```
待交付 → 已交付 → 製作中 → 已完成
                ↓
            異動 → 已確認異動內容 → 製作中
```

## 主要轉移觸發

| From → To | 觸發 | 推進方式 |
|-----------|------|---------|
| 待交付 → 已交付 | 工單交付後任務同步交付 | **自動向上傳遞** |
| 已交付 → 製作中 | 首個生產任務進入製作中 | **自動向上傳遞** |
| 製作中 → 已完成 | 任務下所有生產任務完成 | **自動** |

## 異動路徑

| From → To | 觸發 |
|-----------|------|
| 已交付 / 製作中 → 異動 | 印務發起工單異動（向下傳遞）|
| 異動 → 已確認異動內容 | [[生管]]確認異動收到 |
| 已確認異動內容 → 製作中 | 完成異動安排 |

## 自動推進

**完全自動**。任務本身不被主動操作，只由：

- 上層（[[工單狀態]]交付）→ 任務交付
- 下層（[[生產任務狀態]]製作中 / 完成）→ 任務製作中 / 完成

## 為什麼需要任務層

- 工單下生產任務按**工廠**分組
- 每個工廠對應一個任務（[[glossary-erp#生產排程術語|Work Group]]）
- 任務層方便[[生管]]對「同一工廠的所有生產任務」進行派工

詳見 [[任務]]。

## 齊套邏輯

```
task_completion = min(所有生產任務 completion_ratio)
```

詳見 [[齊套邏輯]] § 二、四層計算精確流程 § 層級 2。
