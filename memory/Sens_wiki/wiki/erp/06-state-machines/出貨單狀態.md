# 出貨單狀態（ShippingStatus）

> [[出貨單]] 的狀態機。**出貨類型決定狀態路徑**（自行出貨 / 第三方物流）。出貨單掛訂單層而非工單或印件層，支援跨工單合併出貨。數量防呆與齊套性規則正本在 [[齊套邏輯]] 與 [[印件生產流程]]，本卡只定義狀態與轉換、不複述規則。

## 狀態列舉（正本）

> 本段是出貨單狀態的正本定義。狀態的新增與修改是商業決策，直接在此卡維護。轉換條件（Given/When/Then）留在 OpenSpec [shipment/spec.md](../../../../openspec/specs/shipment/spec.md) 行為規格中。

### 自行出貨路徑

| 狀態 | 說明 | 階段 |
|------|------|------|
| 未處理 | 出貨單建立後的初始狀態 | 初始 |
| 打包中 | 倉管開始打包 | 準備 |
| 待出貨 | 打包完成，等待出車 | 準備 |
| 出貨中 | 已發車出貨 | 運送 |
| 已送達 | 貨物送達客戶 | 終態 |

### 第三方物流路徑

| 狀態 | 說明 | 階段 |
|------|------|------|
| 未處理 | 出貨單建立後的初始狀態 | 初始 |
| 已出貨 | 交由第三方物流 | 運送 |
| 運送中 | 物流公司運送中 | 運送 |
| 已送達 | 物流送達客戶 | 終態 |

## 出貨單掛訂單層（設計決策）

出貨單掛於**訂單層級**，而非工單或印件層級。理由：一張訂單可能有多張工單，出貨時需跨工單合併出貨（例如同一訂單的不同印件一起打包出貨），若掛在工單層就無法跨工單合併。一張訂單下可建立多張出貨單（支援分批出貨）。

## 狀態路徑

### 1. 自行出貨路徑

```
未處理 → 打包中 → 待出貨 → 出貨中 → 已送達
```

### 2. 第三方物流路徑

```
未處理 → 已出貨 → 運送中 → 已送達
```

## 主要轉移觸發

| From → To | 觸發 | 推進方式 |
|-----------|------|---------|
| - → 未處理 | 訂單建立出貨單 | 手動 |
| 未處理 → 打包中（自行）| 倉管開始打包 | 手動 |
| 打包中 → 待出貨 | 完成打包 | 手動 |
| 待出貨 → 出貨中 | 開始出貨 | 手動 |
| 出貨中 → 已送達 | 貨物送達客戶 | 手動 |
| 未處理 → 已出貨（第三方）| 交由第三方物流 | 手動 |
| 已出貨 → 運送中 | 物流公司開始運送 | 手動 |
| 運送中 → 已送達 | 物流送達 | 手動 |

## 自動推進

**無**。全部由[[出貨人員]]手動推進。

## 數量防呆

```
出貨數量 ≤ QC 入庫數 - 已出貨數
```

範例：QC 入庫 500，已出貨 300 → 新出貨上限 = 200

詳見 [[印件生產流程]] § 五、入庫與出貨數量規則。

## 出貨單作廢回算（SHP-003）

```
新可出貨額度 = 入庫數量 - (已出貨數量 - 作廢數量)
```

詳見 [[出貨單]] § 出貨單作廢回算。

## 多工單齊套性（SHP-002）

當印件含多工單時，**取最小入庫數**為可出貨上限：

```
可出貨 = min(工單 A 入庫, 工單 B 入庫, ...)
```

詳見 [[齊套邏輯]] § 三、典型情境 § 情境 3。

## 分批出貨

分批出貨觸發節點設計尚未確認，詳見 [[SHP-005-分批出貨觸發節點]]（OQ）。[[出貨人員]]角色關切此議題。
