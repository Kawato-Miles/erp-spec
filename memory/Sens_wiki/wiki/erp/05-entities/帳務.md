# 帳務（Order Billing）

> 一句話定位：管理一張訂單的「應該收多少」「開了多少發票」「實際收到多少」三條線，讓業務有操作彈性、靠對帳確保最終一致。本卡承載帳務領域所有實體的業務可見欄位表。行為規則正本在 [[對帳一致性]]、[[付款發票邏輯]]、[[訂單異動規則]]，本卡只引用不重述。

## 這張卡要回答的問題

- 帳務領域有哪些單據？彼此怎麼關聯？
- 一張發票上會記哪些資料？
- 收款紀錄怎麼跟請款期次對應？
- 折讓單是什麼、什麼時候開？

## 營運背景

印刷廠一筆訂單的錢分三條線各自記：業務先算出「這筆訂單客戶應該付多少」（應收）、開了幾張發票合計多少（發票）、實際收到客戶匯來多少（收款）。這三條線是不同人、不同時間、用不同單據各自填進系統的，很容易某一條漏記或填錯。公司的做法不是事先用硬性規定把三條線綁死，而是讓業務照客戶實際狀況自由處理，最後靠對帳把三個數字湊在一起比對——對得起來代表帳沒問題，對不上就要有人去查。「三個數字最後一定要相等」就是這個領域不能退讓的底線。

## 欄位（業務可見）

> 按實體分子表。純技術欄位（id / FK / 時間戳記）不列入。

### 帳務公司（BillingCompany）

帳務公司是發票開立的法律主體，每家對應一個藍新（ezPay）商店帳號。系統管理員維護，業務 / 諮詢 / 會計僅讀取選用。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 公司名稱 | 文字 | 帳務主體名稱 |
| 統一編號 | 文字 | 營利事業統一編號 |
| 藍新商店代號 | 文字 | 對應 ezPay 商店帳號；全系統唯一 |
| 地址 | 文字 | 公司登記地址 |
| 電話 | 文字 | 聯絡電話 |
| 是否預設 | 是否 | 同時間僅允許一筆為預設 |
| 啟用狀態 | 是否 | 停用後不出現在下拉選項 |

### 發票（Invoice）

發票透過藍新平台開立，對應台灣統一發票。一張發票掛在一張訂單下、對應一個請款期次。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 發票種類 | 單選 | B2B / B2C |
| 買受人名稱 | 文字 | 客戶公司或個人名稱 |
| 買受人統編 | 文字 | B2B 必填、B2C 留空 |
| 買受人地址 | 文字 | 選填 |
| 買受人信箱 | 文字 | 選填 |
| 載具類別 | 單選 | 手機條碼 / 自然人憑證 / ezPay 載具（B2C 適用）|
| 載具編號 | 文字 | 載具類別對應的編號 |
| 課稅別 | 單選 | 應稅 / 零稅率 / 免稅 |
| 稅率 | 數值 | 依課稅別自動帶入 |
| 銷售額（未稅）| 金額 | 系統自動計算 |
| 稅額 | 金額 | 系統自動計算 |
| 發票金額（含稅）| 金額 | 業務可微調 |
| 備註 | 文字 | 自由填寫 |
| 狀態 | 單選 | 開立 / 作廢 |
| 發票號碼 | 文字 | 藍新回傳；開立後唯讀 |
| 作廢原因 | 文字 | 作廢時必填（限中文 6 字或英文 20 字）|
| 折讓累計 | 金額 | 該發票所有已確認折讓單金額合計（自動計算）|
| 剩餘可折讓 | 金額 | 發票金額 - 折讓累計（自動計算）|
| 折讓衍生標示 | 單選 | 無折讓 / 已部分折讓 / 已完全折讓（自動計算）|
| 商品明細 | 子結構 | 見下方「發票品項」；可從期次品項一鍵繼承或從訂單印件預填，業務可編輯 |

**發票品項**（InvoiceItem，內嵌於發票的商品明細）：

| 欄位 | 形態 | 說明 |
|------|------|------|
| 商品名稱 | 文字 | 上限 30 字；例：彩色名片 |
| 商品數量 | 整數 | 1 - 99999 |
| 商品單位 | 單選 | 張 / 本 / 冊 / 份 / 個 / 卷 / 盒 / 套 / 批 / 式 / 組（共 11 項）|
| 商品單價 | 整數 | B2B 為未稅、B2C 為含稅 |
| 商品小計 | 整數 | 數量 x 單價（自動計算）|

### 折讓單（SalesAllowance）

折讓單是發票已開立後的部分減額憑證（依台灣統一發票使用辦法第 20 條）。折讓本身不自動建立退款，業務先建退款收款紀錄再開折讓，兩者以訂單活動紀錄串聯。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 折讓金額（含稅）| 金額 | 業務輸入正數，系統內存負值 |
| 折讓金額（未稅）| 金額 | 雙欄計價 |
| 折讓後剩餘發票金額 | 金額 | 藍新回傳（自動計算）|
| 折讓原因 | 文字 | 必填 |
| 狀態 | 單選 | 已確認 / 已作廢 |
| 藍新折讓號 | 文字 | 藍新回傳（A + 14-16 碼數字）|
| 作廢原因 | 文字 | 作廢時填寫 |
| 回簽附件 | 檔案上傳（多檔）| 折讓回簽文件（合併自 SalesAllowanceFile）|

### 收款紀錄（Payment）

收款紀錄記錄客戶每一筆實際付款或退款。收款方式為「退款」時金額為負，與一般收款分軸。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 金額 | 金額 | 收款金額；退款時為負 |
| 收款方式 | 單選 | 現金 / 信用卡 / 銀行轉帳 / 支票 / 退款 / 其他 |
| 付款序號 | 文字 | 銀行入帳序號 / 信用卡授權碼 / 支票號碼 |
| 收款時間 | 日期時間 | 退款情境對應退款日期 |
| 紀錄人 | 文字 | 紀錄操作者姓名 |
| 備註 | 文字 | 退款原因 / 對帳備註 |
| 收款狀態 | 單選 | 處理中 / 已完成；處理中不計入收款淨額 |
| 對帳附件 | 檔案上傳（多檔）| 對帳憑證（合併自 PaymentFile）；退款切「已完成」時至少一檔 |
| 關聯訂單異動 | 關聯 | 由訂單異動介面建立時自動帶入；一般收款為空 |

> 處理中超過 7 天的收款紀錄會顯示「老化 N 天」提示。

### 請款期次（BillingInstallment）

請款期次把一張訂單的應收款拆成若干期（如訂金 30%、尾款 70%），每期各自追蹤開票與收款進度。開票維度與收款維度完全獨立，支援先收後開 / 先開後收。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 期別 | 整數 | 1 / 2 / 3 ...；拆期後兩筆獨立平輩期次各佔一個序號 |
| 描述 | 文字 | 訂金 30% / 尾款 70% / 加印款 / 諮詢費 等 |
| 預計金額 | 金額 | 含稅；等於品項明細合計（品項為空時保留手動填入值）|
| 預計付款方式 | 單選 | 匯款 / 現金 / 信用卡 / 支票 / 其他（規劃值，非事實）|
| 預計收款日 | 日期 | 應收日；核銷「依序填滿」依此由早到晚排序 |
| 預計開票日 | 日期 | 可空；支援尚未決定開票時點的情境 |
| 開票狀態 | 單選 | 未開立 / 已開立 / 已作廢 |
| 收款狀態 | 單選 | 未收 / 部分收款 / 已收訖（依核銷分配累計自動推導）|
| 品項明細 | 子結構 | 同發票品項結構；鏈式預填（訂單印件 → 期次 → 發票）|
| 備註 | 文字 | CSV 對帳資料來源；拆期時自動帶前綴 |
| 來源類型 | 單選 | 手動建立 / 諮詢取消 / 諮詢結束無生產 / 報價流失 / 拆期 |
| 原始預計收款日 | 日期 | 訂單審核通過時凍結為基準 |
| 原始預計開票日 | 日期 | 同上凍結基準 |
| 變更次數 | 整數 | 收款日 + 開票日累計變更次數（自動計算；拆期不計入）|
| 是否取消 | 是否 | 拆期 / 業務取消時標記；取消後不入狀態推導與應收計算 |
| 取消原因 | 文字 | 取消時填寫（如「拆兩期」「業務取消」）|

### 收款核銷分配（PaymentAllocation）

收款核銷分配是收款紀錄與請款期次之間的多對多橋接。系統依「依序填滿」自動分配後，業務可手動覆寫。

| 欄位 | 形態 | 說明 |
|------|------|------|
| 分配金額 | 金額 | 該筆收款分配至該期次的金額 |
| 對應期次 | 關聯 | 指向請款期次；空值表示預收（未分配）|
| 是否自動分配 | 是否 | 系統依序填滿建立時為是 |
| 是否手動覆寫 | 是否 | 最終金額與初次自動分配值不同時為是 |

> 稽核事件（BillingActivityEvent）為純稽核用途的結構化事件記錄，記錄期次建立、收款日變更、開票日變更、拆期、取消、核銷覆寫等操作，供 KPI 量測與審計查詢，不另建欄位表。
