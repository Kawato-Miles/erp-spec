# 任務（Task / WorkGroup）

> 一句話定位：將工單下「**同工廠**的生產任務」歸類為同一任務。屬於工單與生產任務之間的中間層。
> 這張卡負責什麼：寫任務層存在的業務理由、狀態如何向上推進。任務欄位明細以下方 `implemented-by` 指向的 OpenSpec 規格為準（生產任務與其分組），本卡只解釋為什麼要有這層分組，不重抄欄位表。

## 為什麼需要任務層

- 一張工單可能含多個生產任務（印刷、裁切、燙金 ...）
- 多個生產任務可能屬於不同工廠
- **同工廠的生產任務集中為一個任務**，方便派工與管理——生管派工、確認異動時是對「同一工廠的一批活」操作，而不是逐個生產任務操作，所以需要這層分組對象

## 關鍵關聯

- 屬於一張工單 → [[工單]]
- 包含多個生產任務 → [[生產任務]]（同工廠的）

## 狀態與向上推進

```
待交付 → 已交付 → 製作中 → 已完成
```

- 工單交付後任務同步交付
- 首個生產任務進入「製作中」→ 任務自動進入「製作中」
- 所有生產任務完成 → 任務自動完成

詳見 [[任務狀態]]。

## 異動流程

- 印務發起工單異動 → 任務進入「異動」
- [[生管]]確認異動收到 → 「已確認異動內容」
- 完成異動安排 → 回到「製作中」

## 相關角色

- [[生管]]：任務分派、確認異動
- [[印務]]：建立生產任務
