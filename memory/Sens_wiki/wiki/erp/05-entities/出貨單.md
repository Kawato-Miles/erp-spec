# 出貨單（ShipmentRecord）

> 一句話定位：訂單的出貨紀錄。**入庫數量由 QC 通過驅動**，**出貨數量不得超過入庫數量**。
> 規則正本歸屬聲明：本卡只承載欄位／關聯／狀態名。齊套與數量規則正本在 [[齊套邏輯]]、狀態流轉正本在 [[出貨單狀態]]，本卡只引用不重述。

## 這張卡要回答的問題

- 一張出貨單上看得到哪些資料？→ 看「欄位（業務可見）」。
- 一次出貨可以同時包含幾種印件？→ 可含同一張訂單的多種印件（合箱，不可跨訂單；2026-07-20 Miles 拍板），每種各出多少記在出貨明細。
- 一次最多能出多少？→ 逐明細行對該印件的可出貨額度檢查；印件層級統計入庫數取齊套下限（工單 A 入庫 500＋工單 B 入庫 300、各需一份 → 上限 300），規則見 [[齊套邏輯]]。
- 出貨單作廢後額度怎麼回算？→ 逐明細行回補各印件額度：新可出貨額度＝入庫數量－（已出貨數量－作廢數量）；入庫 1000、已出貨 800 含作廢 200 → 額度 400。
- 不同出貨方式走的路一樣嗎？→ 三類方式（自取／專車配送／第三方物流）共用同一套狀態與確認動作，自取不經運送中，其餘經運送中到送達；異常統一為終態、重出建新單，正本見 [[出貨單狀態]]。
- 出貨和送達憑什麼認定？→ 出貨確認與送達確認兩動作（時間＋操作人＋憑證），每張單收尾必留「誰確認、何時、憑什麼說到了」（2026-07-20 拍板，解 [[SHP-012-出貨送達憑證不對稱|SHP-012]]）。

## 營運背景

訂單做完不等於交完——同一張訂單的東西可能分批出、多種印件合在一起出、用不同方式出（宅配、超商、司機配送或客戶自取），每批要能追蹤到了沒。[[業務]]填出貨單決定這一次怎麼出：要合／不合、每種出多少、送到哪，以此告訴揀貨與出貨人員。把每次出貨獨立成一張出貨單，是因為出貨額度要對著入庫數量管控（品檢通過才算入庫、出貨不能超過入庫），分批與作廢都要能逐印件回算額度，沒有獨立單據這筆帳算不清。

## 欄位（業務可見）

> 本段是該實體的欄位正本。欄位的新增與修改是商業決策，直接在本卡維護。出貨模組的行為規格尚待建立（見 [[SHP-006-出貨模組spec待建立|SHP-006]]），不影響本卡欄位正本地位；欄位依雛形系統既有介面整理。

|---------|------|---------|------|---------|
| 所屬訂單 | 這張出貨單屬於哪張訂單 | 連結（指向 [[訂單]]）| 建單時帶入 | 不可 |
| 出貨明細 | 這次出哪些印件、各多少（見下方明細子表；可含同訂單多種印件、不可跨訂單）| 明細（多筆）| 業務建單時選擇 | 建單時設定 |
| 收件資訊 | 送到哪、給誰收（收件人／地址／電話）| 文字 | 業務建單時填寫 | 建單時填寫 |
| 裝箱指示 | 業務對裝箱方式的指定（如「200 個一箱」）；未填時揀貨人員自行依公版箱規判斷 | 文字 | 業務建單時填寫（選填）| 建單時填寫 |
| 出貨方式 | 自取／專車配送（公司自行配送）／第三方物流；自取不經運送中（規則正本見 [[出貨單狀態]]）| 單選 | 業務建單時選擇 | 建單時選擇 |
| 物流商 | 出貨方式＝第三方物流時必填（順豐／新竹物流／超商／其他，選項可增不改流程）| 單選 | 業務建單時選擇 | 建單時選擇 |
| 狀態 | 目前的出貨進度 | 單選（依 [[出貨單狀態]] 定義）| 揀貨段揀貨人員推進、物流段出貨人員推進 | 依狀態機規則 |
| 實際裝箱數量 | 揀貨清點後實際裝箱的總個數；與出貨明細要求不符時回報異常給業務處理 | 數字 | 揀貨人員裝箱回報時填 | 回報時填寫 |
| 箱數 | 裝成幾箱 | 數字 | 揀貨人員裝箱回報時填 | 回報時填寫 |
| 箱規 | 用哪種公版箱體（對應第三方物流箱數計價；留存供每月對帳比對）| 單選（公版箱體）| 揀貨人員裝箱回報時選 | 回報時選擇 |
| 裝箱照片 | 裝箱照、秤重照 | 檔案上傳（多檔）| 揀貨人員裝箱回報時附 | 回報時上傳 |
| 出貨確認 | 貨離廠的確認：時間＋操作人；第三方物流另填托運單號（追蹤碼）| 日期時間＋人員（唯讀）＋文字 | 出貨人員確認時系統寫入、托運單號手填 | 托運單號可補填 |
| 送達確認 | 貨到客戶手上的確認：時間＋操作人＋**送達憑證**（專車＝司機交付照、自取＝現場點交照、第三方＝回填物流商配達時間）| 日期時間＋人員（唯讀）＋檔案或文字 | 出貨人員確認時寫入 | 憑證可補附 |
| 作廢理由／作廢資訊 | 作廢時必填理由；何時作廢 | 文字＋日期時間（唯讀）| 作廢時寫入 | 不可 |

### 出貨明細（每條＝一種印件出多少）

| 欄位 | 說明 |
|------|------|
| 印件 | 出的是哪件印刷品（必須屬於本出貨單的訂單） |
| 數量 | 大於 0，不超過該印件的可出貨額度（入庫數 − 已出貨數 ＋ 作廢回補，正本見 [[齊套邏輯]]） |

> 範圍外點名：內部主鍵、訂單與印件關聯鍵，存在但屬實作規格。分批出貨的系統實現尚未定案，見 [[SHP-005-分批出貨觸發節點|SHP-005]]。運費的帳不做——箱數／箱規留存供每月與第三方物流對帳比對（對帳機制屬後續設計）。

## 關鍵關聯

- 屬於 [[訂單]]（N:1）：一張訂單可分批出貨、有多張出貨單
- 出貨明細指向多筆 [[印件]]（同訂單、可多種印件合箱；2026-07-20 拍板）：額度檢查、作廢回補、送達累計逐明細行對印件計算（規則見 [[齊套邏輯]]）；狀態掛單頭，一箱貨一起送達、一起異常
- 上游依賴 [[QC]]：品檢通過數決定入庫數量，入庫數量決定可出貨上限
