# 出貨單（ShippingRecord）

> 訂單的出貨紀錄。**入庫數量由 QC 通過驅動**，**出貨數量不得超過入庫數量**。

> 例外說明：其他實體卡的欄位明細以 OpenSpec § Data Model 為正本、本卡不重抄；但出貨模組 OpenSpec 目前只有行為規則（§ Requirement: 出貨單管理），尚未建立 § Data Model 的欄位正本（見 [[SHP-006-出貨模組spec待建立]]）。在 OpenSpec Data Model 補齊前，本卡的欄位表暫作參考正本，欄位以雛形系統 `sens-erp-prototype/src/types/order.ts` 的 `ShipmentRecord` / `ShipmentItem` 為實作依據。

## 出貨類型與狀態路徑

| 類型 | 狀態路徑 |
|------|---------|
| 自行出貨 | 未處理 → 打包中 → 待出貨 → 出貨中 → 已送達 |
| 第三方物流 | 未處理 → 已出貨 → 運送中 → 已送達 |

詳見 [[出貨單狀態]]。

## 核心規則（business-processes spec）

### 入庫與出貨數量規則

- **入庫數量僅在 QC 通過後才計入**
- **出貨數量不得超過入庫數量**

### 多工單出貨齊套性（SHP-002）

當出貨涉及多個工單時，系統以**印件層級**統計入庫數為基準，執行齊套性邏輯：

範例：印件含工單 A（入庫 500）+ 工單 B（入庫 300），每份印件需 A、B 各一份 → 可出貨上限 = min(500, 300) = **300**

### 出貨單作廢回算（SHP-003）

```
新可出貨額度 = 入庫數量 - (已出貨數量 - 作廢數量)
```

範例：入庫 1000、已出貨 800（含一筆作廢 200）→ 新可出貨額度 = 1000 - (800 - 200) = 400
