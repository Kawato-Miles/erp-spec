# 出貨單（ShippingRecord）

> 訂單的出貨紀錄。**入庫數量由 QC 通過驅動**，**出貨數量不得超過入庫數量**。

## 核心欄位（推測）

| 欄位 | 說明 |
|------|------|
| `id` | 系統 ID |
| `order_id` | 所屬訂單 → [[訂單]] |
| `print_item_id` | 對應印件 → [[印件]] |
| `quantity` | 出貨數量 |
| `shipping_method` | 出貨方式（自行出貨 / 第三方物流）|
| `status` | 狀態（見 [[出貨單狀態]]）|
| `tracking_no` | 物流追蹤碼 |
| `shipped_at` / `delivered_at` | 出貨時間 / 送達時間 |
| `cancelled_reason` / `cancelled_at` | 作廢相關 |

## 關鍵關聯

- 屬於 `Order` → [[訂單]]
- 對應 `PrintItem` → [[印件]]

## 出貨類型與狀態路徑

| 類型 | 狀態路徑 |
|------|---------|
| 自行出貨 | 未處理 → 打包中 → 待出貨 → 出貨中 → 已送達 |
| 第三方物流 | 未處理 → 已出貨 → 運送中 → 已送達 |

詳見 [[出貨單狀態]]。

## 核心規則（business-processes spec）

### 入庫與出貨數量規則

- **入庫數量僅在 QC 通過後才計入**
- **出貨數量不得超過入庫數量**

### 多工單出貨齊套性（SHP-002）

當出貨涉及多個工單時，系統以**印件層級**統計入庫數為基準，執行齊套性邏輯：

範例：印件含工單 A（入庫 500）+ 工單 B（入庫 300），每份印件需 A、B 各一份 → 可出貨上限 = min(500, 300) = **300**

### 出貨單作廢回算（SHP-003）

```
新可出貨額度 = 入庫數量 - (已出貨數量 - 作廢數量)
```

範例：入庫 1000、已出貨 800（含一筆作廢 200）→ 新可出貨額度 = 1000 - (800 - 200) = 400

## 分批出貨

分批出貨觸發節點設計尚未確認，詳見 [[SHP-005-分批出貨觸發節點]]（OQ）。[[出貨]]角色關切此議題。
