# 轉交單（TransferTicket）

> 串聯上下兩個生產任務的轉交工作單據：上一道做完的產出，要轉給下一道接著做——誰的貨、從哪到哪、送了多少、到了沒。把過去靠紙本工單跟著貨走的流轉，變成系統裡可追蹤、可作為下游到料放行的紀錄。**單據由系統於[[廠務]]確認搬運時自動生成**（2026-07-20 拍板：轉交是常態且滾動，設計核心是「系統告訴下一步」、不是人建表單）——師傅報工累積可轉交量、廠務的待搬清單滾動顯示可送的貨，廠務挑貨搬運、確認送達（附簽收照）即成單；[[印務]]只處理例外（作廢、改路徑）。
>
> 範圍註記：轉交涵蓋場內站間移動，**暫存區是合法的來源與目的地站點**（2026-07-20 拍板）——工廠的暫存區同時放外發回貨、待品檢、品檢完待裝箱、裝箱完待出貨的貨，貨進出暫存區都以轉交留痕，系統才答得出「這批貨在哪」。三類常態轉交：產線 → 產線（或品檢區）、**暫存區 → 產線**（外發回貨點收完成後，下游到料放行觸發待搬清單）、**品檢區 → 暫存區**（品檢通過成品入暫存區，見 [[PT-011-成品移轉揀貨暫存區的搬運留痕|PT-011]] 決議）。場外回場內的物流本身走 [[貨運單]]（中國線）與 [[生產任務狀態]] 在途狀態，非轉交單範圍；台灣外包回場銜接見 [[PT-008-外發回廠流轉的系統設計範圍|PT-008]]。

## 概述

一張轉交單綁定**單一印件**（不跨印件），承載這次運送的對象、數量與送達憑證。同一印件可開多張轉交單（分批轉交，對應分批出貨）；一張轉交單的明細可含同印件、跨工單的多筆生產任務（例如印件 A 的工單 1 印刷＋工單 2 外包模切合併一趟送）。

轉交單是 [[工序相依性規則]] 中「需轉交才可製作」的載體——下一道工序的到料量看「轉交單已送達的數量」，到料量大於 0 即可開工、滾動累積（數量級放行，正本見 [[工序相依性規則]]）。生產任務**預設需轉交**，印務在製程規劃時只標例外「不需轉交」。

### 轉交明細（每條＝從一個生產任務抽取的部分數量）

| 欄位 | 說明 |
|------|------|
| 生產任務 | 這條明細抽哪個生產任務的量（必須屬於本轉交單的印件） |
| 數量 | 大於 0，不超過該生產任務的可申請上限（見下） |

## 關鍵關聯

- 屬於一張 [[印件]]（單一、不跨印件）
- 明細指向多筆 [[生產任務]]（同印件、可跨工單）
- 作為 [[工序相依性規則]] 中下游任務的到料依據（已送達數量計入下游到料量、滾動放行）

## 核心規則

- **可申請上限**：每條明細的數量不得超過「該生產任務已報工量 − 同印件其他未作廢轉交單已抽走的量」；已作廢轉交單的明細不佔用額度（與品檢可申請上限同模式，計算正本見 [[齊套邏輯]]）。
- **建立前置**：貨要先存在才有得搬——產線間轉交以來源生產任務已有報工量為前提；暫存區 → 產線以該批貨已完成點收登記為前提；品檢區 → 暫存區以品檢通過量為前提。
- **不跨印件**：一張轉交單所有明細的生產任務必須同屬一張印件。
- **同單不重複**：同一生產任務不可在一張轉交單的明細出現兩次。
