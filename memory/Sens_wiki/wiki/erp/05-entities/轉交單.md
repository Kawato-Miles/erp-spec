# 轉交單（TransferTicket）

> 串聯上下兩個生產任務的轉交工作單據：上一道做完的產出，要轉給下一道接著做——誰的貨、從哪到哪、送了多少、到了沒。把過去靠紙本工單跟著貨走的流轉，變成系統裡可追蹤、可作為下游派工前置的紀錄。物理搬運由[[廠務]]執行，單據的建立與確認送達由[[印務]]操作。
>
> 範圍註記：目前釐清的轉交都是**場內**轉交（產線到產線、產線到品檢）。場外（外包廠／中國廠）製作完成回場內這段的系統表達（用轉交單還是生產任務的運送中狀態）待設計評估，見 [[PT-008-外發回廠流轉的系統設計範圍|PT-008]]。

## 概述

一張轉交單綁定**單一印件**（不跨印件），承載這次運送的對象、數量與送達憑證。同一印件可開多張轉交單（分批轉交，對應分批出貨）；一張轉交單的明細可含同印件、跨工單的多筆生產任務（例如印件 A 的工單 1 印刷＋工單 2 外包模切合併一趟送）。

轉交單是 [[工序相依性規則]] 中「需轉交才可製作」的載體——下一道工序的前置設為「轉交單已送達」，貨到位才放行派工。

### 轉交明細（每條＝從一個生產任務抽取的部分數量）

| 欄位 | 說明 |
|------|------|
| 生產任務 | 這條明細抽哪個生產任務的量（必須屬於本轉交單的印件） |
| 數量 | 大於 0，不超過該生產任務的可申請上限（見下） |

## 關鍵關聯

- 屬於一張 [[印件]]（單一、不跨印件）
- 明細指向多筆 [[生產任務]]（同印件、可跨工單）
- 作為 [[工序相依性規則]] 中下游任務的前置（「轉交單已送達」放行派工）

## 核心規則

- **可申請上限**：每條明細的數量不得超過「該生產任務已報工量 − 同印件其他未作廢轉交單已抽走的量」；已作廢轉交單的明細不佔用額度（與品檢可申請上限同模式，計算正本見 [[齊套邏輯]]）。
- **建立前置**：印件下至少一個生產任務已有報工量（`已報工 > 0`）才能建轉交單。
- **不跨印件**：一張轉交單所有明細的生產任務必須同屬一張印件。
- **同單不重複**：同一生產任務不可在一張轉交單的明細出現兩次。
