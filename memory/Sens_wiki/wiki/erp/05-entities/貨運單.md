# 貨運單（Waybill）

> 一句話定位：記錄中國供應商把外發製作完成的貨集中運回台灣這一趟跨境物流的單據——誰承運、申報多少運費、發了多重、回台收了多少、實秤多重、運費與關稅怎麼攤回原本的生產任務。屬生產執行領域。本卡只承載欄位／關聯／狀態名；運費與關稅的分攤計算政策正本在 [[印件生產流程]]，本卡只引用不重述。
>
> 範圍註記：貨運單是「中國回台」的內部成本物流單，運費與關稅按重量佔比分攤回各 [[生產任務]]（成本回生產任務）。它與台灣對客戶出貨的 [[出貨單]] 是兩件事——出貨單記的是貨怎麼交到客戶手上，貨運單記的是貨怎麼從中國回到台灣場內，兩者不混用、不互相帶入。

## 這張卡要回答的問題

- 為什麼中國回台的物流要獨立成一張單，不併進生產任務或出貨單？→ 看「營運背景」。
- 一張貨運單上業務看得到哪些資料？→ 看「欄位（業務可見）」。
- 一張貨運單對應幾個生產任務、幾張訂單？→ 看「關鍵關聯」（集運合併：一運單多訂單多生產任務）。
- 運費與關稅怎麼攤回去？攤到哪一層？→ 攤回各 [[生產任務]]，依重量佔比，組成式見欄位段、政策見 [[印件生產流程]]。
- 大陸申報重量和台灣實秤對不上怎麼辦？→ 看「重量差異」欄位。
- 認列狀態算不算狀態機？→ 兩態（待認列／已認列），列在欄位即可，見「相關狀態機」段說明。

## 營運背景

中國線的稿件外發給中國供應商製作，做完之後貨不是一張單一張單分開寄，而是交給集運商集中、整批運回台灣——一趟運送往往同時載著好幾張訂單、好幾個生產任務的貨（集運合併）。這趟運送本身有自己的承運商、申報運費、發貨重量、回台收貨與秤重憑證，而且整趟的運費與關稅要按各筆貨的重量佔比，攤回它們原本所屬的生產任務當作實際成本。這些資料不屬於任何單一生產任務，也不屬於對客戶的出貨單，所以獨立成一張貨運單來承載。

為什麼不併進 [[出貨單]]：出貨單管的是台灣對客戶的交付（貨送到客戶手上），貨運單管的是中國到台灣的內部跨境回場。兩者的對象、方向、成本歸屬都不同，併在一起會造成一單兩用。

### 回台收貨與秤重（台灣端）

> 台灣端點收由[[揀貨人員]]在暫存區執行（2026-07-20 拍板）：對工單逐筆明細清點數量、秤重量——這是數量與狀態確認，不是品檢任務（品質把關仍在入庫前最終品檢）。**點收確認的連動**：明細對應的[[生產任務]]自動推成「已完成」（外發這道工序結束，取代印務手動推進）、該批貨登記在暫存區掛回工單，下游工序的到料放行隨之啟動（見 [[工序相依性規則]]）。點收發現短少或明顯損壞，回報[[印務]]處理（對外部廠商追補屬印務外發主責）。

| 介面欄位 | 說明 | 欄位形態 | 來源 | 可否修改 |
|---------|------|---------|------|---------|
| 收貨點收數量 | 台灣端實際收到的數量（逐筆明細點收，見明細子表） | 整數 | 揀貨人員點收時填寫 | 收貨時可填 |
| 台灣實際秤重 | 台灣端實際秤得的重量合計（公斤） | 數字（公斤） | 揀貨人員點收時填寫 | 收貨時可填 |

## 明細（每條＝一筆貨在這趟運送中的攤算）

> 明細以工單／生產任務為單位逐筆列出，承載各筆貨的點收、實秤與運費關稅分攤。分攤的計算政策正本在 [[印件生產流程]]。

| 欄位 | 說明 |
|------|------|
| 工單／生產任務 | 這條明細是哪張工單、哪個生產任務的貨（含稿件編號） |
| 派單數量 | 這筆貨外發出去的數量 |
| 點收數量 | 台灣端實際收到的數量 |
| 實際重量 | 這筆貨在台灣端實秤的重量（公斤） |
| 重量佔比 | 這筆貨的實際重量佔整趟實秤合計的百分比 |
| 分攤運費 | 依重量佔比攤到這筆貨的運費（人民幣＋新臺幣），攤回該生產任務 |
| 分攤關稅 | 依重量佔比攤到這筆貨的關稅（人民幣＋新臺幣），攤回該生產任務 |

## 關鍵關聯

- 由一家中國 [[中國廠商]] 發貨（中國供應商）。
- 一張貨運單關聯多筆 [[工單]] 與 [[生產任務]]（集運合併：一趟運送載多張訂單、多個生產任務的貨）。
- 運費與關稅按重量佔比攤回各 [[生產任務]]（成本回生產任務），餵入 [[成本差異]] 的實際成本面。
- 與台灣對客戶的 [[出貨單]] 分離（不同對象、不同方向、不同成本歸屬，不互相帶入）。
- 外發製作的進程歸屬 [[派單狀態]]（貨運單只承接「回台物流」這一段，外發委外的製作進程不在本卡）。

## 相關狀態機

認列狀態為簡單兩態（待認列 / 已認列），標記這趟運費與關稅的會計認列是否完成，列在欄位即可，不另開狀態機卡。若後續「認列」需要更細的轉換條件或鎖定規則（如已認列後哪些欄位鎖定、哪些仍可編輯），再評估是否升級為獨立狀態機卡，本卡不硬編。
