# 印件（PrintItem / OrderPrintItem）

> 一句話定位：訂單下的「單個印刷產品項」，是公司實際要生產、審稿、出貨的最小產品對象。**雙維度狀態**：審稿維度 + 印製維度（見 [[印件狀態]]）。
> 這張卡負責什麼：寫一件印件存在的業務理由、哪些設計不直覺、會牽連到哪些單據。**業務可見欄位表為本卡正本**（§ 欄位（業務可見）），OpenSpec 僅保留 Requirements 與 Scenarios。

## 這張卡要回答的問題

- 印件為什麼要從訂單拆出來、不直接掛在訂單上算數量？→ 看「營運背景」。
- 為什麼印件有「兩個狀態」？→ 看「雙維度狀態的業務理由」。
- 印件分哪幾種型別、各從哪來？→ 看「印件型別（三值）」。
- 印件需求是從需求單帶來的，成交後兩邊還會互相同步嗎？→ 看「需求單帶過來後的同步行為」。

## 營運背景

一張訂單可能同時要做好幾件不同的印刷品（型錄、名片、海報），每件的規格、數量、稿件、難易度、生產流程都不一樣，做完的時間點也不同。把「每一件印刷品」獨立成印件，是因為審稿、排工單、品檢、出貨全都以「單件印刷品」為單位推進——訂單只是把它們收在一起向客戶收款的容器，真正被生產系統追蹤的是印件。

## 雙維度狀態的業務理由

印件同時被兩條不相干的流程推進，所以拆成兩個維度各自記狀態（欄位明細見 OpenSpec 正本，這裡只解釋為什麼要兩個）：

- **審稿維度**：稿件有沒有上傳、審了沒、合格還是退件——這條由審稿人員推進。
- **印製維度**：工單有沒有交付、做到哪、入庫了沒、出貨了沒——這條由印務 / 出貨推進。

兩條互不從屬（稿件還在審，工單可能還沒建；稿件合格了，生產才開始），硬合成一個狀態會互相打架，所以分開記。

> **「已棄用」的特殊性**：印製維度有一個「已棄用」狀態，唯一觸發場景是打樣結果判定為「NG-稿件問題」時系統自動把原印件棄用、另複製一件新印件，見 [[打樣流程]]。被棄用的印件雙維度狀態都保留，作為稽核軌跡。複製出來的新印件帶「衍生來源印件」追溯欄位指回原印件（見欄位表），補足純文字備註無法反查的問題。

## 印件型別（三值）

| 型別 | 業務語意 | 建立時機 |
|------|---------|---------|
| 打樣印件 | 訂單成立後業務先打樣確認用、數量少 | 訂單階段 |
| 大貨印件 | 正式量產印件 | 訂單階段 |
| 補印印件 | 售後服務單觸發、因瑕疵 / 規格不符 / 物流問題等需重新製作 | 售後服務單階段 |

**補印印件特殊規則**：

- 恆定約束：補印印件**必定掛在某張售後服務單底下**——型別為補印卻沒有來源售後單的印件不允許存在
- **跳過審稿環節**：建立同 transaction 系統自動產生「售後補印自動通過審稿輪次」（審稿輪次來源標為「售後補印」、指向來源印件），並沿用來源印件最終合格稿件
- 印製狀態直接進入「等待中」分類，印務主管立即可在印件總覽分配工單
- 走完整生產流程（工單 → 排程 → 生產任務 → 品檢 → 出貨），與大貨印件等效
- **補印不可同時改稿**（[[AFT-6-補印改稿情境入口設計|AFT-6-補印改稿情境入口設計（已封存）]]）：客戶要改稿則走新訂單流程，不在售後路徑
- **跨頁面識別**：三種型別在各頁面以統一標示呈現，補印件可一眼識別並反查來源售後服務單（標示樣式與互動屬實作規格）

**加印（additional print）**：訂單期間客戶追加印件、不形式化為型別列舉值（隱含設計、未來需要時擴充）。

對齊業界 MES 主流（Tharstern / PrintEPS / Printavo / PrintNow 普遍預設 reorder / reprint 跳 artwork approval 直接到 production）。

## 需求單帶過來後的同步行為

印件需求在需求單階段就先填了（難易度、是否免審、檔案備註），成交生成訂單後，有些欄位兩邊還會連動、有些則各走各的，這個分工不直覺，故記下：

- **難易度 / 是否免審**：parity 同步——需求單側或訂單側改了，另一側跟著變（因為它們影響審稿與排程，要保持一致）。
- **印件檔案備註**：脫鉤型——成交後需求單側與訂單側各自獨立編輯、不再同步（成交後兩邊備註用途已分家）。

### 印件檔案（PrintItemFile，子實體）

印件檔案不獨立建卡，作為印件的 1:N 子實體附屬說明。每件印件包含多份上傳檔案，分兩種角色（印件檔 / 縮圖），按審稿輪次分組管理。詳見 [[稿件管理規則]]。

### 預計產線（PrintItemExpectedLine，關聯表）

預計產線為印件與產線之間的多對多關聯表，業務意義已在上方「預計產線」欄位涵蓋，不另行描述。
