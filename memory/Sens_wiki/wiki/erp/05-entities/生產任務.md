# 生產任務（ProductionTask）

> 一句話定位：工單內的**單一生產動作**（如印刷、裁切、燙金）。每個生產任務有負責的廠商。
> 這張卡負責什麼：寫生產任務存在的業務理由、廠商類型如何決定流程、終態規則。完整欄位明細以下方 `implemented-by` 指向的 OpenSpec 規格 § Data Model: ProductionTask 為正本，本卡只解釋業務意義，不重抄欄位表。

## 這張卡要回答的問題

- 工單已經是一條生產線上的活了，為什麼底下還要拆生產任務？→ 看「營運背景」。
- 為什麼同樣是生產任務，廠商不同走的狀態路徑就不同？→ 看「工廠類型決定狀態路徑」。
- 印件要在不同站點之間運送時靠什麼憑證？→ 看「轉交單」。

## 營運背景

一張工單做出來，中間往往要經過好幾道工序，而且這些工序可能交給不同對象做：印刷自己印、裁切外包、燙金又是另一家廠。把工單底下「每一道實際要執行的生產動作」獨立成生產任務，是因為每道工序的負責廠商、產線、工期、報工、品檢都各自獨立，而且要分廠追蹤。同一工廠的多個生產任務會再集中為一個 [[任務]] 方便派工，但被報工、被品檢、被算完成度的最小單位是生產任務。

## 需要解釋的資料

生產任務上要記哪些欄位以 OpenSpec 正本為準，這裡只解釋不直覺的：

- **生產任務對工單的倍數**：記「完成 1 份工單需要多少份這個生產任務的產出」，用來把生產任務的完成數量換算回工單進度，規則正本見 [[數量換算規則]]（技術欄位：`quantity_per_work_order`）。
- **是否計入工單完成度**：有些生產任務（如純準備性工序）不影響「工單做完沒」的判定，靠這個旗標排除（技術欄位：`affects_product`）。

## 關鍵關聯

- 屬於一張工單 → [[工單]]
- 屬於一個任務（同工廠分組）→ [[任務]]
- 包含報工紀錄
- 包含品檢（見 [[wiki/erp/05-entities/QC]]）
- 引用 BOM：材料 / 工序 / 裝訂三類互斥

## 工廠類型決定狀態路徑

廠商性質不同，運送與交付的環節就不同，所以狀態路徑跟著工廠類型走（這是設計上把工廠類型獨立成欄位的理由）：

| 工廠類型 | 狀態路徑 |
|---------|---------|
| 自有工廠 | 待處理 → 製作中 → 已完成 |
| 加工廠 | 待處理 → 製作中 → 已完成 |
| 外包廠 | 待處理 → 製作中 → 運送中 → 已完成 |
| 中國廠商 | 待處理 → 製作中 → 已送集運商 → 運送中 → 已完成 |

詳見 [[生產任務狀態]]。

## 轉交單（TransferTicket）

印件級跨站點運送憑證（add-production-task-transfer change 引入）：

- 透過印件 FK 關聯印件，一印件可有多張以支援分批轉交
- 每張單可含明細抽取同印件下多個生產任務的產出，但**不可跨印件**
- 狀態機：運送中 / 已送達 / 已作廢

詳見 [[glossary-erp#生產排程術語]] § 轉交單。

## 終態與作廢

- **已作廢**：待處理時可作廢
- **報廢**：製作中時可報廢
- 生產任務作廢時，**關聯的品檢自動作廢**（見 [[wiki/erp/05-entities/QC]]）
