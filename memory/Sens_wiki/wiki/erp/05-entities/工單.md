# 工單（WorkOrder）

> 生產排程的基本執行單位。**一張印件可有多張工單**，**一張工單僅對應一位印務**。

## 核心欄位

| 欄位 | 說明 |
|------|------|
| `id` / `wo_no` | 系統 ID / 編號 |
| `print_item_id` | 所屬印件 → [[印件]] |
| `type` | 打樣 / 大貨（見 [[打樣流程]]）|
| `region` | 區域（決定工廠類型路由）|
| `status` | 狀態（見 [[工單狀態]]）|
| `assigned_to` | 分派印務（一張工單僅一位印務）|
| `quantity_per_print_item` | **完成 1 份印件需要多少份此工單**（見 [[數量換算規則]]）|
| `wo_target_qty` | 目標生產數量 |
| `wo_produced_qty` | 累計生產數量 |
| `wo_warehouse_qty` | 累計入庫數量（QC 通過後計入）|

## 關鍵關聯

- 屬於一張 `PrintItem` → [[印件]]
- 包含 `ProductionTask[]` → [[生產任務]]，按廠商分組為 [[任務]]
- 包含 `BOMLineItem[]`：BOM 行項目（材料 + 工序 + 裝訂）
- 包含 `WorkOrderModification[]`：異動紀錄
- 關聯多筆 `QCRecord`（透過 ProductionTask 間接）→ [[wiki/erp/05-entities/QC]]

## 建單與分派流程

```
審稿合格 → 印務主管建工單草稿 → 分派印務 → 印務填寫 → 製程確認中
  → 印務主管審核 → 製程審核完成 → 工單已交付 → 製作中 → 已完成
```

詳見 [[印件生產流程]]。

## 異動流程

- 印務發起異動 → 工單從正常狀態進入「異動」
- 任務層 bubble-up 驅動工單異動狀態
- 異動期間完成度計算持續運作
- 異動完成後 → 回到正常狀態

## 自動完成

當 `累計 QC 入庫 >= wo_target_qty` 時，工單自動推進至「已完成」。
