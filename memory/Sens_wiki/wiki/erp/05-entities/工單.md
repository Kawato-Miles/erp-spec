# 工單（WorkOrder）

> 一句話定位：生產排程的基本執行單位。**一張印件可有多張工單**，**一張工單僅對應一位印務**。
> 這張卡負責什麼：寫工單存在的業務理由、生命週期、會牽連到哪些單據。完整欄位明細以下方 `implemented-by` 指向的 OpenSpec 規格 § Data Model: WorkOrder 為正本，本卡只解釋業務意義，不重抄欄位表。

## 這張卡要回答的問題

- 印件已經是最小產品了，為什麼底下還要拆工單？→ 看「營運背景」。
- 工單怎麼被建立、走完一生？→ 看「建單與分派」與「自動完成」。
- 為什麼工單對印件要記一個「倍數」？→ 看「需要解釋的資料」。

## 營運背景

一件印件（例如一本型錄）做出來，往往不是一道工序一次完成的：本體要印、內頁要印、配件要另外做，可能還要分不同產線或不同廠。把「印件底下每一條實際排去產線生產的單」獨立成工單，是因為排程、分派印務、報工、品檢、入庫都以「單條生產線上的活」為單位推進，而不是以整件印件為單位。所以一張印件可以對應多張工單，每張工單只交給一位印務負責。

## 需要解釋的資料

工單上要記哪些欄位以 OpenSpec 正本為準，這裡只解釋不直覺的：

- **工單對印件的倍數**：記「完成 1 份印件需要多少份這張工單的產出」。因為一張印件可能要好幾張工單各湊一份（本體 1 份 + 配件 2 份），數量不是 1:1，要靠這個倍數換算「印件做完了沒」，規則正本見 [[數量換算規則]]（技術欄位：`quantity_per_print_item`）。
- **累計入庫數量**：只有品檢通過的數量才計入入庫，這是後續「能不能出貨」的依據（技術欄位：`wo_warehouse_qty`）。

## 關鍵關聯

- 屬於一張印件 → [[印件]]

## 建單與分派

```
審稿合格 → 印務主管建工單草稿 → 分派印務 → 印務填寫 → 製程確認中
  → 印務主管審核 → 製程審核完成 → 工單已交付 → 製作中 → 已完成
```

詳見 [[印件生產流程]]。

## 異動流程

- 印務發起異動 → 工單從正常狀態進入「異動」
- 任務層狀態向上傳遞驅動工單異動狀態
- 異動期間完成度計算持續運作
- 異動完成後 → 回到正常狀態

## 自動完成

當累計品檢入庫數量達到工單目標生產數量時，工單自動推進至「已完成」。
