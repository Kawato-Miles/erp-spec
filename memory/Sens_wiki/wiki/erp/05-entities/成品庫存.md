# 成品庫存

> 完工良品「打包入庫」成為成品庫存的一筆獨立交易記錄，屬生產執行領域。它把「完工」（做出多少品檢通過的良品，是生產績效）與「入庫」（進了多少成品進倉，是庫存收貨）分成兩件事。入庫數扣帳給出貨、跨訂單調撥等的決策政策正本在 [[齊套邏輯]]，本卡只記欄位與「為什麼要存」。本實體在後續階段（Phase 2/3）才啟用，架構先預留掛點。

## 這張卡要回答的問題

- **為什麼完工與入庫要分成兩筆？** 完工良品數回答「生產做出多少品檢通過的良品」（生產績效），入庫數回答「進了多少成品到倉」（庫存收貨）；兩者分離才能管「完工待出貨的在庫」「分批入庫」「跨訂單成品調撥」，而不是把品檢通過數直接當可出貨量。
- **入庫數與完工良品數差在哪？** 完工良品數是工單上品檢通過的數量，未必當下全數入倉；入庫數是實際打包進倉的成品數，可能分批入庫、可能小於累計完工數。差額代表「完工但尚未入庫」的在製成品。
- **出貨怎麼從成品庫存扣？** 出貨單從成品庫存扣帳，可出貨數為入庫數扣除已出貨的餘量；按單生產（MTO）下 Phase 1 案件多為完工即出貨、不經此實體，Phase 2/3 啟用後才有真正的在庫扣帳。

## 營運背景

現況把「品檢通過的完工良品」直接當「可出貨量」，中間沒有成品庫存這一層。依業界做法把「完工（做出多少良品）」與「入庫（進了多少成品庫存）」分開，才能管完工待出貨的在庫、分批入庫、跨訂單成品調撥。按單生產下多數案件完工即出貨、不長期入庫，因此本實體在 Phase 2/3 才啟用；本卡先寫欄位骨架、架構先預留掛點。

## 關鍵關聯

- 成品庫存 N:1 [[印件]]——一個印件可分多筆入庫（分批入庫），每筆成品庫存只來源於一個印件。
- 成品庫存 N:1 [[工單]]——每筆成品庫存追溯到一張生產來源工單。
- [[出貨單]] 引用成品庫存扣帳——出貨時從成品庫存的可出貨數扣減。
