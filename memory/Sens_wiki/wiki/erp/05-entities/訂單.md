# 訂單（Order）

> 一句話定位：訂單是客戶確認下單後，公司據以收款、生產、出貨的核心單據，橫跨報價、生產、款項、出貨多個環節。分**三種來路**：一般訂單（需求單成交）、諮詢訂單（諮詢結尾收尾）、線上訂單（客戶在線上前台自己下單，電商網站）。
> 這張卡負責什麼：只寫一張訂單上要記哪些資料、會牽連到哪些單據、訂單會有哪些狀態名稱。訂單成立後金額怎麼增減的規則正本在 [[訂單異動規則]]、款項與發票對帳的規則正本在 [[付款發票邏輯]]、狀態怎麼流轉的正本在 [[訂單狀態]]，本卡只引用、不重述。

## 營運背景

訂單是客戶答應下單、公司開始收錢與動工的依據。它的來路有三種：業務報價談成的「一般訂單」、客戶先付諮詢費談需求後談成或收尾的「諮詢訂單」、客戶自己在線上前台下單的「線上訂單」。三種來路後續走的狀態路徑略有不同（例如線上訂單有客戶付款時間、線下訂單有報價回簽時間），但都是同一張訂單，只是「從哪裡來的」這項標記不一樣。把它獨立成一張核心單據，是因為從收款、排生產、開發票到出貨，全公司都圍著這張單轉，必須有一個能被各環節共同指認的對象。

## 需要解釋的資料

訂單上要記哪些欄位，完整清單以下方 `implemented-by` 指向的 OpenSpec 規格 § Data Model: Order 為正本，本卡不重抄。以下只挑「不看說明會誤解、或藏有業務理由」的幾項解釋：

- **訂單來路**：分一般 / 諮詢 / 線上（電商）三種，後續走的狀態路徑略有不同（例如線上單有客戶付款時間、線下單有報價回簽時間），但都是同一張訂單，只是「從哪裡來的」這項標記不一樣（技術欄位：`order_type`，線上值為 `EC`）。
- **帳務公司**：發票要開給誰、用哪家公司名義開；從需求單帶過來，訂單成立後不能改，避免成交後被改掉（技術欄位：`billing_company_id`）。
- **收款條件備註**：從需求單帶過來並鎖定，避免成交後收款條件被偷改；畫面上會標明「（來自需求單）」（技術欄位：`payment_terms_note`）。
- **來源需求單 / 來源諮詢單**：用來反查這張訂單是從哪張需求單成交、或哪次諮詢收尾來的（技術欄位：`linked_quote_request_id` → [[需求單]]、`linked_consultation_request_id` → [[諮詢單]]）。
- **三個依內容主題分的備註欄位（訂單須知 / 交貨備註 / 付款備註）**：業務或諮詢在訂單階段補充、要給客戶看的標準化條款，與按來源分的另一組備註並存——兩組備註怎麼分工，見下方「訂單備註資料（兩種分法並存）」。

## 相關單據

- 一張訂單底下會有好幾件印件（要做的每件印刷品）→ [[印件]]
- 每件印件底下又會有好幾張工單（印件實際排去產線的生產單）→ [[工單]]
- 一張訂單會有好幾張訂單異動單：訂單成立後要加錢或退錢，就靠訂單異動單來記；規則見 [[訂單異動規則]]、狀態流轉見 [[訂單異動狀態]]
- 一張訂單會有好幾張發票和折讓單（開給客戶的發票與折讓）→ [[付款發票邏輯]]
- 一張訂單會有好幾筆收款紀錄和分期請款（實際收到的錢與分期請款計畫）→ [[分期請款狀態]]
- 一張訂單會有好幾張出貨單（這張訂單的出貨）→ [[出貨單]]
- 訂單完成後可能會有售後服務單（完成後的客訴 / 不良 / 規格不符受理）→ [[售後服務]]

**互相牽連**：這兩件事互相牽連——訂單異動一發生，這張訂單「客戶總共要付多少」就會變。所以日後動到訂單金額相關的地方，要一起對照 [[訂單異動規則]]；反過來，異動規則改了，訂單上金額怎麼算、怎麼顯示也會跟著受影響。

## 訂單備註資料（兩種分法並存）

一張訂單上有兩組備註，是從兩個不同角度分的：

- **既有「訂單備註三類」**（正式規格見 [order-management/spec.md § Data Model](../../../../openspec/specs/order-management/spec.md)）按「來源 / 看得到看不到」分：
  - 客戶填的備註（只有線上單）— 客戶自己在線上前台下單時填，我們不能改
  - 內部備註（線上線下都有）— 只有公司內部看得到，客戶看不到
  - 製作備註（只有線下單）— 把訂單製作 / 交易 / 出貨的備註整理在一起
- **另一組「訂單備註區」**（後來新增，2026-05-20）按「內容主題」分成三項：
  - 訂單須知 / 交貨備註 / 付款備註 — 業務或諮詢在訂單階段補充、要給客戶看的標準化條款，後續會匯出到報價單 / 訂單確認單帶出（[[ORD-012-訂單備註匯出至客戶文件路徑]]）
  - 編輯時機：訂單完成前可編輯；訂單完成 / 取消後鎖定為唯讀
  - 可以編輯的人：負責這張訂單的業務、諮詢人員、業務主管、訂單管理人

這兩組備註各管什麼、怎麼分工，見 [[ORD-013-新三類與既有production_note職責分工]]；備註編輯為什麼用獨立的編輯視窗（不沿用訂單資訊編輯視窗）見 [[ORD-014-訂單備註與訂單資訊編輯dialog分開]]。
