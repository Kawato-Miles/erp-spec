# 訂單（Order）

> 一句話定位：訂單是客戶確認下單後，公司據以收款、生產、出貨的核心單據，橫跨報價、生產、款項、出貨多個環節。依**來路**分類型，現階段支援：一般訂單（需求單成交）、諮詢訂單（諮詢收尾）、線上訂單（客戶在線上前台自己下單，電商網站）；未來將有更多類型。
> 這張卡負責什麼：完整記錄一張訂單上業務使用者看得到的所有欄位（見「訂單欄位」段）、會牽連到哪些單據、訂單會有哪些狀態名稱。欄位表是正本——欄位的新增與修改是商業決策，直接在此卡維護。訂單成立後金額怎麼增減的規則正本在 [[訂單異動規則]]、款項與發票對帳的規則正本在 [[付款發票邏輯]]、狀態怎麼流轉的正本在 [[訂單狀態]]，本卡只引用、不重述。

## 這張卡要回答的問題

- 訂單有哪些來路？資料上怎麼分？→ 看「營運背景」和「訂單欄位」裡的「訂單類型」。
- 一張訂單上會記哪些資料？每個欄位業務看到什麼、能不能改？→ 看「訂單欄位（業務可見）」完整欄位表。
- 哪些資料比較不直覺、需要解釋為什麼要記下來？→ 收款條件備註、來源諮詢單編號、兩組備註資料，見欄位表 + 下方「訂單備註資料」。
- 訂單會牽涉到哪些單據？→ 見「相關單據」（印件、工單、發票、折讓單、收款紀錄、出貨單、售後服務單）。
- 訂單有狀態嗎？在哪張卡？→ 有，在 [[訂單狀態]]，見「相關狀態機」。

## 營運背景

訂單是客戶答應下單、公司開始收錢與動工的依據。它的來路現階段有三種：業務報價談成的「一般訂單」、客戶先付諮詢費談需求後談成或收尾的「諮詢訂單」、客戶自己在線上前台下單的「線上訂單」；未來將支援更多類型。各來路後續走的狀態路徑略有不同（例如線上訂單有客戶付款時間、線下訂單有報價回簽時間），但都是同一張訂單，只是「從哪裡來的」這項標記不一樣。把它獨立成一張核心單據，是因為從收款、排生產、開發票到出貨，全公司都圍著這張單轉，必須有一個能被各環節共同指認的對象。

### 訂單額外費用明細

> 訂單建立時即確定的非印件費用項目（運費、急件費、諮詢費等）。與「訂單異動」的差別：額外費用是訂單明細的一部分、不需審核；異動是訂單成立後的金額增減、需走審核。

| 介面欄位 | 說明 | 欄位形態 | 可否修改 |
|---------|------|---------|---------|
| 費用類型 | 諮詢費 / 運費 / 急件費 / 其他 | 單選 | 訂單完成前可改 |
| 金額 | 該筆費用金額 | 金額 | 訂單完成前可改；完成後鎖定 |
| 說明 | 費用描述（如「諮詢費（諮詢單編號 CR-XXX）」）| 文字 | 訂單完成前可改 |

## 相關單據

- 一張訂單底下會有好幾件印件（要做的每件印刷品）→ [[印件]]

- 一張訂單會有好幾張發票和折讓單（開給客戶的發票與折讓）→ [[付款發票邏輯]]

- 一張訂單會有好幾張出貨單（這張訂單的出貨）→ [[出貨單]]

## 訂單備註資料（兩種分法並存）

一張訂單上有兩組備註，是從兩個不同角度分的：

- **既有「訂單備註三類」**（正式規格見 [order-management/spec.md § Data Model](../../../../openspec/specs/order-management/spec.md)）按「來源 / 看得到看不到」分：
  - 客戶填的備註（只有線上單）— 客戶自己在線上前台下單時填，我們不能改
  - 內部備註（線上線下都有）— 只有公司內部看得到，客戶看不到
  - 製作備註（只有線下單）— 把訂單製作 / 交易 / 出貨的備註整理在一起
- **另一組「訂單備註區」**（後來新增，2026-05-20）按「內容主題」分成三項：
  - 訂單須知 / 交貨備註 / 付款備註 — 業務或諮詢在訂單階段補充、要給客戶看的標準化條款，後續會匯出到報價單 / 訂單確認單帶出（[[ORD-012-訂單備註匯出至客戶文件路徑]]）
  - 編輯時機：訂單未取消即可編輯（含訂單完成後）；已取消後鎖定為唯讀
  - 可以編輯的人：負責這張訂單的業務、諮詢人員、業務主管、印務主管（原訂單管理人；對訂單內容的編輯範圍同業務）

這兩組備註各管什麼、怎麼分工，見 [[ORD-013-新三類與既有production_note職責分工]]；備註編輯為什麼用獨立的編輯視窗（不沿用訂單資訊編輯視窗）見 [[ORD-014-訂單備註與訂單資訊編輯dialog分開]]。
