# 用語規範（共用）

> 三視角審查 Agent 必讀。本卡取代三個 agent.md 內各自重述的語言規範段。

## 一、主語言

- 所有輸出 **MUST** 使用 **台灣繁體中文** 用詞
- 避免大陸用語（PRC 用語）
- 字句通順、避免機械翻譯感

## 二、台灣 vs 大陸用語對照

| 大陸用語（避免） | 台灣用語（採用）|
|---------------|---------------|
| 項目 | 專案 / 品項（依語境）|
| 對話框 | 對話視窗 |
| 用戶 | 使用者 |
| 文件 | 檔案（file）/ 文件（document，依語境）|
| 登錄 | 登入 |
| 優先級 | 優先順序 |
| 默認 | 預設 |
| 緩存 | 快取 |
| 信息 | 訊息 / 資訊 |
| 視頻 | 影片 |
| 軟件 | 軟體 |
| 網絡 | 網路 |
| 程序 | 流程 / 程式（依語境）|
| 移動設備 | 行動裝置 |

## 三、技術術語維持英文

技術名詞維持英文原文，**MUST NOT** 翻譯成中文：

- `webhook` / `Payment` / `OrderAdjustment` / `Invoice` / `Phase`
- `API` / `FK` / `UUID` / `PRD` / `BRD` / `OQ`
- `BOM` / `WIP` / `PO` / `MRP`

**MUST NOT** 把英文直接當作中文句子的主詞、形容詞、主動詞。需要描述時用括號附註：
- 反例：「沉澱的 reusable pattern」
- 正例：「累積出可重複使用的設計範式（reusable pattern）」

## 四、避免大陸技術用語

| 大陸技術用語 | 台灣業界慣用 |
|------------|------------|
| 暴露 API / store | 對外提供 API / store、將 ... 外露 |
| 全綠 | 全部通過 / 無失敗項目 |
| 拋錯 / 噴錯 | 拋出例外 / 顯示錯誤 |
| 報錯 | 可用，但避免口語感 |

## 五、PM / 非工程受眾的語境

對 PM、業務、老闆說明時 **MUST** 優先業務術語，工程內部 jargon 須轉譯：

- 反例（對 PM）：「e2e 全綠」「resetStore 沒清乾淨」
- 正例（對 PM）：「端對端測試 15 項全部通過」「測試重置邏輯沒清理乾淨上一輪殘留」

## 六、CLI 指令 / 旗標 / 技術代號

CLI 指令、旗標、技術代號 **MUST** 完整描述上下文，**MUST NOT** 直接夾在中文句子中當成詞彙：

- 反例：「緊急可 `--no-verify` 跳過」
- 正例：「緊急情況可使用 `git push --no-verify` 旗標跳過此檢查（不建議）」

## 七、不使用 Emoji / 視覺符號

所有審查輸出 **MUST NOT** 使用 emoji 或視覺符號（如 ✅、❌、🔴）。表達是非用文字（「正確」「錯誤」「風險」「不建議」）或標準 Markdown 符號（如 `→`）。
