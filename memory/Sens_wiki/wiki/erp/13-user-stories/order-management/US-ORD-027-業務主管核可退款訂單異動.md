# US-ORD-027 業務主管核可退款訂單異動

> 一句話定位：本卡是「業務主管」執行「退款送審」這條規則的操作步驟（單一角色單一情境）——業務主管收到退款異動單時怎麼一步步審核、核可、確認退款到位。退款送審的判斷與認列規則本體見 source 指向的 [[訂單異動規則#退款送審]]，本卡只管業務主管怎麼做。

## 這張卡要回答的問題

- 業務主管要審核哪些訂單異動單、哪些不用審？ → 結論見「成功條件第 1 條」
- 業務主管核可後，退款異動單何時才算「已執行」？實際退錢又算哪一步？ → 結論見「業務流程第 4-5 步」與「成功條件第 2 條」（核可即已執行，實際退錢另在對帳核銷）
- 退款的錢要不要算進原本客戶分期收款的紀錄？ → 結論見「成功條件第 3 條」
- 業務主管多久內要回應一張退款異動單？ → 結論見「成功條件第 4 條」

## 業務情境

### 作為
[[業務主管]]

### 我希望
專心審核「要退錢給客戶」的退款訂單異動單，不再因為業務調整客戶的收款安排（分期、改收款日）就被叫回來重審

### 以便
把關責任聚焦在「公司把錢退出去」這個方向；業務調整客戶收款安排不再需要事前找主管批准，減少對主管的打擾。要退錢給客戶仍由主管核可，要向客戶多收錢（補收）則由業務自己直接執行——兩個方向刻意不對稱，因為退錢的風險由主管把關、多收則是客戶下單追加的延伸。

### 前置條件
- 業務已建立要退錢給客戶的訂單異動單（金額為負）並送出審核
- 業務主管有這張訂單的核可權限

### 業務流程
1. 業務主管在「待主管審核」清單看到這張退款訂單異動單
2. 業務主管檢視退款原因、退款明細、責任歸屬（是公司的責任還是客戶的責任）
3. 業務主管決定核可，或退回要求業務修正
4. 業務主管一核可，系統當下就把這張退款異動單標示為「已執行」、執行時間記為核可時點、訂單應收當下減去這筆退款金額——不必等實際退錢（依 [[訂單異動規則#退款已執行認列]]）
5. 之後業務真的把錢匯出去，在款項紀錄建一筆退款款項（記負數）、補上實際退款日期與匯款證明、標示為「已完成」，在對帳階段把「應退差額」核銷歸零；這一步不會回頭改異動單狀態（異動單維持已執行）
6. 業務後續再處理發票：同一個月內的就把原發票作廢、重開；已經跨月的就開折讓單

### 成功條件（這張單做到什麼程度才算對）

1. 只有「要退錢給客戶」的訂單異動單（金額為負）需要業務主管核可；「向客戶多收錢」的補收（金額為正）由業務直接執行，不送主管
2. 業務主管一核可，退款訂單異動單當下就推進到「已執行」、應收當下減去這筆退款金額，不需先把錢退出去（認列跟著主管核可走，與實際退錢脫鉤）；實際退好沒由對帳的「應退差額」盯住——退款款項標成「已完成」且帳能對平，應退差額才歸零
3. 退款不會去動到客戶原本一期一期收款的對帳紀錄，原本的分期收款紀錄保持完整可供日後查核
4. 業務主管最好在 8 小時內處理完一張退款異動單。老闆會用這個 8 小時來看一個指標（指標 7）：免審的補收上線後，退款這邊會不會因為要等主管，反而變成新的塞車點
