# US-ORD-026 業務新增補收的訂單異動單免主管核可、直接生效

> 一句話定位：本卡是「[[業務]]」執行「補收免審規則」的操作步驟（這張卡只講一個人、一種狀況：業務遇到客戶加買要多收錢時怎麼做）——客戶在製作中加買、要多付錢時，業務建一張要跟客戶多收錢的訂單異動單，建好就直接生效，不必等業務主管核可。完整規則（什麼情況會觸發、要不要送審、收到的錢怎麼認）見這張卡往上指的依據 [[訂單異動規則#補收免審]]，本卡只管「業務怎麼一步步做到」。

## 這張卡要回答的問題

- 業務建一張要跟客戶多收錢的訂單異動單時，要不要先送業務主管核可？ → 結論見「業務流程」第 3-4 步與「成功條件第 1 條」
- 多收的這筆錢要等實際收到才算進應收嗎？ → 結論見「成功條件第 2 條」
- 多收的金額很大時，業務主管怎麼知道？ → 結論見「成功條件第 3-4 條」

## 業務情境（穩定層）

### 作為
[[業務]]

### 我希望
建一張要跟客戶多收錢的訂單異動單時建好就直接生效、不必等業務主管核可，讓多收的這筆錢當場加進應收

### 以便
客戶加買是送上門的生意，等主管核可會拖到交期、甚至讓客戶改變主意；多收錢這個方向風險低，由業務當場拍板就好

### 前置條件
- 這張訂單是這位業務負責、可以動的
- 這筆訂單異動是要跟客戶多收錢的方向（金額大於 0、屬於加印追加／加運費／急件費／規格變更要多收的部分／之前退太多、現在要補收回來的部分）
- 訂單仍在製作中（訂單完成後要多收錢改走售後服務單）

### 業務流程
1. 業務在訂單的訂單異動處新建一張要跟客戶多收錢的訂單異動單
2. 業務填入要多收的金額、原因與明細
3. 業務直接送出，不必經過等主管審核、主管核可這兩道關
4. 依 [[訂單異動規則#補收免審]]，這張單子不必送審，建好就直接生效，系統會把核可人記成業務本人（等於業務自己拍板，不另外找主管），多收的這筆錢當場加進訂單應收
5. 業務之後再開一筆請款期次（這次要跟客戶收這筆錢的分期單），把這筆多收的錢掛進去，再依這筆期次去開發票、跟客戶收款
6. 若這筆多收的金額超過大額門檻（建議起始 5 萬元），系統會在這張訂單的操作紀錄裡用紅字標一筆，並通知業務主管事後查看（屬事後監督，不擋業務操作）

### 成功條件（驗收標準）

1. 這張要跟客戶多收錢的訂單異動單不必經過等主管審核、主管核可這兩道關，建好就直接生效
2. 多收的這筆錢當場加進訂單應收，不必等實際收到才算進去（與退款方向相反——退款要等實際把錢退給客戶才算數）
3. 多收的金額很大時（超過大額門檻，建議起始 5 萬元），系統會在這張訂單的操作紀錄裡用紅字標一筆，並通知業務主管事後查看
4. 這筆大額多收的通知屬事後監督、不擋業務當場操作（業務可以馬上建單生效，業務主管事後從訂單操作紀錄或通知查看這筆金額偏大的多收）

### 介面入口
- 訂單詳情頁 → 訂單異動區塊 → 「新增異動」按鈕

### 操作步驟
- 在新增異動對話框選要跟客戶多收錢的類型、填金額與原因明細，按「直接執行」就生效

### 介面元素
- 新增異動對話框顯示綠色提示橫幅「補收正項 — 直接執行（免主管核可）」（橫幅原文照畫面，畫面上的「補收正項」就是指這筆是要跟客戶多收的錢，可直接生效、免主管核可）
- 多收金額很大時顯示黃橘色警示橫幅，提示「業務主管將收到事後通知」
