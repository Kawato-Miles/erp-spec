# US-ORD-013 會計執行退款處理

> **新增紀錄（2026-05-22）**：本卡為 [[US-ORD-011-訂單取消與退款]] 拆分新增；依「user story 單角色單動作」紀律，業務動作（取消 + 退款申請）與會計動作（執行退款 + 折讓）拆為兩張 user story。

## 業務情境

### 作為
[[業務]]

### 我希望
能建立退款付款紀錄、上傳匯款證明後切「已完成」以核銷對帳應退差額；退款已跨月時於原發票手動開立折讓單沖減發票淨額

### 以便
退款依匯款證明逐筆留痕，對帳三軸（應收 / 發票淨額 / 收款淨額）回平

### 前置條件
- 訂單存在對帳應退差額（收款淨額 > 應收）：來源可能為訂單完成後退款訂單異動經業務主管核可生效，或訂單完成前明細減量致應退
- 操作者已登入系統

### 業務流程

1. 退款情境來源二分：
   - 訂單完成後售後退款：業務於售後服務單內建立退款訂單異動（金額為負），送業務主管核可，核可即應收調整生效（已執行）
   - 訂單完成前明細減量致應退：無關聯訂單異動，退款付款紀錄的關聯訂單異動（linkedOrderAdjustmentId）留空
2. 操作者於訂單異動介面或對帳面板建立退款付款紀錄（金額為負、付款方式為退款），預設狀態「處理中」
3. 執行銀行匯款（系統外動作）
4. 上傳匯款證明附件並填寫款項實際完成日，將退款付款紀錄切「已完成」——此為退款完成的物理錨點
5. 系統核銷對帳應退差額、收款淨額減少；退款付款紀錄不建立入帳明細（不進正向期次）
6. 多筆退款各自挂匯款證明、各自切「已完成」，帳平以應退差額歸零判定，不逐筆勾稽訂單異動
7. 已開立發票之沖減：未跨月作廢原發票重開；已跨月於原發票手動建立折讓單並關聯該退款付款紀錄
8. 系統寫入活動紀錄（事件描述：退款執行 + 操作者姓名 + 款項實際完成日 + 退款金額）

### 成功條件

1. 操作者可建立退款付款紀錄（金額為負、付款方式為退款），預設「處理中」、不計入收款淨額
2. 退款付款紀錄切「已完成」前必須已附匯款證明附件且填寫款項實際完成日，否則被擋
3. 退款付款紀錄切「已完成」後對帳收款淨額減少、應退差額歸零（退款完成判定）
4. 已完成退款付款紀錄不可改回「處理中」；誤建只能取消後重建
5. 取消已完成退款付款紀錄後對帳應退差額重現，關聯退款訂單異動維持「已執行」不回退
6. 已跨月之退款須於原發票手動開立折讓單並關聯該退款付款紀錄，使發票淨額同步減少、三軸回平
7. 退款執行寫入活動紀錄供事後稽核（含時間 / 操作者 / 退款金額）
