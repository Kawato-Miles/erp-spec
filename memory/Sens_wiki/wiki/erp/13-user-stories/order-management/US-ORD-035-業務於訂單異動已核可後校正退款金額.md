# US-ORD-035 業務於訂單異動已核可後校正退款金額

> 一句話定位：這張卡講的是——[[業務]]在「退款已經核可、錢還沒退出去之前要改金額」這種情況下，一步一步怎麼把金額改對。什麼狀態能改、改了要不要再送主管審一次，完整規則寫在 [[訂單異動規則|訂單異動規則]] 那張，這張只講業務一步一步怎麼把金額改對。

## 這張卡要回答的問題

- 退款訂單異動在主管已核可後校正金額，需不需要重新送主管審核？ → 結論見「成功條件第 1 條」
- 哪個階段之後金額就不能再改？ → 結論見「成功條件第 3 條」
- 諮詢取消系統自動建的退款異動，業務能不能比照校正？ → 結論見「成功條件第 4 條」

## 業務情境

### 作為
[[業務]]

### 我希望
在退款[[訂單異動規則|訂單異動]]已取得業務主管核可、但款項尚未實際退出去之前，校正這筆退款的金額而不必重新送主管審核

### 以便
退款前若客戶與公司就退多少達成新的共識（例如折讓比例談定、運費爭議釐清），業務能即時把金額校正到正確值，不必為了一個主管已原則同意的退款重跑一輪審核而拖延退款時程；同時保留「主管核可金額 vs 當前金額」對照供事後監督，一次校正就把金額改到對的數字，不用反覆來回改

### 前置條件
- 這筆訂單異動是退款（系統記成負數），且已由[[業務主管]]核可、這筆異動就變成「已核可」
- 這筆退款實際還沒退給客戶（退款的款項狀態還沒變成「已完成」，這筆異動也還沒算「已執行」）
- 改正後仍然是退款（系統記成負數）

### 業務流程

1. 退款訂單異動經[[業務主管]]核可後停在「已核可」，等實際退款款項退出去
2. 退款前業務與客戶就退款金額達成新共識（金額需校正）
3. 業務直接校正該退款訂單異動的金額，不必重新送主管審核，狀態維持「已核可」（依 [[訂單異動規則#金額編輯閘門]]，已核可可改且不重審）
4. 系統保留「主管核可金額 vs 當前金額」對照，並留下這次改金額的紀錄：什麼時候改的、誰改的、改之前多少、改之後多少、改的時候是什麼狀態，供[[業務主管]]事後監督
5. 業務之後建立實際退款付款紀錄並把款項狀態改成「已完成」，累計退出去的金額達到改正後的金額時，系統就把這筆退款異動標示為「已執行」、正式從這張訂單的應收金額裡扣掉

### 成功條件

1. 退款訂單異動在「已核可」狀態下，業務校正金額後狀態維持「已核可」，不退回「待主管審核」、不需重新送審
2. 每次改金額都會留下紀錄：什麼時候改的、誰改的、改之前多少、改之後多少、改的時候是什麼狀態，且「主管核可金額 vs 當前金額」對照可被[[業務主管]]事後查核
3. 校正僅限「已核可且尚未執行」；訂單異動已推進「已執行」後金額鎖定、業務不可再校正
4. 改正後實際退款付款紀錄的款項狀態變成「已完成」、累計退出去的金額達到改正後的金額時，系統就把這筆退款異動標示為「已執行」，並從這張訂單的應收金額裡扣掉（依改正後的金額）（諮詢取消時系統自動產生的那筆退款異動，業務一樣可以用這個方式改金額，規則相同）
