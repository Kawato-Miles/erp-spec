# US-AR-007 執行印件審稿

## 業務情境

### 作為
[[審稿人員]]

### 我希望
能完成單一印件單輪審稿（合格 / 不合格判定），並在送出後修正本輪審稿備註

### 以便
- 每輪審稿留結構化紀錄取代過去散落在即時通訊軟體的溝通與紙本簽收，所有決策可稽核
- 不合格率 KPI 排除技術性退件後反映真實品質問題（技術退件分類獨立統計，不污染品質 KPI 分母）
- 送出後可修改備註讓糾正能即時傳到下游印務與補件方，不必走額外流程

### 前置條件
- 印件原稿已由補件方（B2B 業務或 B2C 會員）上傳
- 系統已將該印件分派給該審稿員
- 印件審稿維度狀態為「等待審稿」或「已補件」

### 業務流程

1. 審稿員查看分派給自己的待審印件清單（待審清單預設排序規則與急單標記機制詳見 [[AR-5-待審清單預設排序與急單標記]]）
2. 審稿員選取要審稿的印件，檢視：
   - 印件規格（紙張 / 尺寸 / 印色 / 加工等）
   - 原稿（補件方上傳的印件檔；審稿員不可替換此檔案）
   - 稿件備註（補件方建立印件時填寫的說明）
   - 歷史輪次紀錄（每輪結果 / 退件原因 / 審稿備註 / 時間，**含該輪三類檔案：印件檔 / 審稿後檔案 / 縮圖 三欄分開顯示**）
   - 本輪送審時補件方的補件說明（若有）
3. 審稿員下載原稿進行加工（系統外處理，系統不追蹤加工過程，僅在步驟 4 收回加工後檔案）
4. 審稿員判定本輪結果：
   - **合格**：上傳審稿後檔案（加工後版本，與原印件檔並存供稽核軌跡）與縮圖；可選填審稿備註留言給後續角色（如印務 / 補件方）。系統將印件推進至製程審核階段。
   - **不合格**：自退件原因 10 項分類擇一（出血不足 / 解析度過低 / 色彩模式錯誤 / 缺少必要元素 / 版面超出安全區 / 尺寸不符 / 特殊工藝圖層異常 / 字型未外框 / 技術性退件 / 其他），並填寫審稿備註補充說明。系統通知補件方。其中「技術性退件」（如檔案損毀 / 字型缺失）獨立統計，不計入不合格率 KPI。
5. 系統建立本輪審稿輪次紀錄（檔案 / 結果 / 退件原因 / 備註 / 時間），並依結果推進印件狀態
6. 審稿員於送出後仍可回頭修改本輪審稿備註（含合格 / 不合格輪次皆可修改，如糾正錯字或補充說明），每次修改留稽核紀錄；若補件方此時正在補件頁停留，系統同步通知補件方備註已更新。**合格輪次的審稿備註修改不影響已建工單的檔案鎖定範圍**（檔案鎖定只含加工檔與縮圖；備註不在鎖定範圍內，糾正可即時傳至下游印務）
7. 合格判定送出後本印件審稿終止，本 user story 結束；後續若需改動印件內容，由跨模組情境流程處理（詳見 [[wiki/erp/07-scenarios/README#情境 6：同印件補件後再審]]，業務情境 spec），不在本 user story 範疇

### 成功條件

1. 審稿員可在待審清單看到分派給自己的所有印件（非分派給自己的，於個人待審清單不可見；但在「依訂單檢視待審」展開訂單時，同訂單其他人負責的印件唯讀可見，供審稿員掌握整張訂單脈絡）
2. 審稿員開啟單一印件後可查看：印件規格 / 原稿 / 稿件備註 / 歷史輪次（含每輪三類檔案三欄分開顯示）/ 本輪補件說明
3. 合格判定須上傳審稿後檔案與縮圖（兩者皆必填），可選填審稿備註；提交後印件推進至製程審核階段
4. 不合格判定須選退件原因（10 項分類擇一）並填審稿備註補充說明；系統自動通知補件方；技術性退件分類獨立統計不計入不合格率 KPI
5. 審稿員可於送出後修改本輪審稿備註（含合格 / 不合格輪次），每次修改留稽核紀錄並通知補件頁停留中的補件方；合格輪次修改不影響已建工單的檔案鎖定範圍
