# 師傅

> [!info] OpenSpec spec 狀態：**缺漏**
> `openspec/specs/user-roles/spec.md` 中無此角色，需後續評估是否補建。詳見 [[_alignment-report]]。

## 基本資料

- **平台**：工廠平台
- **利害關係程度**：高
- **參與階段**：打樣印製
- **任務操作**：R/W
- **報價單 / 訂單**：X
- **工單**：X
- **需求單**：X

## 主要工作職責

執行生產任務

## 關切點

- 工單指示清楚
- 異動即時通知
- 師傅需能快速理解工單內容

## 預期阻力（**重要 risk**）

- 電腦報工的操作習慣改變，老師傅尤其抗拒
- **事後補輸或完全不輸的風險是產能可見度 95% 目標的最大障礙**，需要管理層明確政策支持

## 備註

僅可編輯任務進度回覆，不可更改任務內容

## 關鍵風險

師傅報工是 Phase 2 北極星指標達成的關鍵環節（見 [[success-metrics]]）。設計報工功能時必須優先考量：
- UI 極簡
- 行動裝置友善（依 [[PT-001-師傅報工行動裝置例外]] OQ 決議；現行 ERP 政策為桌機限定）
- 預設值 / 一鍵填入降低操作門檻
