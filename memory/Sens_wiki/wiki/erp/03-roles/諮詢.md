# 諮詢

## 基本資料

- **平台**：業務平台
- **利害關係程度**：高
- **參與階段**：評估 / 訂購 / 審稿 / 打樣印製 / 出貨（全程）
- **任務操作**：X
- **報價單 / 訂單**：R/W
- **工單**：X（僅查閱狀態、QC 紀錄）
- **需求單**：R/W

## 主要工作職責

Sales process（售前評估、成本評估、需求單管理、線下訂單管理）；EC 客服問題處理

## 關切點

- 報價流程順暢
- 訂單狀態可追蹤
- 補單或規格變更有記錄
- 客戶改稿後的追加費用可在系統補記，避免漏收

## 預期阻力

- 補單與規格異動是日常常態，系統化後需輸入更多資訊，可能被認為增加操作負擔
- 改稿追加費用漏收的責任歸屬，可能引發內部核算摩擦

## 備註

- 需可「分享」單據給其他業務 / 諮詢（職務代理 / 檢視，不改負責人）；「轉派負責業務」（改 owner）收歸業務主管，諮詢比照業務不可自行轉派（add-sales-manager-reassign-owner change 2026-05-30 收斂：分享 ≠ 轉派）
- 工單僅查閱狀態、QC 紀錄
- 售後 ticket 模組：諮詢 SHALL 具備與業務角色相同的權限與操作能力（建單 / 送出決議 / 結案 / append 補述 / 修改 case_category / responsibility / resolution / 貼 Slack URL），諮詢繼承業務權限原則（add-my-after-sales-action-page-and-remove-owner-transfer change 顯式化）
- 訂單異動相關動作等價於[[業務]]（建立補收 / 退款異動單、於售後 ticket 內加掛關聯異動單），補收免審 vs 退款送審的判斷準則正本見 [[訂單異動規則]]（諮詢繼承業務權限的顯式化）

## 與業務角色的差異

「諮詢」與[[業務]]的主要差異在於：
- **諮詢**：負責 EC（線上電商）客服問題處理 + 售前評估
- **業務**：負責線下訂單管理 + 款項、發票處理

兩者在系統權限與職責上大部分重疊。
