## 三、order-fields-reference.md（200 行欄位對照表）摘要

完整內容見 `../../../decks/order-fields-reference.md`。**關鍵摘要**：

### 訂單類型與來源

| 維度 | 值 |
|------|---|
| `order_type` | 一般 / 諮詢 / 點數 |
| `order_source` | 線下 / 線上 / 線上自定義 |

### 線上單 vs 線下單行為差異重點

| 欄位 | 線上單 | 線下單 |
|------|--------|--------|
| 訂單狀態 | 自動更新（付款完成觸發推進） | 手動更新（業務確認回簽）|
| 案名 | 自 EC 商品名帶入 | 自需求單名稱帶入 |
| 客戶交期 | 不適用（以審稿後預計出貨日為準） | 業務手動填寫 |
| 審稿前 / 後預計出貨日 | 系統自動計算 | 不適用 |
| 諮詢費 | 不適用 | 手動填寫（諮詢轉訂單）|
| 紅利折抵 | 自動帶入 EC 紅利 | 不適用 |
| 稿件上傳開放 | 訂單確認後自動開放 | 業務手動開放 |

→ 完整 200 行欄位對照見 decks 原檔。

## 四、print-stage-changes.md（400 行認知對齊簡報）摘要

完整內容見 `../../../decks/print-stage-changes.md`。**核心觀念**：

### 三大變化

1. **EC / Ragic 工單整合**：線上單與線下單從兩套系統合併到 ERP 一套
2. **工單層級變化**：4 層（訂單 → 商品 → 工單 → 工序）→ 6 層（訂單 → 印件 → 工單 → **任務** → **生產任務** → 紙本工單）
3. **報工與齊套性規則**：數量統計從以工單為單位改為**以印件為單位**

### 6 層架構（新）

```
訂單 → [[印件]] → [[工單]] → [[任務]] → [[生產任務]] → 紙本工單
```

### 關鍵新概念

- **每份印件需生產數量**：見 [[數量換算規則]]
- **不影響成品標記**：採購類生產任務不參與齊套性計算（[[齊套邏輯]] § `affects_product`）
- **任務層按廠商自動分**：[[任務]] § 為什麼需要任務層
- **紙本工單可合併**：同製程內容的多印件樞紐成一張紙本

### 三方影響速覽

| 角色 | 影響 |
|------|------|
| [[印務]] | 不再手動搬到 Ragic、一印件最多 2 張工單、廠商自動分流 |
| [[生管]] | 派工單位從「工單」改為「任務」、紙本工單可合併 |
| [[師傅]] | 必須填寫報工紀錄、可分多次報工、用生產任務數填（不是印件數） |

### 報價建議

業務 / 印務報價時：
- 用**實際數量**取代「1 式」
- 打樣 / 大貨**分開建印件**
- 各別出貨需求時**分開建印件**

→ 對應 [[報價邏輯]] § Phase B 補完。
