# ERP Phase 規劃

> Phase 會持續迭代擴充，不鎖死數量。以下為目前已定義的階段。

## Phase 1：商品資料基礎（進行中）

**核心任務**：建立共用 BOM 表，讓後續 Phase 進入 MES 時工單可基於 BOM 建立。

**背景**：過往 EC 商品是獨立的資料結構，工單內容攏統，無法追蹤到實際製作狀況。

**範疇**：
- EC 規格品 BOM 建立（含工序 + 材料 + 機台設備）
- 商品成本試算
- 業務情境驗證

**北極星指標**：EC 規格品 BOM 覆蓋率 ≥ 80%（已完成 BOM 建立的 EC 現行規格品數 / EC 現行販售規格品總數）

**結束標準**：
- BOM 覆蓋率達標
- 設定方式無預期外障礙
- 商品管理可完整移轉至 ERP

## Phase 2：線下單全流程走通（進行中）

**核心使命**：取代 Ragic + Slack + 紙本組合。

**背景**：線下單是全客製化訂單、情境最複雜。先讓線下單全流程可通，後續才把 EC 規格品訂單整併進來。

**範疇**：
線下單從頭到尾的完整流程系統化——諮詢/需求單 → 報價 → 訂單審核 → 審稿 → 生產與報工 → 品檢 → 出貨 → 收款 → 售後。

**北極星指標**：訂單流程完整完成率
- 上線後第 1 個月：≥ 60%
- 上線後第 3 個月：≥ 80%

**定義**：訂單從起點（線下訂單：「訂單確認」）到「出貨完成」的完整流程均在系統內完成的訂單比率。

**為什麼選這個指標**：
1. 走完完整流程會強制讓所有環節角色真正進入 ERP 完成業務
2. 出貨完成是公司服務流程兌現的時刻，若未達到此節點代表無法滿足服務流程

## Phase 3：EC 訂單整合進 ERP（未開始）

**核心任務**：將 EC 線上訂單整併進 ERP，與線下單共用同一套流程管理。

**背景**：Phase 2 線下單流程走通後，EC 規格品訂單（較簡單）才整併進來，實現線上/線下統一。

**範疇**：
- EC 訂單自動建立流程
- EC 與線下單共用訂單管理、生產、出貨流程
- EC 規格品與 BOM 的對接（承接 Phase 1 成果）

**北極星指標**：同 Phase 2 訂單流程完整完成率（擴大分母至含 EC 訂單）
