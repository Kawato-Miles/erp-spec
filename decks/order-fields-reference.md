# 訂單欄位對照表（線上單 vs 線下單）

**版本**：2026-05-07（依 refactor-order-payment-and-invoice-with-billing-company change 更新）
**用途**：供同仁確認訂單系統各欄位在線上單（EC）與線下單的行為差異

---

## 說明

### 訂單類型與來源對照

| 欄位 | 說明 |
|------|------|
| **訂單類型（order_type）** | 一般 / 諮詢 / 點數（依訂單內容分類）|
| **訂單來源（order_source）** | 線下 / 線上 / 線上自定義（依建立方式分類）|

本文件以「**線下單**」與「**線上單**」兩欄呈現行為差異。其中「線上單」涵蓋 EC 一般線上單與線上自定義單（自定義單另有特殊備注說明）。

### 欄位值標示說明

| 標示 | 意思 |
|------|------|
| 自動帶入 | 系統從來源資料自動填入，無需人工操作 |
| 自動計算 | 系統依規則計算產生 |
| 自動更新 | 流程推進時系統自動變更（如付款完成後狀態自動更新）|
| 手動填寫 | 需由人員填入 |
| 手動控制 | 由人員手動觸發開關或操作 |
| 手動更新 | 由人員手動確認後更新 |
| 唯讀 | 系統計算，僅供檢視，不可人工編輯 |
| 空值 | 欄位存在，但此訂單類型通常不填 |
| 無 | 此訂單類型不適用此欄位 |

## Section 4：付款資訊

付款資訊分為兩層：**付款計畫**（定義分幾期收、各期金額與時程）和**收款記錄**（實際每筆收款明細）。每張訂單可有多筆付款計畫與多筆收款記錄。

### 付款計畫（每期收款安排）

來源：PaymentPlan 子表（一張訂單可有多筆，如訂金 30%、尾款 70%）

| 欄位名稱 | 英文名稱 | 說明 | 線上單（EC） | 線下單 | 備注 |
|---------|---------|------|-------------|--------|------|
| 期別 | installment_no | 第幾期（1、2、3…）| 手動填寫 | 手動填寫 | 業務建立付款計畫時手動設定各期 |
| 描述 | description | 期別說明 | 手動填寫 | 手動填寫 | 如「訂金 30%」、「尾款 70%」|
| 預定金額 | scheduled_amount | 此期應收金額（含稅）| 手動填寫 | 手動填寫 | 各期合計 MUST 等於訂單應收總額，不符時系統顯示差額警告 |
| 預計收款日 | scheduled_date | 此期預計收款日期 | 手動填寫 | 手動填寫 | 空白時不顯示逾期天數 |
| 狀態 | status | 此期收款進度 | 自動更新 | 自動更新 | 未收 / 部分收款 / 已收訖；系統依關聯收款記錄自動計算 |
| 逾期天數 | overdue_days | 距預計收款日的逾期天數 | 自動計算（唯讀）| 自動計算（唯讀）| 狀態為「已收訖」時不顯示；預計收款日為空時不顯示 |

## Section 5：發票資訊

發票管理分為兩層：**發票**（電子發票本體）和**折讓單**（發票開立後的部分減額憑證）。每張訂單可開立多張發票，每張發票可建立多筆折讓單。付款記錄與發票解耦，可先開後收、後收先開，亦支援多筆收款對一張發票或一筆收款拆多張發票。

### 發票（電子發票）
